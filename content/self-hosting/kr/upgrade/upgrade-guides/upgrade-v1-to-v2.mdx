# Langfuse v1을 v2로 마이그레이션

Langfuse v2([릴리스](https://github.com/langfuse/langfuse/releases/tag/v2.0.0) 2024년 1월 30일)는 LLM 생성(generation)에 대한 사용량 및 비용 추적 시스템을 새롭게 구축한 Langfuse의 주요 릴리스입니다. 이 업데이트는 기존 트레이스의 정확한 LLM 비용을 보장하기 위해 과거 데이터에 대해 일회성 마이그레이션 스크립트를 실행해야 합니다.

## 변경 사항

### 무엇이 변경되었나요?

- LLM 생성(generation)을 위한 사용량/비용 추적 시스템을 완전히 새롭게 구축
- 다음을 지원하는 새로운 모델 정의 추상화:
  - 새롭게 등장하는 모델에 대한 빠른 지원
  - 시간에 따른 모델 가격 변경 추적
  - 프로젝트 수준의 커스텀 모델/가격
- 트레이스 수집 시 API를 통해 사용량 및 비용을 설정할 수 있는 기능 추가
- 모든 UI 테이블 및 API에서 사용량 및 비용 정보 확인 가능

### 무엇이 변경되지 않았나요?

API 및 인프라 구성 요소를 포함한 나머지 모든 부분은 동일하게 유지됩니다. 호환성을 깨는(breaking) 변경 사항은 없습니다.

## 업그레이드 중 누가 조치를 취해야 하나요?

- 다음의 경우 **조치가 필요하지 않습니다**:
  - Langfuse Cloud를 사용하는 경우
  - 새로 수집된 트레이스에만 관심이 있는 경우
  - 비용 추적 기능을 사용하지 않는 경우

- 다음의 경우 **조치가 필요합니다**:
  - Langfuse를 셀프 호스팅하는 경우
  - 과거 트레이스에 대한 정확한 비용 데이터를 원하는 경우

## 마이그레이션 단계

<Callout type="info">
  이 프로세스는 비차단(non-blocking) 방식이며 Langfuse 배포의 가용성에
  영향을 미치지 않습니다.
</Callout>

<Steps>

### Langfuse를 v2로 업데이트

배포 가이드를 따라 Langfuse 배포를 v2로 업그레이드합니다.

- 프로덕션 배포의 경우 [업그레이드 가이드](/self-hosting/v2/deployment-guide#update)를 참조하세요
- docker compose를 사용하는 경우 [업그레이드 가이드](/self-hosting/v2/docker-compose)를 참조하세요

### 새로운 모델 로직과 가격을 기존 데이터에 적용

Langfuse는 [사용량 및 비용 추적](/docs/model-usage-and-cost)을 위해 지원되는 모델 목록을 포함하고 있습니다. Langfuse 업데이트에 새로운 모델에 대한 지원이 포함된 경우, 이는 새로 수집된 트레이스/생성(generation)에만 적용됩니다.

원한다면 다음 단계를 통해 새로운 모델 정의를 기존 데이터에도 적용할 수 있습니다. 마이그레이션 중에도 데이터베이스는 계속 사용 가능한 상태(비차단)로 유지됩니다.

1. 저장소를 클론하고 `.env` 파일을 생성합니다:

   ```bash
   # Langfuse 저장소를 클론합니다
   git clone https://github.com/langfuse/langfuse.git

   # Langfuse 디렉터리로 이동합니다
   cd langfuse

   # Langfuse v2 브랜치로 체크아웃합니다
   git checkout v2

   # 모든 의존성을 설치합니다
   pnpm i

   # .env 파일을 생성합니다
   cp .env.dev.example .env
   ```

2. 로컬 머신에서 데이터베이스에 연결하도록 `.env`를 수정합니다:

   ```bash filename=".env"
   NODE_ENV=production

   # 여러분의 데이터베이스 연결 문자열로 교체하세요
   DATABASE_URL=postgresql://postgres:postgres@localhost:5432/postgres
   ```

3. 마이그레이션을 실행합니다. 데이터베이스 크기에 따라 시간이 다소 걸릴 수 있습니다.

   ```bash
   pnpm run models:migrate
   ```

4. 정리: 로컬 머신에서 프로덕션 데이터베이스에 연결되는 것을 방지하기 위해 `.env` 파일을 삭제합니다.

</Steps>
