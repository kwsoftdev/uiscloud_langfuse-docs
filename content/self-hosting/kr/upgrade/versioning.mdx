# 버전 관리

버전 관리는 Langfuse Server, SDK, 그리고 Public API를 통한 사용자 정의 통합 간의 호환성을 보장하는 데 핵심적인 역할을 합니다. 이러한 이유로 우리는 [시맨틱 버저닝(semantic versioning)](https://semver.org/)을 중요하게 다룹니다.

## 시맨틱 버저닝의 적용 범위

다음과 같은 변경 사항은 호환성을 깨뜨리는 것으로 간주되어 **메이저 버전이 상승**합니다:

- 인프라 변경
- 기존 Public API의 제거, 또는 기존 Public API 파라미터의 제거/변경

다음과 같은 변경 사항은 내부 구현 세부 사항으로 간주되어 **메이저 버전이 상승하지 않습니다**:

- 데이터베이스 스키마
- 프론트엔드 API

## Langfuse Server와 SDK 간의 호환성

Langfuse Server와 SDK는 구성 요소를 더 유연하게 업그레이드할 수 있도록 서로 독립적으로 버전이 관리됩니다:

- **Server**: 릴리스 노트에 명시적으로 언급되지 않는 한, SDK 버전과 무관하게 독립적으로 업그레이드할 수 있습니다
- **SDK**: 더 최신 버전의 서버를 실행하는 동안에도 이전 버전을 그대로 유지할 수 있습니다
- **호환성**: 새로운 SDK 기능은 최신 서버 버전을 필요로 할 수 있습니다

모든 기능과 보안 업데이트에 접근할 수 있도록 Langfuse Server를 항상 최신 상태로 유지하는 것을 권장합니다.
