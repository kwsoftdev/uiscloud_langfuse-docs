# 캐싱 기능

Langfuse는 성능을 개선하고 데이터베이스 부하를 줄이기 위해 API 키와 프롬프트에 대한 내장 캐싱 기능을 제공합니다.
이 기능들은 기본적으로 활성화되어 있으며, 필요에 따라 설정을 변경하거나 비활성화할 수 있습니다 (v3.81.0부터).

## 개요

Langfuse의 캐싱은 자주 사용되는 데이터를 Redis에 저장하여 반복적인 데이터베이스 쿼리의 필요성을 줄임으로써 성능을 최적화합니다. 이는 특히 다음과 같은 경우에 유용합니다:

- **API 키 검증**: 모든 API 요청에 대한 데이터베이스 조회를 줄입니다
- **프롬프트 조회**: 동일한 프롬프트에 자주 접근하는 애플리케이션의 프롬프트 가져오기 속도를 높입니다

## API 키 캐싱

API 키 캐싱은 인증 과정에서 반복적인 데이터베이스 조회를 피하기 위해 검증된 API 키를 Redis에 저장합니다.
이를 통해 수집(ingestion) 요청의 응답 시간이 크게 개선됩니다.
API 키는 절대 평문으로 저장되지 않으며, 해시 값만 캐시됩니다.

### 기본 동작

- **상태**: 기본적으로 활성화됨
- **TTL**: 300초 (5분)
- **범위**: 모든 API 키 검증

### 설정

| 변수                                 | 기본값 | 설명                             |
| ------------------------------------ | ------ | -------------------------------- |
| `LANGFUSE_CACHE_API_KEY_ENABLED`     | `true` | API 키 캐싱 활성화 또는 비활성화 |
| `LANGFUSE_CACHE_API_KEY_TTL_SECONDS` | `300`  | 캐시된 API 키의 유효 시간(초)    |

### API 키 캐싱 비활성화

API 키 캐싱을 비활성화하려면 다음 환경 변수를 설정하세요:

```bash
LANGFUSE_CACHE_API_KEY_ENABLED=false
```

## 프롬프트 캐싱

프롬프트 캐싱은 동일한 프롬프트를 정기적으로 가져오는 애플리케이션의 조회 성능을 개선하기 위해 자주 사용되는 프롬프트를 Redis에 저장합니다.

### 기본 동작

- **상태**: 기본적으로 활성화됨
- **TTL**: 300초 (5분)
- **범위**: API를 통한 모든 프롬프트 조회

### 설정

| 변수                                | 기본값 | 설명                               |
| ----------------------------------- | ------ | ---------------------------------- |
| `LANGFUSE_CACHE_PROMPT_ENABLED`     | `true` | 프롬프트 캐싱 활성화 또는 비활성화 |
| `LANGFUSE_CACHE_PROMPT_TTL_SECONDS` | `300`  | 캐시된 프롬프트의 유효 시간(초)    |

### 프롬프트 캐싱 비활성화

프롬프트 캐싱을 비활성화하려면 다음 환경 변수를 설정하세요:

```bash
LANGFUSE_CACHE_PROMPT_ENABLED=false
```

## 캐시 무효화

Langfuse는 데이터 일관성을 보장하기 위해 캐시 무효화를 자동으로 처리합니다:

- **API 키**: 키가 삭제, 비활성화되거나 권한이 변경되면 캐시된 API 키가 무효화됩니다
- **프롬프트**: 프롬프트가 업데이트, 삭제되거나 새 버전이 게시되면 캐시된 프롬프트가 무효화됩니다

## 이전 버전에서의 마이그레이션

<Callout type="info">
  이전 버전의 Langfuse에서 업그레이드하는 경우, 이 캐싱 기능들은 기본 설정으로
  자동 활성화됩니다. 동작을 커스터마이즈하고 싶은 경우가 아니라면 별도의
  설정은 필요하지 않습니다.
</Callout>
