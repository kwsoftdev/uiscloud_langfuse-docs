# 헬스 체크 및 레디니스 체크 엔드포인트

Langfuse는 셀프 호스팅한 배포의 상태(health)와 준비 상태(readiness)를 확인할 수 있는 모니터링 엔드포인트를 제공합니다. 이 엔드포인트는 로드 밸런서, 오케스트레이션 시스템, 모니터링 도구에서 필수적으로 사용됩니다.

## 개요

| 컨테이너            | 엔드포인트           | 목적                                           |
| ------------------- | -------------------- | ---------------------------------------------- |
| **langfuse-web**    | `/api/public/health` | 웹 서비스가 정상 상태인지 확인                 |
| **langfuse-web**    | `/api/public/ready`  | 웹 서비스가 트래픽을 받을 준비가 되었는지 확인 |
| **langfuse-worker** | `/api/health`        | 워커 서비스가 정상 상태인지 확인               |

## 헬스 체크 엔드포인트

헬스 체크는 애플리케이션이 실행 중이며 정상적으로 동작하고 있는지를 확인합니다.

### 웹 컨테이너 헬스 체크

```bash
curl http://localhost:3000/api/public/health
```

**기본 동작:**

- 기본적으로 이 엔드포인트는 API가 실행 중인지만 확인합니다
- 데이터베이스가 일시적으로 사용할 수 없는 상태여도 트래픽을 계속 처리할 수 있도록, 데이터베이스 연결 상태는 검증하지 **않습니다**
- 헬스 체크에 데이터베이스 연결 상태를 포함하려면 다음과 같이 쿼리 파라미터를 추가합니다.
  ```bash
  curl http://localhost:3000/api/public/health?failIfDatabaseUnavailable=true
  ```

**응답 코드:**

- `200 OK` - API가 정상적으로 동작 중 (파라미터를 사용한 경우 데이터베이스에도 정상적으로 연결됨)
- `503 Service Unavailable` - API가 정상적으로 동작하지 않거나 (파라미터를 사용한 경우) 데이터베이스에 연결할 수 없음

### 워커 컨테이너 헬스 체크

```bash
curl http://localhost:3030/api/health
```

**응답 코드:**

- `200 OK` - 워커 서비스가 정상적으로 동작 중이며 데이터베이스 연결에 성공함
- `503 Service Unavailable` - 워커 서비스가 정상적으로 동작하지 않거나 데이터베이스에 연결할 수 없음

## 레디니스 체크 엔드포인트

레디니스 체크는 웹 애플리케이션이 트래픽을 받을 준비가 되었는지를 나타내며, 특히 그레이스풀 셧다운(graceful shutdown) 과정에서 유용합니다.

```bash
curl http://localhost:3000/api/public/ready
```

**응답 코드:**

- `200 OK` - 애플리케이션이 트래픽을 처리할 준비가 됨
- `500 Internal Server Error` - 애플리케이션이 종료 신호(SIGTERM 또는 SIGINT)를 수신하여 새로운 트래픽을 받아서는 안 되는 상태
