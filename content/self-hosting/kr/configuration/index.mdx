# 환경 변수

Langfuse(셀프 호스팅)는 환경 변수를 통해 다양한 설정 옵션을 제공합니다. 이 값들은 모든 애플리케이션 컨테이너에 전달되어야 합니다.

| 변수                                            | 필수 여부 / 기본값 | 설명                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------------------------------------------- | ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `DATABASE_URL`                                  | 필수               | Postgres 데이터베이스의 연결 문자열입니다. `DATABASE_URL` 대신 `DATABASE_HOST`, `DATABASE_USERNAME`, `DATABASE_PASSWORD`, `DATABASE_NAME`, `DATABASE_ARGS`를 사용할 수도 있습니다.                                                                                                                                                                                                                                         |
| `DIRECT_URL`                                    | `DATABASE_URL`     | 데이터베이스 마이그레이션에 사용되는 Postgres 데이터베이스의 연결 문자열입니다. 마이그레이션에 다른 사용자를 사용하려 하거나 `DATABASE_URL`에서 커넥션 풀링을 사용하는 경우 이 값을 사용하세요. **대규모 배포의 경우**, 마이그레이션 완료까지 시간이 걸릴 수 있으므로 타임아웃이 긴 데이터베이스 사용자를 구성하세요.                                                                                                      |
| `SHADOW_DATABASE_URL`                           |                    | 데이터베이스 사용자에게 `CREATE DATABASE` 권한이 없는 경우, 섀도우 데이터베이스를 생성하고 "SHADOW_DATABASE_URL"을 구성해야 합니다. 클라우드 데이터베이스를 사용하는 경우 흔히 발생합니다. 자세한 안내는 [Prisma 문서](https://www.prisma.io/docs/orm/prisma-migrate/understanding-prisma-migrate/shadow-database#cloud-hosted-shadow-databases-must-be-created-manually)를 참고하세요.                                    |
| `CLICKHOUSE_MIGRATION_URL`                      | 필수               | ClickHouse 인스턴스의 마이그레이션 URL(TCP 프로토콜)입니다. 패턴: `clickhouse://<hostname>:(9000/9440)`                                                                                                                                                                                                                                                                                                                    |
| `CLICKHOUSE_MIGRATION_SSL`                      | `false`            | 데이터베이스 마이그레이션 시 ClickHouse에 SSL 연결을 설정하려면 `true`로 설정하세요.                                                                                                                                                                                                                                                                                                                                       |
| `CLICKHOUSE_URL`                                | 필수               | ClickHouse 인스턴스의 호스트명입니다. 패턴: `http(s)://<hostname>:(8123/8443)`                                                                                                                                                                                                                                                                                                                                             |
| `CLICKHOUSE_USER`                               | 필수               | ClickHouse 데이터베이스의 사용자 이름입니다. SELECT, ALTER, INSERT, CREATE, DELETE 권한이 필요합니다.                                                                                                                                                                                                                                                                                                                      |
| `CLICKHOUSE_PASSWORD`                           | 필수               | ClickHouse 사용자의 비밀번호입니다.                                                                                                                                                                                                                                                                                                                                                                                        |
| `CLICKHOUSE_DB`                                 | `default`          | 사용할 ClickHouse 데이터베이스의 이름입니다.                                                                                                                                                                                                                                                                                                                                                                               |
| `CLICKHOUSE_CLUSTER_ENABLED`                    | `true`             | ClickHouse 명령을 `ON CLUSTER`로 실행할지 여부입니다. 단일 컨테이너 구성에서는 `false`로 설정하세요.                                                                                                                                                                                                                                                                                                                       |
| `LANGFUSE_AUTO_CLICKHOUSE_MIGRATION_DISABLED`   | `false`            | 시작 시 자동 ClickHouse 마이그레이션을 비활성화할지 여부입니다.                                                                                                                                                                                                                                                                                                                                                            |
| `CLICKHOUSE_READ_ONLY_URL`                      |                    | 트레이스, observation, 점수, 세션에 대한 UI 및 공개 API 읽기 쿼리에 사용되는 선택적 읽기 전용 엔드포인트입니다. 설정하지 않으면 `CLICKHOUSE_URL`을 사용합니다. `CLICKHOUSE_USER`, `CLICKHOUSE_PASSWORD`, `CLICKHOUSE_DB`를 그대로 재사용합니다. 컴퓨팅과 스토리지가 분리된 ClickHouse 클러스터(ClickHouse Cloud, BYOC)에서 특히 유용합니다. [스케일링](/self-hosting/configuration/scaling#clickhouse-read-only-url) 참고. |
| `REDIS_CONNECTION_STRING`                       | 필수               | Redis 인스턴스의 연결 문자열입니다. `REDIS_CONNECTION_STRING` 대신 `REDIS_HOST`, `REDIS_PORT`, `REDIS_USERNAME`, `REDIS_AUTH`를 사용할 수도 있습니다. TLS를 구성하려면 자세한 [캐시 설정 문서](/self-hosting/deployment/infrastructure/cache#configuration)를 확인하세요.                                                                                                                                                  |
| `REDIS_CLUSTER_ENABLED`                         | `false`            | Redis 클러스터 모드를 활성화하려면 `true`로 설정하세요. 활성화하는 경우 `REDIS_CLUSTER_NODES`도 함께 제공해야 합니다.                                                                                                                                                                                                                                                                                                      |
| `REDIS_CLUSTER_NODES`                           |                    | `host:port` 형식의 Redis 클러스터 노드 목록으로, 쉼표로 구분합니다. `REDIS_CLUSTER_ENABLED`가 `true`일 때 필수입니다. 예: `redis-node1:6379,redis-node2:6379,redis-node3:6379`                                                                                                                                                                                                                                             |
| `REDIS_CLUSTER_SLOTS_REFRESH_TIMEOUT`           | `5000`             | Redis 클러스터 슬롯 새로고침 작업의 타임아웃(밀리초)입니다. 클러스터 토폴로지 업데이트 중 타임아웃 오류가 발생하면 이 값을 늘리세요.                                                                                                                                                                                                                                                                                       |
| `REDIS_SENTINEL_ENABLED`                        | `false`            | Redis Sentinel 모드를 활성화하려면 `true`로 설정하세요. 클러스터 모드와 동시에 활성화할 수 없습니다. 활성화하는 경우 `REDIS_SENTINEL_NODES`와 `REDIS_SENTINEL_MASTER_NAME`도 함께 제공해야 합니다.                                                                                                                                                                                                                         |
| `REDIS_SENTINEL_NODES`                          |                    | `host:port` 형식의 Redis Sentinel 노드 목록으로, 쉼표로 구분합니다. `REDIS_SENTINEL_ENABLED`가 `true`일 때 필수입니다. 예: `sentinel1:26379,sentinel2:26379,sentinel3:26379`                                                                                                                                                                                                                                               |
| `REDIS_SENTINEL_MASTER_NAME`                    |                    | Redis Sentinel 마스터의 이름입니다. `REDIS_SENTINEL_ENABLED`가 `true`일 때 필수입니다. Sentinel 구성에 설정된 마스터 이름과 일치해야 합니다.                                                                                                                                                                                                                                                                               |
| `REDIS_SENTINEL_USERNAME`                       |                    | Redis Sentinel 인증을 위한 사용자 이름입니다(선택 사항). Sentinel이 인증을 요구하는 경우 사용됩니다.                                                                                                                                                                                                                                                                                                                       |
| `REDIS_SENTINEL_PASSWORD`                       |                    | Redis Sentinel 인증을 위한 비밀번호입니다(선택 사항). Sentinel이 인증을 요구하는 경우 사용됩니다.                                                                                                                                                                                                                                                                                                                          |
| `REDIS_SENTINEL_TLS_ENABLED`                    | `false`            | Redis 마스터/레플리카 연결뿐 아니라 Sentinel 노드로의 연결에도 TLS를 적용하려면 `true`로 설정하세요. `REDIS_TLS_ENABLED=true`가 필요하며, 그렇지 않으면 무시됩니다. [캐시 설정](/self-hosting/deployment/infrastructure/cache#redis-sentinel-mode) 참고.                                                                                                                                                                   |
| `REDIS_SOCKET_TIMEOUT_MS`                       | `30000`            | BullMQ 큐 워커를 포함한 모든 Redis 연결의 소켓 타임아웃(밀리초)입니다. 비활성화하려면 `0`으로 설정하고, 양수 값을 사용하는 경우 `BZPOPMIN`과 같은 블로킹 작업을 처리할 수 있도록 최소 `10000` 이상이어야 합니다. [캐시 설정](/self-hosting/deployment/infrastructure/cache#timeout-configuration) 참고.                                                                                                                    |
| `REDIS_AUTH`                                    |                    | Redis 인스턴스 또는 클러스터에 대한 인증 문자열입니다.                                                                                                                                                                                                                                                                                                                                                                     |
| `NEXTAUTH_URL`                                  | 필수               | Langfuse 웹 배포의 URL입니다. 예: `https://yourdomain.com` 또는 `http://localhost:3000`. OAUTH를 통한 인증 성공과 Slack 통합을 통한 유효한 링크 전송에 필요합니다.                                                                                                                                                                                                                                                         |
| `NEXTAUTH_SECRET`                               | 필수               | 로그인 세션 쿠키를 검증하는 데 사용됩니다. `openssl rand -base64 32`를 사용해 최소 256비트 엔트로피를 가진 시크릿을 생성하세요.                                                                                                                                                                                                                                                                                            |
| `SALT`                                          | 필수               | 해시된 API 키를 솔트 처리하는 데 사용됩니다. `openssl rand -base64 32`를 사용해 최소 256비트 엔트로피를 가진 시크릿을 생성하세요.                                                                                                                                                                                                                                                                                          |

| `LANGFUSE_CSP_ENFORCE_HTTPS`                    | `false`            | HTTPS 연결만 허용하도록 CSP 헤더를 설정하려면 `true`로 설정하세요.                                                                                                                                                                                                                                                                                                                                                         |
| `PORT`                                          | `3000` / `3030`    | 서버가 리스닝하는 포트입니다. 웹은 3000, 워커는 3030입니다.                                                                                                                                                                                                                                                                                                                                                                |
| `HOSTNAME`                                      | `localhost`        | 일부 환경(예: Google Cloud Run)에서는 컨테이너 외부에서 접근 가능하도록 `0.0.0.0`으로 설정해야 합니다.                                                                                                                                                                                                                                                                                                                     |
| `LANGFUSE_CACHE_API_KEY_ENABLED`                | `true`             | API 키 캐싱을 활성화 또는 비활성화합니다. `false`로 설정하면 API 키 캐싱이 비활성화됩니다. 평문 키는 Redis에 저장되지 않으며, 해시되거나 암호화된 키만 저장됩니다.                                                                                                                                                                                                                                                         |

| `LANGFUSE_AUTO_POSTGRES_MIGRATION_DISABLED`     | `false`            | Docker 시작 시 자동 데이터베이스 마이그레이션을 비활성화하려면 `true`로 설정하세요. 권장하지 않습니다.                                                                                                                                                                                                                                                                                                                     |
| `LANGFUSE_LOG_LEVEL`                            | `info`             | 애플리케이션의 로그 레벨을 설정합니다. 가능한 값은 `trace`, `debug`, `info`, `warn`, `error`, `fatal`입니다.                                                                                                                                                                                                                                                                                                               |
| `LANGFUSE_LOG_FORMAT`                           | `text`             | 애플리케이션의 로그 형식을 설정합니다. 가능한 값은 `text`, `json`입니다.                                                                                                                                                                                                                                                                                                                                                   |
| `LANGFUSE_LOG_PROPAGATED_HEADERS`               |                    | OpenTelemetry baggage를 통해 로그에 전파할 HTTP 헤더 이름 목록으로, 쉼표로 구분합니다. 헤더 이름은 대소문자를 구분하지 않으며 소문자로 정규화됩니다. 디버깅 및 관측성에 유용합니다. 예: `x-request-id,x-user-id`                                                                                                                                                                                                           |
| `LANGFUSE_MCP_ALLOWED_HOSTS`                    |                    | 리버스 프록시가 `NEXTAUTH_URL`과 다른 `Host` 헤더를 전달할 때 MCP 엔드포인트가 허용할 추가 호스트명 또는 오리진 목록으로, 쉼표로 구분합니다. 가능하면 프록시에서 공개 호스트를 유지하는 것을 권장합니다. 값은 정확한 호스트명 또는 오리진이어야 하며, 와일드카드와 경로는 지원되지 않습니다.                                                                                                                               |
