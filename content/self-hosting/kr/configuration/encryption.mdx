# 암호화

<Callout type="info">

보안과 개인정보 보호는 Langfuse의 핵심 설계 목표입니다. Langfuse 팀은 ISO27001, SOC2 Type 2, HIPAA, GDPR을 준수하는 Langfuse Cloud에서 프로덕션 환경으로 Langfuse를 운영하고 있습니다 ([Langfuse Cloud 보안 페이지](/security), [보고서 요청 양식](/request-security-docs)).

</Callout>

이 가이드는 다양한 암호화 방법과 고려 사항을 다룹니다.
Langfuse의 [아키텍처](/self-hosting#architecture)에 대해 이미 알고 있다고 가정합니다.

## 전송 중 암호화 (HTTPS) [#https]

전송 중 암호화를 위해 HTTPS 사용을 강력히 권장합니다.
Langfuse 자체는 HTTPS를 직접 처리하지 않습니다.
대신 HTTPS는 일반적으로 인프라 수준에서 관리됩니다.
Langfuse에서 HTTPS를 처리하는 두 가지 주요 방법이 있습니다:

1. 로드 밸런서 종료(Termination):
   이 방식에서는 로드 밸런서 수준에서 HTTPS가 종료됩니다.
   로드 밸런서가 SSL/TLS 인증서와 암호화를 처리한 뒤, 복호화된 트래픽을 HTTP를 통해 Langfuse 컨테이너로 전달합니다.
   이는 특히 클라우드 환경에서 일반적이고 간단한 방법입니다.

- 장점: 일반적으로 완전 관리형 서비스(예: AWS ALB)이므로 인증서 관리가 간소화되고, 애플리케이션 서버에서 암호화 부담을 덜어줍니다.
- 단점: 로드 밸런서와 Langfuse 컨테이너 간의 트래픽은 암호화되지 않습니다(다만 일반적으로 안전한 네트워크 내부에 있습니다).

2. 서비스 메시 사이드카:
   이 방법은 Istio나 Linkerd와 같은 서비스 메시를 사용합니다.
   각 Langfuse 컨테이너와 함께 사이드카 프록시가 배포되어 HTTPS를 포함한 모든 네트워크 트래픽을 처리합니다.

- 장점: 종단 간 암호화(상호 TLS)를 제공하며, 고급 트래픽 관리 및 관측성을 제공합니다.
- 단점: 배포에 복잡성이 추가되며, 서비스 메시 개념에 대한 이해가 필요합니다.

HTTPS가 활성화되면 `LANGFUSE_CSP_ENFORCE_HTTPS=true`를 설정하여 Langfuse 사용 시 브라우저가 HTTPS 연결만 허용하도록 구성할 수 있습니다.

## 저장 시 암호화 (데이터베이스) [#encryption-at-rest]

모든 Langfuse 데이터는 Postgres 데이터베이스, Clickhouse, Redis, S3/Blob Store에 저장됩니다.
안전한 프로덕션 배포를 위해서는 데이터베이스 수준 암호화를 권장하며, 이는 여러 클라우드 제공업체에서 사용할 수 있습니다.

Langfuse Cloud에서는 모든 데이터베이스에 걸쳐 AES-256을 사용합니다.

ClickHouse에서 저장 시 암호화를 활성화하는 방법에 대한 자세한 내용은 [ClickHouse 암호화 문서](/self-hosting/deployment/infrastructure/clickhouse#encryption)를 참고하세요.
Postgres, Redis, S3/Blob Storage의 경우, 일반적으로 저장 시 암호화가 내장된 클라우드 제공업체의 관리형 서비스를 사용하는 것을 권장합니다.
자체 관리형 컨테이너 배포의 경우, 해당 데이터베이스 시스템의 문서를 참고하세요.

## 추가 애플리케이션 수준 암호화 [#application-level-encryption]

전송 중 암호화 및 저장 시 암호화 외에도, 민감한 데이터는 애플리케이션 수준에서도 암호화되거나 해시 처리됩니다.

| 데이터                              | 암호화                             |
| ----------------------------------- | ---------------------------------- |
| API 키                              | `SALT`를 사용하여 해시 처리        |
| Langfuse 콘솔 JWT                   | `NEXTAUTH_SECRET`을 통해 암호화    |
| Langfuse에 저장된 LLM API 자격 증명 | `ENCRYPTION_KEY`를 사용하여 암호화 |
| 통합 자격 증명 (예: PostHog)        | `ENCRYPTION_KEY`를 사용하여 암호화 |
