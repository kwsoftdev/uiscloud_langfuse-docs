# 트랜잭션 이메일

선택적으로 SMTP 서버를 구성하여 트랜잭션 이메일을 전송할 수 있습니다.
이는 비밀번호 재설정, 프로젝트/조직 초대, 배치 내보내기 완료 시 알림 등에 사용됩니다.

## 구성

트랜잭션 이메일을 활성화하려면 애플리케이션 컨테이너에 다음 환경 변수를 설정하세요.

| 변수                  | 설명                                                                                                                                                                                                                                                                                                                    |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `SMTP_CONNECTION_URL` | 트랜잭션 이메일을 위한 연결 URL입니다. Nodemailer([문서](https://nodemailer.com/smtp))로 전달되는 표준 SMTP URL(`smtp://` 또는 `smtps://`)을 사용하거나, 기본 AWS 자격 증명 체인(IAM 역할, SSO, 환경 변수)을 사용해 [AWS SES](#aws-ses)를 통해 전송하려면 `ses://<region>`(예: `ses://us-east-1`)을 사용할 수 있습니다. |
| `EMAIL_FROM_ADDRESS`  | 트랜잭션 이메일의 발신 주소를 구성합니다. `SMTP_CONNECTION_URL`이 설정된 경우 필수입니다.                                                                                                                                                                                                                               |

## AWS SES [#aws-ses]

`SMTP_CONNECTION_URL`이 `ses://<region>`으로 설정되면, Langfuse는 SESv2 API를 사용해 [AWS SES](https://aws.amazon.com/ses/)를 통해 트랜잭션 이메일을 전송합니다.
추가 구성 변수는 필요하지 않습니다 — 자격 증명은 [기본 AWS 자격 증명 체인](https://docs.aws.amazon.com/sdkref/latest/guide/standardized-credentials.html)(IAM 역할, IRSA, ECS 작업 역할, SSO, `AWS_PROFILE`, 또는 환경 변수)을 통해 확인되며, 이는 [S3 블롭 스토리지](/self-hosting/deployment/infrastructure/blobstorage)에 사용되는 자격 증명 처리 방식과 동일합니다.

Langfuse가 AWS에서 실행 중이라면, SES를 통한 전송 권한이 있는 IAM 역할을 Langfuse 컨테이너에 연결하는 것을 권장합니다.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Action": ["ses:SendEmail"],
      "Effect": "Allow",
      "Resource": "*",
      "Sid": "LangfuseSendEmail"
    }
  ]
}
```

`EMAIL_FROM_ADDRESS`가 동일한 리전에서 [검증된 자격](https://docs.aws.amazon.com/ses/latest/dg/creating-identities.html)임을 확인하고, 임의의 수신자에게 이메일을 보내야 하는 경우 계정이 SES 샌드박스에서 벗어나 있는지 확인하세요.

구성 예시:

```yaml
SMTP_CONNECTION_URL=ses://us-east-1
EMAIL_FROM_ADDRESS=no-reply@example.com
```

## FAQ

- **어떤 SMTP 서비스를 사용해야 하나요?** 전달률을 보장하고 남용을 방지하기 위해 트랜잭션 이메일에는 평판이 좋은 SMTP 서비스를 사용하는 것이 좋습니다. 클라우드 제공업체에서 선호하는 서비스가 없다면 [Resend](https://resend.com/) 또는 [Postmark](https://postmarkapp.com/)를 사용해 보세요. 두 서비스 모두 설정이 쉽고 넉넉한 무료 요금제를 제공합니다.
- **개인 메일함을 사용할 수 있나요?** 아니요, GMail과 같은 개인 메일함은 일반적으로 권장되지 않으며 올바르게 구성하기 어렵습니다.
