# 자동 액세스 프로비저닝

선택적으로 신규 사용자를 위한 자동 액세스 프로비저닝을 구성할 수 있습니다.
이를 통해 신규 사용자는 가입 시 특정 역할과 함께 기본 조직 및 프로젝트에 자동으로 추가됩니다.

사용 가능한 역할, 스코프, 조직/프로젝트에 대한 자세한 내용은 [RBAC 문서](/docs/rbac)를 참고하세요.

<Callout type="info">

사용자 프로비저닝 프로세스를 더 프로그래밍 방식으로 제어하려면 [Admin API (SCIM)](/docs/admin-api)를 사용할 수도 있습니다.

</Callout>

## 구성

애플리케이션 컨테이너에 다음 환경 변수를 설정하세요.

| 변수                            | 필수 여부 / 기본값 | 설명                                                                                                                                                           |
| ------------------------------- | ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `LANGFUSE_DEFAULT_ORG_ID`       |                    | 신규 사용자를 위한 선택적 기본 조직을 구성합니다. 사용자가 계정을 생성하면 이 조직에 자동으로 추가됩니다.                                                      |
| `LANGFUSE_DEFAULT_ORG_ROLE`     | `VIEWER`           | 기본 조직에서의 사용자 역할입니다(설정된 경우). 가능한 값은 `OWNER`, `ADMIN`, `MEMBER`, `VIEWER`, `NONE`입니다. 자세한 내용은 [역할](/docs/rbac)을 참고하세요. |
| `LANGFUSE_DEFAULT_PROJECT_ID`   |                    | 신규 사용자를 위한 선택적 기본 프로젝트를 구성합니다. 사용자가 계정을 생성하면 이 프로젝트에 자동으로 추가됩니다.                                              |
| `LANGFUSE_DEFAULT_PROJECT_ROLE` | `VIEWER`           | 기본 프로젝트에서의 사용자 역할입니다(설정된 경우). 가능한 값은 `OWNER`, `ADMIN`, `MEMBER`, `VIEWER`입니다. 자세한 내용은 [역할](/docs/rbac)을 참고하세요.     |
