# Kubernetes (Helm)

이 가이드에서는 Helm 패키지 관리자를 사용하여 Kubernetes에 Langfuse를 배포하는 단계를 안내합니다.
Kubernetes 클러스터에 대한 액세스 권한과 로컬 머신에 설치된 Helm이 필요합니다.
이 가이드에서는 로컬 minikube 인스턴스를 사용하지만, 각 단계는 GKE, EKS, AKS와 같은 관리형 Kubernetes 서비스로도 확장할 수 있습니다.

기본적으로 이 차트는 Langfuse 애플리케이션 컨테이너와 데이터 스토어([아키텍처 개요](/self-hosting#architecture))를 배포합니다.
선택적으로 기존의 PostgreSQL, Clickhouse, Redis 인스턴스를 가리키도록 설정할 수 있습니다.
자세한 내용은 [Readme](https://github.com/langfuse/langfuse-k8s/blob/main/README.md)를 참조하세요.

## Helm 차트 가져오기 및 values 사용자 지정

`langfuse-k8s` Helm 차트를 가져옵니다.

```bash
helm repo add langfuse https://langfuse.github.io/langfuse-k8s
helm repo update
```

로컬에서 테스트해 보는 용도라면 values.yaml 파일에 사전 구성된 변수만으로도 대개 충분합니다.

애플리케이션에 민감한 데이터를 조금이라도 전송하거나 더 오래 운영할 계획이라면, 필요에 맞게
values.yaml 파일을 수정하는 것을 권장합니다.

사용 가능한 모든 환경 변수와 구성 옵션에 대한 자세한 개요는 [구성 가이드](/self-hosting/configuration)와 [Readme](https://github.com/langfuse/langfuse-k8s/blob/main/README.md)를 참조하세요.

## Helm 차트 배포

Langfuse 배포를 위한 새 네임스페이스를 생성합니다(선택 사항). 예:

```bash
kubectl create namespace langfuse
```

네임스페이스에 Helm 차트를 설치합니다:

```bash
helm install langfuse langfuse/langfuse -n langfuse
```

이 차트는 `langfuse`라는 이름으로 설치되는 것을 전제로 합니다.
다른 이름으로 설치하려면 `values.yaml`에서 Redis 호스트명을 그에 맞게 조정해야 합니다.

이 시점에서 Kubernetes는 Langfuse 애플리케이션과 그 종속 요소들을 배포하기 시작합니다.
이 작업은 최대 5분 정도 걸릴 수 있습니다.
`kubectl get pods -n langfuse` 명령으로 진행 상황을 모니터링할 수 있으며, 결국 모든 파드가 실행 중인 상태가 되어야 합니다.
데이터베이스가 프로비저닝되는 동안 langfuse-web 및 langfuse-worker 컨테이너는 몇 차례 재시작됩니다.

## UI 스모크 테스트

Langfuse UI는 기본적으로 ClusterIP 서비스로 제공됩니다.
`kubectl get services -n langfuse`를 사용하여 `langfuse-web`을 검색하면 포트 매핑을 확인할 수 있습니다.
`kubectl port-forward svc/langfuse-web -n langfuse <local-port>:<nodeport>`로 포트 포워딩을 생성한 후, 브라우저에서 `http://localhost:<local-port>`로 UI에 접속합니다.
회원가입을 진행하고 새 조직과 프로젝트를 생성하여 Langfuse를 살펴보세요.

## 종료

리소스를 정리하려면 Helm 릴리스와 네임스페이스를 삭제하면 됩니다:

```bash
helm uninstall langfuse -n langfuse
kubectl delete namespace langfuse
```

## 업그레이드 방법

다음 명령을 실행하여 Helm 차트를 최신 버전으로 업그레이드합니다:

```bash
helm repo update
helm upgrade langfuse langfuse/langfuse -n langfuse
```

업그레이드에 대한 자세한 내용은 [업그레이드 가이드](/self-hosting/upgrade)를 참조하세요.
