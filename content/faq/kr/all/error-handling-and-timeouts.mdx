# 프롬프트를 가져올 때 재시도와 타임아웃을 설정하는 방법은?

기본적으로 Langfuse SDK는 프롬프트 가져오기 작업이 실패하면 두 번 재시도하며 20초의 타임아웃을 적용합니다. 더 엄격하거나 더 느슨한 설정이 필요하다면 두 값을 모두 재정의할 수 있습니다.

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab>

```python
# 서버에서 프롬프트를 가져올 때의 재시도 횟수입니다. 기본값은 2입니다.
prompt = langfuse.get_prompt("movie-critic", max_retries=3)

# Langfuse API 호출당 타임아웃(초)입니다. 기본값은 20입니다.
prompt = langfuse.get_prompt("movie-critic", fetch_timeout_seconds=3)
```

</Tab>

<Tab>

```ts
// 서버에서 프롬프트를 가져올 때의 재시도 횟수입니다. 기본값은 2입니다.
const promptWithMaxRetries = await langfuse.getPrompt(
  "movie-critic",
  undefined,
  {
    maxRetries: 5,
  }
);

// Langfuse API 호출당 타임아웃(밀리초)입니다. 기본값은 10초입니다.
const promptWithFetchTimeout = await langfuse.getPrompt(
  "movie-critic",
  undefined,
  {
    fetchTimeoutMs: 5000,
  }
);
```

</Tab>

</LangTabs>

위 예제는 프롬프트 조회를 위한 신뢰성 관련 매개변수를 세밀하게 조정하는 방법을 보여줍니다. 애플리케이션 요구 사항에 맞게 복원력과 지연 시간의 균형을 고려하여 값을 조정하세요.
