# 셀프 호스팅 Langfuse에서 발생하는 PostgreSQL 테이블 소유권 및 마이그레이션 실패

다음과 같은 PostgreSQL 마이그레이션 중 권한 오류가 발생하는 경우, 일반적으로 마이그레이션을 실행하는 데이터베이스 사용자가 변경되었기 때문입니다.
새 사용자는 기존 테이블을 소유하지 않으므로 해당 테이블을 수정할 수 없습니다.

```
DbError { severity: "ERROR", code: SqlState(E42501), message: "permission denied for table projects" }
```

테이블 소유자를 확인하여 소유권이 혼재되어 있는지 확인할 수 있습니다.

```sql
\dt public.*
```

## 해결 방법

**옵션 1: 테이블 소유권 이전**

```sql
-- 'new_user'를 현재 마이그레이션 사용자로 교체하세요
ALTER TABLE table_name OWNER TO new_user;
```

**옵션 2: 전용 마이그레이션 사용자 사용**

마이그레이션 전용으로 슈퍼유저 또는 테이블 소유자를 사용하도록 `DIRECT_URL` 환경 변수를 설정합니다.

```bash
DIRECT_URL=postgresql://migration_user:password@host:port/database
```

이렇게 하면 Prisma가 마이그레이션에는 다른 자격 증명을 사용하는 동시에, 애플리케이션 작업에는 기존의 `DATABASE_URL`을 계속 사용할 수 있습니다.
