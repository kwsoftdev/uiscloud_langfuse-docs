# Langfuse 프롬프트 관리로 에이전트 스킬을 관리할 수 있나요?

프롬프트 관리는 원래 단일 프롬프트 파일을 위해 설계되었습니다. 핵심 프리미티브(버전 관리, 라벨, 조회, 트레이스와의 연결)는 스킬에도 잘 들어맞지만, 여러 스킬 파일을 하나로 묶어 관리하거나, 이를 세트로 탐색하거나, 특정 에이전트 실행에서 어떤 스킬 조합이 활성화되었는지 파악하는 부분에는 아직 공백이 있습니다. 저희는 이러한 워크플로우를 위한 더 네이티브한 지원 방안을 검토하고 있습니다.

**_스킬 관리를 위해 Langfuse를 사용 중이거나 사용을 고려하고 계시다면, [이 GitHub Discussion](https://github.com/orgs/langfuse/discussions/12290)에서 투표하고 여러분의 구성, 어려움, 아이디어를 공유해 주세요. 저희는 이 의견들을 바탕으로 Langfuse의 스킬 지원 방향을 결정합니다._**

그동안은 아래 권장 사항이 실제로 가장 잘 작동하는 것으로 확인된 방법입니다.

## 오늘 권장하는 패턴

### 스킬 파일당 프롬프트 하나

각 스킬 파일을 개별 Langfuse 프롬프트로 취급하세요. 프롬프트 이름으로 스킬을 식별하고(예: `skill/code-review`, `skill/data-analysis`), 스킬 파일 전체 내용을 프롬프트 본문으로 저장합니다. 이렇게 하면 스킬별로 Langfuse의 전체 버전 관리, 라벨(예: `production`, `staging`), 롤백 기능을 활용할 수 있습니다.

### 각 트레이스에서 사용된 스킬 버전 추적하기

에이전트 실행은 종종 여러 스킬 파일을 함께 불러오므로, 특정 트레이스를 만들어낸 스킬 버전 조합을 정확히 재구성할 수 있어야 합니다. 가장 간단한 방법은 다음과 같습니다.

1. `get_prompt()`로 스킬 파일을 가져올 때 반환된 버전 번호를 캡처합니다.
2. 해석된 버전 정보를 스킬 이름과 버전의 맵 형태로 [트레이스 메타데이터](/docs/observability/features/metadata)에 기록합니다.
3. 이후 실행을 디버깅하거나 평가할 때, 메타데이터를 읽어 어떤 스킬 파일이 정확히 활성화되어 있었는지 확인할 수 있습니다.

Python으로 작성한 최소 예제입니다.

```python
from langfuse import observe, get_client, propagate_attributes

langfuse = get_client()

@observe()
def run_agent(user_input: str):
    skills_to_load = ["skill/code-review", "skill/data-analysis"]
    skills = {name: langfuse.get_prompt(name) for name in skills_to_load}

    # 이번 실행에서 사용된 스킬 버전을 기록합니다.
    # 전파되는(propagated) 메타데이터 키는 영숫자여야 하고 값은 문자열이어야 하므로,
    # 프롬프트 이름을 스킬별 키로 평탄화합니다.
    skill_version_metadata = {
        f"skill_{name.replace('skill/', '').replace('-', '_')}_version": str(prompt.version)
        for name, prompt in skills.items()
    }

    with propagate_attributes(metadata=skill_version_metadata):
        # ... skills[name].prompt를 에이전트 내 스킬 파일 내용으로 사용합니다 ...
        pass
```

### 스킬을 트레이스와 연결하기

각 스킬 프롬프트 객체를 해당하는 generation 또는 span에 전달하여 Langfuse UI에서 연결된 형태로 표시되게 할 수도 있습니다. [프롬프트를 트레이스와 연결하기](/docs/prompt-management/features/link-to-traces)를 참고하세요. 이는 하나의 스킬 파일이 특정 모델 호출과 직접 대응될 때 유용합니다. 여러 스킬을 사전에 불러오는 에이전트의 경우, 위에서 설명한 메타데이터 방식이 더 실용적인 경향이 있습니다.

## 알아두어야 할 제약 사항

- **아직 네이티브한 폴더 단위 그룹화는 지원하지 않습니다.** 각 파일은 독립적인 프롬프트로 저장됩니다. 스킬 폴더(`SKILL.md`와 참조 파일들)나 여러 스킬의 묶음을 하나의 단위로 관리하려면 현재는 애플리케이션 코드에서 직접 처리해야 합니다.
- **Playground와 프롬프트 실험**은 단일 프롬프트를 기준으로 설계되어 있습니다. 스킬 파일 하나를 반복 개선하는 데는 적합하지만, 여러 스킬로 구성된 에이전트 설정 전체를 다루는 데는 적합하지 않습니다.
- **변수 감지**는 여전히 Langfuse의 `{{variable}}` 문법을 사용합니다. 스킬 파일이 다른 템플릿 스타일을 사용한다면 [외부 템플릿 라이브러리 사용하기](/faq/all/using-external-templating-libraries)를 참고하세요.

## 의견 공유하기

스킬 관리를 위해 Langfuse 프롬프트 관리를 사용 중이거나 도입을 고려하고 계시다면, [이 GitHub Discussion](https://github.com/orgs/langfuse/discussions/12290)에 무엇이 잘 작동하는지, 무엇이 아쉬운지, 네이티브 스킬 지원에 무엇을 기대하는지 댓글로 남겨 주세요.
