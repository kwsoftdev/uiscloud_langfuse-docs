# 챗봇 분석: Langfuse로 AI 챗봇을 개선하는 방법

<Frame fullWidth>
  ![Chatbot Analytics](/images/blog/faq/chatbot-analytics.png)
</Frame>

AI로 구동되는 챗봇은 고객 지원부터 개인 비서에 이르기까지 다양한 애플리케이션에서 사용자 상호작용의 핵심 요소가 되었습니다. 많은 이점을 제공하지만, 이러한 AI 기반 시스템을 모니터링하는 것은 효과적으로 작동하고, 컴플라이언스를 유지하며, 긍정적인 사용자 경험을 제공하도록 보장하는 데 중요합니다.

## AI 챗봇의 과제

대규모 언어 모델(LLM)은 예측하기 어렵거나 사용자 의도와 어긋나는 응답을 생성할 수 있습니다. 예를 들면 다음과 같습니다.

- **일관성 없는 응답**: 사용자 질문과 무관하거나 주제에서 벗어난 답변을 제공합니다.
- **컴플라이언스 문제**: 규제 기준이나 회사 정책을 위반하는 정보를 공유합니다.
- **부정적인 사용자 경험**: 사용자를 불쾌하게 하거나 화나게 할 수 있는 부적절하거나 불쾌한 콘텐츠를 생성합니다.
- **성능 병목**: 실시간 상호작용을 저해하는 지연이나 오류가 발생합니다.

## 챗봇 시스템의 관측성

관측성(observability)을 통해 개발자는 시스템의 출력을 바탕으로 내부 상태를 파악할 수 있습니다. 챗봇의 경우 다음을 의미합니다.

- **대화 모니터링**: 사용자 상호작용을 추적하여 챗봇이 적절하게 응답하는지 확인합니다.
- **성능 메트릭 분석**: 응답 시간, 오류율, 사용자 참여 수준을 평가합니다.
- **이상 징후 감지**: 반복적인 오류나 부적절한 응답과 같은 문제를 나타낼 수 있는 비정상적인 패턴을 식별합니다.
- **컴플라이언스 보장**: 상호작용이 법적·조직적 가이드라인을 준수하는지 자동으로 확인합니다.

관측성을 구현하면 챗봇 시스템의 신뢰성과 효과성을 유지하는 데 도움이 됩니다.

## 챗봇을 위한 모니터링 전략

챗봇을 평가하고 개선하려면 다음 전략을 도입하는 것을 고려하세요.

### 1. 로깅 및 트레이싱

사용자 대화와 시스템 프로세스에 대한 상세한 로그를 수집합니다. 이 데이터는 문제를 진단하고 사용자 상호작용을 이해하는 데 필수적입니다.

- **대화 로그**: 사용자 입력과 챗봇 응답을 기록합니다.
- **시스템 트레이스**: API 호출과 데이터베이스 쿼리를 포함한 내부 프로세스를 모니터링합니다.

Langfuse로 [로깅 및 트레이싱](/docs/observability/overview)을 설정하는 방법을 알아보세요.

### 2. 피드백 통합

챗봇 인터페이스 내에서 사용자가 직접 피드백을 제공할 수 있는 메커니즘을 도입합니다.

- **사용자 피드백 수집**: 사용자가 응답을 평가하거나 문제를 신고할 수 있도록 합니다.
- **피드백 분석**: 수집된 데이터를 활용해 챗봇 성능을 개선합니다.

Langfuse는 사용자 피드백을 수집하고 분석하는 데 도움을 줍니다. [자세히 알아보기](/docs/observability/features/user-feedback).

### 3. 분석 대시보드

핵심 메트릭을 시각화하고 시간에 따른 챗봇 성능을 모니터링합니다.

- **비용 메트릭**: LLM 사용 비용을 모니터링합니다.
- **참여 메트릭**: 상호작용 수, 세션 지속 시간, 사용자 유지율을 추적합니다.
- **성능 메트릭**: 응답 시간과 오류율을 모니터링합니다.

Langfuse에서 챗봇 메트릭을 [분석](/docs/metrics/overview)하는 방법을 살펴보세요.

### 4. 모델 기반 평가

자동화된 평가(LLM-as-a-judge)를 사용해 챗봇 응답의 품질을 평가합니다.

- **자동 채점**: 모델 기반 점수 산정을 구현하여 관련성, 일관성, 정확성을 기준으로 응답을 평가합니다.
- **지속적인 모니터링**: 챗봇이 품질 기준을 충족하는지 정기적으로 평가합니다.
- **벤치마킹**: 챗봇 성능을 사전 정의된 벤치마크와 비교합니다.

Langfuse는 모델 기반 평가를 설정하고 관리할 수 있습니다—[자세히 알아보기](/docs/evaluation/evaluation-methods/llm-as-a-judge).

### 5. 외부 평가 파이프라인

[OpenAI Evaluations](https://github.com/openai/evals)나 RAG 파이프라인을 위한 [Ragas](https://docs.ragas.io/en/stable/)와 같은 외부 평가 프레임워크로 챗봇 응답을 평가하는 파이프라인을 구축합니다.

- **품질 점수 산정**: 관련성, 정확성, 어조를 기준으로 응답을 평가합니다.
- **컴플라이언스 확인**: 응답이 규제 기준을 충족하는지 자동으로 검증합니다.
- **프롬프트 인젝션**: 조작된 입력을 통해 챗봇 응답을 조종하려는 시도를 감지하고 완화합니다.

Langfuse는 평가 메트릭을 한데 모아줍니다—[여기서 시작하기](/docs/evaluation/evaluation-methods/scores-via-sdk).

## 관측성 도구

관측성 도구는 다음과 같은 방식으로 챗봇을 더욱 개선하고 확장하는 데 도움을 줍니다.

### 1. 통합 데이터 집계

로그, 메트릭, 트레이스를 한 곳에 중앙 집중화하여 더 쉽게 분석할 수 있습니다.

- **데이터 중앙화**: 여러 시스템을 모니터링하는 복잡성을 줄입니다.
- **연관 인사이트**: 시스템의 여러 부분이 서로 어떻게 영향을 미치는지 파악합니다.

### 2. 프롬프트 관리

AI 모델이 사용하는 프롬프트를 효과적으로 관리하여 일관되고 관련성 있는 출력을 보장합니다.

- **프롬프트 버전 관리**: 시간에 따른 프롬프트 변경 사항을 추적합니다.
- **최적화**: 어떤 프롬프트가 최상의 응답을 만들어내는지 분석합니다.

[Langfuse의 프롬프트 관리](/docs/prompt-management/get-started)를 확인해보세요.

### 3. 사용자 행동 분석

사용자가 챗봇과 어떻게 상호작용하는지 파악합니다.

- **상호작용 패턴**: 일반적인 사용자 의도와 요구를 식별합니다.
- **이탈 분석**: 사용자가 어디서 이탈하는지 파악합니다.

[사용자 행동 추적](/docs/metrics/overview)에 대해 자세히 알아보세요.

### 4. 통합

기존 기술 스택에 모니터링 도구를 통합합니다.

- **SDK 및 API**: 제공된 도구를 사용해 빠르게 통합합니다.
- **PostHog**: Langfuse의 PostHog [통합](/integrations/analytics/posthog)을 활용해 데이터를 한 곳에서 분석합니다.

Langfuse는 다양한 [통합 및 SDK](/integrations)를 제공합니다.

### 5. 확장성

모니터링 솔루션이 성장에 대응할 수 있도록 합니다.

- **성능 최적화**: 데이터 양이 증가해도 효율성을 유지합니다.
- **분산 시스템 지원**: 여러 서버나 리전에 배포된 챗봇을 모니터링합니다.

Langfuse는 여러분의 요구에 맞춰 확장할 수 있도록 설계되었습니다.

## Langfuse로 챗봇 추적 시작하기

[`@observe()` 데코레이터](/docs/sdk/python/decorators)를 사용하면 모든 Python LLM 애플리케이션을 손쉽게 트레이싱할 수 있습니다. 이 퀵스타트에서는 Langfuse의 [OpenAI 통합](/integrations/model-providers/openai-py)을 사용해 모든 모델 파라미터를 자동으로 캡처합니다.

OpenAI를 사용하지 않으시나요? [Langfuse로 모든 LLM을 트레이싱하는 방법](/docs/observability/get-started)을 확인해보세요.

1.  [Langfuse 계정 생성](https://cloud.langfuse.com/auth/sign-up) 또는 [셀프 호스팅](/self-hosting)
2.  새 프로젝트 생성
3.  프로젝트 설정에서 새 API 자격 증명 생성

```bash
pip install langfuse openai
```

```python
LANGFUSE_SECRET_KEY="sk-lf-..."
LANGFUSE_PUBLIC_KEY="pk-lf-..."
# 🇪🇺 EU region
LANGFUSE_BASE_URL="https://cloud.langfuse.com"
# Other Langfuse data regions include 🇺🇸 US: https://us.cloud.langfuse.com, 🇯🇵 Japan: https://jp.cloud.langfuse.com and ⚕️ HIPAA: https://hipaa.cloud.langfuse.com
```

```python
from langfuse import observe
from langfuse.openai import openai # OpenAI integration

@observe()
def story():
    return openai.chat.completions.create(
        model="gpt-4o-mini",
        max_tokens=100,
        messages=[
          {"role": "system", "content": "You are a great storyteller."},
          {"role": "user", "content": "Once upon a time in a galaxy far, far away..."}
        ],
    ).choices[0].message.content

@observe()
def main():
    return story()

main()
```

## 주요 챗봇 분석 메트릭

챗봇을 효과적으로 측정하고 개선하려면 다음 핵심 메트릭을 추적하세요.

| 메트릭                 | 측정 대상                                     | 중요한 이유                                        |
| ---------------------- | --------------------------------------------- | -------------------------------------------------- |
| **세션당 메시지 수**   | 대화당 평균 메시지 수                         | 참여 깊이와 대화 복잡도를 나타냅니다               |
| **세션 지속 시간**     | 사용자가 챗봇과 상호작용하는 시간             | 사용자 참여도와 만족도를 파악하는 데 도움이 됩니다 |
| **해결률**             | 사용자 목표를 달성한 대화의 비율              | 챗봇 효과성을 직접적으로 측정합니다                |
| **에스컬레이션율**     | 대화가 상담원에게 인계되는 빈도               | 챗봇 역량의 부족한 부분을 파악합니다               |
| **응답 지연 시간**     | 사용자 메시지와 챗봇 응답 사이의 시간         | 사용자 경험과 유지율에 결정적입니다                |
| **대화당 비용**        | 세션당 토큰 사용량 및 관련 비용               | 예산 책정과 최적화에 필수적입니다                  |
| **사용자 만족도 점수** | 사용자의 직접적인 피드백(좋아요/싫어요, 평점) | 챗봇 품질을 있는 그대로 측정합니다                 |
| **환각 발생률**        | 챗봇이 부정확한 정보를 생성하는 빈도          | 신뢰와 컴플라이언스에 결정적입니다                 |

Langfuse에서 [커스텀 대시보드](/docs/metrics/features/custom-dashboards)를 사용해 이러한 메트릭을 시간과 사용자 세그먼트별로 시각화하세요.

## 챗봇 분석 도구

적합한 챗봇 분석 도구는 무엇을 측정해야 하는지, 챗봇이 어떻게 구축되었는지에 따라 달라집니다. 주요 카테고리는 다음과 같습니다.

- **LLM 관측성 플랫폼** — [Langfuse](/), [LangSmith](https://smith.langchain.com/), [Arize Phoenix](https://phoenix.arize.com/)와 같은 도구는 LLM 기반 챗봇에 대한 심층적인 트레이싱과 모니터링을 제공하며, 모든 모델 호출, 검색 단계, 도구 상호작용을 캡처합니다.
- **제품 분석** — [PostHog](https://posthog.com/)와 Mixpanel 같은 도구는 사용자 행동 패턴, 퍼널 분석, 참여 메트릭을 추적하는 데 도움이 됩니다. Langfuse는 [PostHog](/integrations/analytics/posthog)와 통합되어 LLM 분석과 제품 분석을 결합할 수 있습니다.
- **평가 프레임워크** — [Ragas](https://docs.ragas.io/)와 [Promptfoo](https://www.promptfoo.dev/) 같은 도구는 자동화된 평가 파이프라인을 통해 응답 품질을 평가하는 데 도움이 됩니다.
- **피드백 수집** — 내장된 피드백 위젯과 [사용자 피드백 수집](/docs/observability/features/user-feedback) 기능으로 사용자의 직접적인 평점과 코멘트를 수집합니다.

대부분의 팀에게는 Langfuse와 같은 LLM 관측성 플랫폼이 기반이 되며, 구체적인 분석 요구가 발생함에 따라 추가 도구를 도입하게 됩니다.

## 챗봇 분석 플랫폼 선택하기

챗봇 분석 플랫폼을 평가할 때는 다음과 같은 핵심 기능을 고려하세요.

- **대화 트레이싱** — 모든 LLM 호출, 검색 단계, 도구 호출을 포함해 각 대화의 전체 흐름을 확인할 수 있나요?
- **세션 그룹화** — 관련된 메시지를 [세션](/docs/observability/features/sessions)으로 그룹화하여 멀티턴 대화를 분석할 수 있나요?
- **평가 지원** — 플랫폼이 LLM-as-a-Judge를 포함한 응답 품질의 [자동화된 평가](/docs/evaluation/evaluation-methods/llm-as-a-judge)를 지원하나요?
- **비용 추적** — 대화별, 사용자별로 [토큰 사용량과 비용](/docs/observability/features/token-and-cost-tracking)을 추적할 수 있나요?
- **커스텀 대시보드** — 팀에 중요한 메트릭을 시각화하는 [커스텀 대시보드](/docs/metrics/features/custom-dashboards)를 만들 수 있나요?
- **통합 지원** — 플랫폼이 여러분의 기술 스택과 함께 작동하나요? Langfuse는 [LangChain](/integrations/frameworks/langchain), [LlamaIndex](/integrations/frameworks/llamaindex), [OpenAI](/integrations/model-providers/openai-py) 등 [다양한 통합](/integrations)을 지원합니다.
- **오픈소스 및 셀프 호스팅** — 데이터 프라이버시와 컴플라이언스 요구사항을 위해 플랫폼을 [셀프 호스팅](/self-hosting)할 수 있나요?

<Summary>챗봇 분석이란 무엇인가요?</Summary>

챗봇 분석은 AI 챗봇의 데이터를 수집, 측정, 분석하여 성능을 파악하고 사용자 경험을 개선하는 작업입니다. 여기에는 응답 품질, 대화 길이, 사용자 만족도, 대화당 비용, 오류율과 같은 메트릭 추적이 포함됩니다. 적절한 분석을 통해 챗봇이 어려움을 겪는 지점을 파악하고, 프롬프트를 최적화하며, 데이터 기반으로 개선할 수 있습니다.

</Details>

<Details>
<Summary>어떤 챗봇 분석 도구를 사용해야 하나요?</Summary>

최적의 도구 조합은 여러분의 요구에 따라 다릅니다. LLM 기반 챗봇의 경우, 대화 트레이싱, 비용 추적, 품질 평가를 위해 Langfuse와 같은 LLM 관측성 플랫폼부터 시작하세요. 사용자 행동 분석을 위해 PostHog와 같은 제품 분석 도구를, 자동화된 품질 평가를 위해 Ragas와 같은 평가 프레임워크를 추가하세요. 대부분의 팀은 2~3개 도구의 조합으로 분석 요구를 충족한다고 느낍니다.

</Details>

<Details>
<Summary>챗봇 성능은 어떻게 측정하나요?</Summary>

챗봇 성능은 여러 차원에서 측정합니다. (1) **품질** — LLM-as-a-Judge 평가와 사용자 피드백 점수를 사용합니다. (2) **효율성** — 응답 지연 시간, 대화당 비용, 토큰 사용량을 추적합니다. (3) **효과성** — 해결률, 에스컬레이션율, 사용자 만족도를 측정합니다. (4) **안전성** — 환각 발생률, 컴플라이언스 위반, 프롬프트 인젝션 시도를 모니터링합니다. 커스텀 대시보드를 사용해 시간에 따른 추이를 시각화하세요.

</Details>

<Details>
<Summary>챗봇 응답을 어떻게 개선하나요?</Summary>

먼저 챗봇 분석을 통해 문제 영역을 파악하세요. 대화 트레이싱을 사용해 품질이 낮은 응답을 확인하고 실패한 이유를 파악합니다. 자동화된 평가를 설정하여 유용성, 정확성, 어조 등 핵심 차원에서 응답 점수를 매깁니다. 문제가 있는 대화들로 데이터셋을 구축하고, 프롬프트 개선안을 프로덕션에 배포하기 전에 실험을 실행하여 테스트하세요. 프로덕션 성능을 지속적으로 모니터링하고 엣지 케이스를 테스트 데이터셋에 다시 반영하세요.

</Details>
