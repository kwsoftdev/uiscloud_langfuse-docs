# 조건에 따라 삽입할 서브 프롬프트를 동적으로 선택할 수 있나요?

공유 지침이 포함된 메인 프롬프트가 있고, 사용자가 무료 플랜인지 프리미엄 플랜인지에 따라 서로 다른 지침 하위 집합을 삽입하고 싶다고 가정해 보겠습니다.

[Langfuse 프롬프트 참조](/docs/prompt-management/features/composability)는 `{% if %}` 문과 같은 조건부 로직을 직접 지원하지 않습니다. 하지만 애플리케이션 코드에서 로직을 처리하고 선택된 프롬프트를 [변수](/docs/prompt-management/features/variables) 값으로 전달하면 동적 프롬프트 선택을 구현할 수 있습니다.

**구체적인 방법을 살펴보기 전에, 먼저 한 걸음 물러나 어떤 옵션들이 있는지 살펴보겠습니다.**

| 방식                                                                                 | 사용 시점                                        | 트레이드오프                                                                                                                                                                             |
| ------------------------------------------------------------------------------------ | ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [**정적 프롬프트 참조**](/docs/prompt-management/features/composability)             | 서브 프롬프트가 항상 동일한 경우                 | ✅ 완전히 선언적이며 Playground에서 자동으로 해석됨<br/>❌ 런타임 값에 기반한 동적 선택 불가                                                                                             |
| **변수를 통한 동적 선택** (본 문서)                                                  | 서브 프롬프트가 런타임 조건에 따라 달라지는 경우 | ✅ 모든 (서브) 프롬프트가 Langfuse UI에서 관리됨<br/>✅ (코드를 통한) 선택 로직 지원<br/>✅ Langfuse가 변수를 자동으로 감지<br/>❌ Playground와 프롬프트 실험에서 자동으로 해석되지 않음 |
| [**외부 템플릿 엔진**](/faq/all/using-external-templating-libraries) (Jinja, Liquid) | 단일 템플릿 내에서 복잡한 조건문이 필요한 경우   | ✅ 로직 측면에서 완전한 유연성<br/>❌ Playground와 프롬프트 실험에서 프롬프트가 자동으로 해석되지 않음 <br/>❌ Langfuse가 변수를 자동으로 감지하지 못함                                  |

## 작동 방식

[프롬프트 조합 기능](/docs/prompt-management/features/composability)(`@@@langfusePrompt:...@@@`)으로 프롬프트를 정적으로 삽입하는 대신, 여러 프롬프트를 가져온 뒤 애플리케이션 코드에서 사용할 프롬프트를 선택합니다.

1. [변수](/docs/prompt-management/features/variables) 플레이스홀더(예: `{{selected_prompt}}`)가 포함된 부모 프롬프트를 생성합니다
2. 조건에 따라 삽입할 서브 프롬프트들을 생성합니다
3. `get_prompt()`를 사용해 모든 프롬프트를 가져옵니다
4. 로직에 따라 적절한 서브 프롬프트를 선택합니다
5. 선택된 프롬프트 텍스트를 부모 프롬프트의 `compile()` 메서드에 전달합니다

다음은 Python 예시입니다.

```python
from langfuse import Langfuse

langfuse = Langfuse()

# Langfuse에 등록된 부모 프롬프트:
# "Here is your final prompt:\n\n{{selected_prompt}}"

# 프로덕션 버전 가져오기
parent_prompt = langfuse.get_prompt("parent-prompt")
prompt_a = langfuse.get_prompt("prompt-a")
prompt_b = langfuse.get_prompt("prompt-b")

# 불리언 또는 다른 조건에 따라 선택
use_prompt_a = True  # 여기에 로직 작성
selected = prompt_a if use_prompt_a else prompt_b

# 부모 프롬프트에 삽입
final_prompt = parent_prompt.compile(selected_prompt=selected.prompt)
```
