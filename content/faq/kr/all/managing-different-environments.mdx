# Langfuse에서 서로 다른 환경을 관리하는 방법은?

Langfuse는 프로덕션, 스테이징, 개발과 같은 환경을 관리하는 두 가지 방법을 제공합니다: 내장 환경 기능 또는 별도의 프로젝트입니다.

## 어떤 방식을 사용해야 하나요?

팀이 모든 환경에 대한 접근 권한을 공유할 수 있다면 **내장 Environments 기능**을 사용하세요. 프롬프트, 데이터셋, 평가기 및 기타 설정이 프로젝트 내에서 자동으로 공유되어 환경 간 동기화가 필요 없기 때문에 이 방법을 권장합니다.

프로덕션 데이터를 소수의 팀원으로 제한하는 등 환경별로 서로 다른 접근 제어가 필요하다면 **별도의 프로젝트**를 사용하세요. 이 방식은 프로젝트 간 프롬프트와 설정을 동기화하기 위한 추가 설정이 필요하지만, 완전한 격리를 제공합니다.

## Environments (권장)

Langfuse에서 서로 다른 환경을 관리하는 데 권장되는 방법은 내장 [Environments](/docs/observability/features/environments) 기능을 사용하는 것입니다. 이를 통해 동일한 프로젝트 내에서 프로덕션, 스테이징, 개발 등 서로 다른 컨텍스트의 트레이스, observation, 점수를 정리할 수 있습니다.

`LANGFUSE_TRACING_ENVIRONMENT` 환경 변수를 설정하거나 클라이언트 초기화 시 `environment` 매개변수를 사용하여 환경을 구성할 수 있습니다.
Python SDK에서는 LLM 프록시와 같은 공유 서비스가 요청별로 호출자의 환경을 상속받아야 할 때 `propagate_attributes(environment="...", as_baggage=True)`를 사용하세요.

## 별도의 프로젝트

환경마다 서로 다른 [역할 기반 접근 제어(RBAC)](/docs/rbac)를 지정해야 한다면, 환경마다 별도의 프로젝트를 사용하는 것을 고려하세요. 접근 관리를 단순화하기 위해 모든 환경을 동일한 조직 내에 그룹화할 수 있습니다. 각 프로젝트는 자체적인 멤버, 프롬프트, API 키 집합을 갖습니다.

### 프로젝트 간 프롬프트 동기화

환경마다 별도의 프로젝트를 사용하는 경우, 프롬프트 및 기타 설정은 자동으로 공유되지 않습니다. 이를 동기화하는 방법에는 두 가지 옵션이 있습니다.

[GitHub 통합](/docs/prompt-management/features/github-integration)을 사용하면 저장소에 프롬프트를 저장하고 이를 서로 다른 Langfuse 프로젝트에 자동으로 배포할 수 있습니다. 이를 통해 프롬프트가 버전 관리되고 기존 Git 워크플로우를 통해 개발에서 스테이징, 프로덕션으로 승격되는 CI/CD 워크플로우를 구현할 수 있습니다.

커스텀 워크플로우의 경우, [Prompt Management API](/docs/prompt-management)를 사용하여 한 프로젝트에서 프롬프트를 프로그래밍 방식으로 읽어 다른 프로젝트에 쓸 수 있습니다. 이는 자체적인 승격 스크립트를 구축하거나 내부 도구와 통합하려는 경우에 유용합니다.
