# Langfuse vs. Datadog

이 가이드는 엔지니어링 팀이 적합한 LLM 관측성 구성을 선택할 수 있도록 **[Langfuse](/)**와 **[Datadog LLM Observability](https://docs.datadoghq.com/llm_observability/)**(2026년 7월 기준 Datadog 문서에서는 Agent Observability로 표기됨)의 주요 차이점을 정리합니다.

**요약:**

- LLM 트레이싱, 평가, 런타임 프롬프트 관리, 데이터셋/실험을 하나의 목적에 특화된 도구에서 사용하고 싶고, 완전히 셀프 호스팅할 수 있는 오픈소스(MIT) 플랫폼과 월 $29부터 시작하는 단위 기반 가격 정책을 원한다면 **Langfuse를 선택하세요**.
- 조직이 이미 Datadog APM을 운영 중이고 나머지 인프라와 상관관계를 맺은 LLM 로그가 주로 필요하다면 **Datadog을 선택하세요**.
- **많은 팀이 두 도구를 함께 사용합니다**: 애플리케이션 및 인프라 모니터링에는 Datadog을, LLM 평가 워크플로에는 Langfuse를 사용하며, 동일한 OpenTelemetry 계측에서 데이터를 공급받습니다. [Langfuse와 Datadog을 함께 사용하기](#coexistence)를 참고하세요.

## 오픈소스 및 배포 방식

Langfuse는 오픈소스(MIT)이며 [셀프 호스팅](/self-hosting)은 Langfuse Cloud와 동일한 코드베이스를 사용하는 퍼스트클래스 배포 옵션입니다. Datadog은 독점 SaaS 플랫폼으로, LLM Observability를 포함한 제품들이 Datadog이 호스팅하는 지역별 사이트에서 실행되며 데이터도 Datadog의 인프라에 저장됩니다([출처](https://docs.datadoghq.com/getting_started/site/), 2026년 7월 확인).

| 기능            | Langfuse                                                                                                                                                                            | Datadog                                                                                                                     |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **모델**        | **[오픈소스](https://github.com/langfuse/langfuse)** (MIT 라이선스)                                                                                                                 | **독점 SaaS** (비공개 소스)                                                                                                 |
| **GitHub 스타** | [![Langfuse GitHub stars](https://img.shields.io/github/stars/langfuse/langfuse?style=for-the-badge&label=%20&labelColor=black&color=orange)](https://github.com/langfuse/langfuse) | 해당 없음                                                                                                                   |
| **셀프 호스팅** | **퍼스트클래스 지원**: AWS, Azure, GCP용 Docker Compose, Kubernetes(Helm), Terraform 템플릿 제공.                                                                                   | **미지원**: 플랫폼은 Datadog 호스팅 SaaS로만 운영됩니다(Datadog Agent는 로컬에서 실행되지만 데이터는 로컬에 남지 않습니다). |
| **데이터 주권** | **높음**: 자체 VPC 또는 온프레미스에서 완전히 에어갭 상태로 운영 가능.                                                                                                              | **지역 선택 가능**: 9개 호스팅 사이트(미국, EU, 일본, 호주, 영국, US-gov) 중 선택; 데이터는 Datadog에 저장됩니다.           |

셀프 호스팅이나 에어갭 배포가 필수 요건이라면 이 항목만으로 비교가 이미 결정됩니다. Datadog은 셀프 호스팅 플랫폼을 제공하지 않습니다(2026년 7월 확인).

## LLM 및 에이전트 트레이싱

두 플랫폼 모두 토큰 사용량, 지연 시간, 오류, 비용을 포함한 LLM 애플리케이션 및 에이전트의 계층적 트레이스를 캡처합니다. 차이는 범위와 주변 컨텍스트에 있습니다: Langfuse는 [LLM 관측성에 특화되어](/docs/observability/overview) 100개 이상의 프레임워크와 모델 제공업체를 지원하는 반면, Datadog은 LLM 트레이스를 자사의 범용 관측성 플랫폼에 통합해 APM 서비스, 로그, 실제 사용자 세션과 상관관계를 맺습니다.

| 기능                | Langfuse                                                                                  | Datadog                                                                                                                       |
| ------------------- | ----------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **SDK**             | **Python 및 JS/TS**, OpenTelemetry 기반 네이티브 구현.                                    | **Python, Node.js, Java**, 자동 및 수동 계측 지원([출처](https://docs.datadoghq.com/llm_observability/instrumentation/sdk/)). |
| **자동 계측**       | OpenAI, LangChain, LlamaIndex, Vercel AI SDK, LiteLLM, CrewAI를 포함한 100개 이상의 통합. | OpenAI, Anthropic, LangChain, AWS Bedrock, Gemini, Vertex AI, CrewAI, Strands Agents 등을 포함한 통합.                        |
| **그 외 언어**      | Java, Go 및 OTLP 엔드포인트를 통해 OTel을 지원하는 모든 스택.                             | SDK를 지원하지 않는 언어를 위한 HTTP API.                                                                                     |
| **플랫폼 컨텍스트** | 세션, 사용자, 환경, 릴리스, 에이전트 그래프 뷰.                                           | Datadog APM, 인프라 메트릭, 로그, RUM 세션과의 상관관계.                                                                      |

## 평가

두 플랫폼 모두 실제로 동작하는 평가 루프를 제공하지만, 평가가 실행되는 위치와 확장 범위에서 차이가 있습니다. [Langfuse 평가](/docs/evaluation/overview)에는 관리형 LLM-as-a-judge 평가기, 코드 기반 평가기, SDK/API를 통한 커스텀 점수, 사람 주석 큐, 데이터셋 실험이 포함되며 이 모든 기능은 셀프 호스팅 배포에서도 동일하게 동작합니다. Datadog은 LLM-as-a-judge 템플릿(Hallucination, Failure to Answer, Toxicity, Prompt Injection 포함), 자연어로 정의하는 커스텀 LLM-as-a-judge 평가기, API를 통해 제출하는 외부 평가, 주석 큐를 제공합니다([출처](https://docs.datadoghq.com/llm_observability/evaluations/), 2026년 7월 확인).

| 기능                 | Langfuse                                                               | Datadog                                                      |
| -------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------ |
| **LLM-as-a-judge**   | 트레이싱된 모든 데이터에 적용 가능한 관리형 평가기, 툴 콜 평가기 포함. | 템플릿 및 커스텀 판정 평가기.                                |
| **코드 기반 평가기** | 결정론적 평가기를 네이티브로 실행.                                     | API 또는 서드파티 프레임워크(예: DeepEval)를 통한 외부 실행. |
| **사람 검토**        | 주석 큐.                                                               | 주석 큐.                                                     |
| **데이터셋/실험**    | UI, SDK, API를 통한 데이터셋, 실험, 프롬프트 실험.                     | Playground 통합을 갖춘 데이터셋 및 실험.                     |
| **평가 실행 위치**   | 클라우드 또는 자체 인프라(셀프 호스팅).                                | Datadog의 SaaS 플랫폼.                                       |

## 프롬프트 관리

이 부분은 두 제품 간 가장 큰 기능 격차가 있는 영역입니다. [Langfuse 프롬프트 관리](/docs/prompt-management/overview)는 런타임 시스템으로, 프롬프트가 Langfuse에서 버전 관리되고, 릴리스 레이블을 통해 배포되며, 캐싱되는 SDK 호출을 통해 애플리케이션에서 가져오고, 이를 통해 생성된 트레이스와 평가 점수와 연결됩니다. 프롬프트 조회(fetch)는 무료 티어를 포함한 모든 플랜에서 무제한입니다.

Datadog은 Prompt Tracking 기능을 제공하는데, 이는 코드에 정의된 프롬프트 템플릿과 버전을 LLM 스팬과 연결해 버전 이력, 차이(diff), 트레이스 필터링을 지원합니다. 애플리케이션이 런타임에 Datadog에서 프롬프트를 가져오지는 않으며, 이는 프롬프트 배포 시스템이 아니라 관측성 기능입니다([출처](https://docs.datadoghq.com/llm_observability/monitoring/prompt_tracking/), 2026년 7월 확인).

코드 배포 없이 프롬프트를 업데이트하거나 롤백하고, 프롬프트 버전 간 A/B 테스트를 실행하거나, 프롬프트 반복 작업을 비엔지니어에게 맡기고 싶다면 이는 Langfuse가 지원하는 워크플로이며 Datadog은 지원하지 않습니다.

## 가격 정책

두 플랫폼의 과금 모델은 구조적으로 다르므로 숫자를 비교하기 전에 단위를 꼼꼼히 확인해야 합니다. Langfuse는 [유닛](/docs/administration/billable-units) 단위로 과금하며, 하나의 유닛은 트레이스, 관찰(observation), 또는 점수 하나에 해당합니다. Datadog은 LLM 추론 스팬에만 과금하며, 툴, 워크플로, 에이전트, 임베딩, 검색 스팬은 무료입니다([출처](https://www.datadoghq.com/pricing/list/), 2026년 7월 확인). 따라서 동일한 하나의 에이전트 요청이라도 플랫폼별로 과금되는 수치가 달라질 수 있으므로, 가장 공정한 비교는 자신의 실제 트래픽을 두 플랫폼 모두에 대입해 모델링해보는 것입니다.

| 항목               | Langfuse                                                                  | Datadog LLM Observability                                                       |
| ------------------ | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| **무료 티어**      | 월 5만 유닛, 30일간 데이터 접근 가능. 셀프 호스팅 OSS: 무제한, 무료(MIT). | 월 4만 LLM 스팬, 15일 보관, 전체 기능 제공.                                     |
| **유료 시작 플랜** | **Core: $29/월** (10만 유닛 포함).                                        | **Pro: 연 단위 청구 시 $160/월**(월 단위 청구 시 $200)로 첫 10만 LLM 스팬 포함. |
| **추가 사용량**    | 10만 유닛당 $8, 사용량이 많아지면 10만 유닛당 $6까지 인하.                | 추가 LLM 스팬 1만 개당 $3.50(연 단위 청구 기준).                                |
| **데이터 보관**    | Core: 90일. Pro($199/월): 3년.                                            | 기본 15일 포함; 30/60/90일로 연장 시 스팬 1만 개당 각각 $1.50/$3.00/$4.00 추가. |
| **엔터프라이즈**   | [$2,499/월](/pricing), 감사 로그, SCIM, SLA 포함.                         | 맞춤형, 일반적으로 더 넓은 범위의 Datadog 계약의 일부.                          |

실무적으로 참고할 두 가지 사항이 있습니다. 첫째, Datadog의 무료 및 Pro 티어는 전체 기능에 접근할 수 있어 이미 Datadog 생태계 안에 있는 팀에게는 진입 장벽이 낮습니다. 둘째, Datadog의 기본 15일 보관 기간은 과거 프로덕션 트레이스로부터 평가 데이터셋을 구축하는 등의 LLM 엔지니어링 워크플로에는 짧은 편이며, 보관 기간 연장은 유료 부가 기능인 반면 Langfuse Core는 기본으로 90일을, Pro는 3년을 제공합니다.

## Langfuse와 Datadog을 함께 사용하기 [#coexistence]

Langfuse를 선택한다고 해서 Datadog을 포기해야 하는 경우는 드뭅니다. 저희가 가장 흔하게 보는 프로덕션 구성은 애플리케이션 및 인프라 모니터링에는 Datadog을 그대로 유지하면서, 심층 트레이스 분석, 평가, 프롬프트 관리, 데이터셋 큐레이션 같은 LLM 엔지니어링 작업을 위해 Langfuse를 추가하는 방식입니다. [Langfuse SDK는 OTel 네이티브](/integrations/native/opentelemetry)이므로, 공유된 `TracerProvider`에 두 개의 스팬 프로세서를 연결하는 것만으로 하나의 계측 레이어에서 두 백엔드 모두에 데이터를 공급할 수 있습니다.

```python
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
from langfuse.opentelemetry import LangfuseSpanProcessor

provider = TracerProvider()

# Langfuse는 LLM 및 GenAI 스팬을 받습니다(기본 필터)
provider.add_span_processor(LangfuseSpanProcessor())

# Datadog(또는 다른 APM)은 기존 OTLP 엔드포인트를 통해 모든 스팬을 받습니다
provider.add_span_processor(
    BatchSpanProcessor(OTLPSpanExporter(endpoint="http://localhost:4318/v1/traces"))
)

trace.set_tracer_provider(provider)
```

Langfuse 스팬 프로세서는 기본적으로 LLM 관련 스팬만 필터링하므로, APM은 애플리케이션 전체 그림을 유지하는 동시에 Langfuse는 GenAI 레이어에 집중할 수 있습니다. Datadog도 OpenTelemetry GenAI 스팬을 직접 수용하므로(OTLP를 통한 시맨틱 컨벤션 1.37+), 컬렉터 기반으로 두 백엔드에 팬아웃하는 방식도 가능합니다. 이 패턴에 대한 구성 세부 사항, 프로바이더를 분리하는 설정, 트러블슈팅은 [기존 OpenTelemetry 설정과 함께 Langfuse 사용하기](/faq/all/existing-otel-setup)에서 다룹니다.

## FAQ

### Langfuse는 Datadog LLM Observability의 대안이 될 수 있나요? [#langfuse-datadog-alternative]

네, LLM 애플리케이션 레이어에서는 그렇습니다. Langfuse는 트레이싱, 평가, 프롬프트 관리, 데이터셋/실험을 지원하며, 여기에 Datadog이 제공하지 않는 셀프 호스팅과 MIT 오픈소스 라이선스까지 추가로 제공합니다. Langfuse는 APM이 아니므로 Datadog의 인프라 및 애플리케이션 모니터링을 대체하기보다는 보완하는 역할을 합니다.

### Datadog LLM Observability를 셀프 호스팅할 수 있나요? [#self-host-datadog]

아니요. Datadog은 9개 지역 사이트에서 호스팅되는 SaaS로 운영되며, 텔레메트리 데이터는 Datadog의 인프라에 저장됩니다(2026년 7월 확인). Langfuse는 오픈소스(MIT)이며 Docker Compose, Kubernetes, Terraform 템플릿을 통해 에어갭 배포를 포함해 [셀프 호스팅](/self-hosting)할 수 있습니다.

### Datadog LLM Observability의 가격은 얼마인가요? [#datadog-llm-observability-pricing]

2026년 7월 기준, Datadog은 월 4만 LLM 스팬과 15일 보관을 제공하는 무료 티어와, 첫 10만 LLM 스팬을 포함하는 월 $160(연 단위 청구 기준) Pro 플랜을 제공하며, 추가 스팬은 1만 개당 $3.50입니다. LLM 추론 스팬만 과금되며 툴, 워크플로, 검색 스팬은 무료입니다. 30, 60, 90일로 보관 기간을 연장하는 것은 유료 부가 기능입니다.

### Langfuse와 Datadog을 동시에 사용할 수 있나요? [#langfuse-and-datadog-together]

네, 흔한 구성입니다. 두 제품 모두 OpenTelemetry 트레이스를 수신하므로, 하나의 계측 레이어에서 애플리케이션 내 두 개의 스팬 프로세서 또는 컬렉터 기반 팬아웃을 통해 APM용 Datadog과 LLM 엔지니어링용 Langfuse 모두에 데이터를 공급할 수 있습니다. 실제 동작하는 구성 예시는 [기존 OpenTelemetry 설정과 함께 Langfuse 사용하기](/faq/all/existing-otel-setup)를 참고하세요.
