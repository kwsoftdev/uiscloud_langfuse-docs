# Linear 사용하기

우리는 모든 내부 티켓팅 및 작업 관리를 위해 Linear를 사용합니다.

Linear는 소규모 팀으로서 대규모 커뮤니티 고객 및 OSS 사용자와 함께 작업하면서도 빠르게 움직일 수 있게 해주는 [제품 운영](/handbook/product-engineering/how-we-work/product-ops)의 핵심입니다.

## 원칙

- 모든 구성원은 자신의 이슈와 프로젝트를 직접 관리합니다.
  - 우리 팀에는 백로그를 중앙에서 관리하는 담당자가 없습니다 → 담당자가 배정되지 않은 이슈가 있어서는 안 됩니다.
- 향후 6개월 이내에 발생할 가능성이 있는 작업에 대해 Linear 이슈를 생성합니다. 그 이후의 작업은 [로드맵](/docs/roadmap)과 [GitHub ideas](/ideas)에서 추적할 수 있습니다.

## Linear 사용 방법

- SLA: 하루에 여러 번 Linear Inbox를 정리합니다. 이곳은 주요 주제별 논의 채널입니다.
- Git:
  - 변경 작업을 할 때는 Linear 이슈에서 브랜치 이름을 복사하면, PR이 Linear 이슈에 자동으로 연결됩니다.
  - PR이 병합되면 이슈는 `merged` 상태로 이동합니다. 변경 사항이 프로덕션에 배포되고 잠재적인 후속 작업이 모두 추적/완료되면 `done` 상태로 이동합니다.

## Linear 설정

### 필수

- Slack 및 GitHub에 연결합니다.
- Engineering, Langfuse, GTM 팀에 참여합니다.

### 선택

- Cursor Background Agents에 연결하여 AI에게 작업을 위임합니다.
