# Slack 사용하기

우리는 내부 커뮤니케이션과 고객, 파트너, 벤더와의 대화를 위해 Slack을 사용합니다. 팀에 합류하면 Google 계정을 통해 Slack(`langfuse.slack.com`)에 가입할 수 있습니다.

우리는 노이즈를 줄이고 모두가 자신의 업무에 집중할 수 있도록 Slack을 대부분 비동기적으로 사용합니다. 긴급한 사항이 있다면 DM을 보내거나, 통화를 시작하거나, 사무실에 함께 있다면 어깨를 두드려 주세요. 더 넓은 범위의 비동기 규범에 대해서는 [Working Remotely](/handbook/how-we-work/remote)를 참고하세요.

## 권장 설정

- 멘션, 다이렉트 메시지, 그리고 아래에 언급된 sync/notifying 채널에 대해서만 알림을 활성화하세요.
- 동료에게 알림이 가지 않는 메시지를 보내기 위해 `queue-<name>-x-<name>` 비공개 채널을 설정하세요. SLA는 24시간 이내 응답입니다.
- 읽지 않은 메시지의 백로그를 빠르게 처리하기 위해 메인 메뉴에서 `All unreads`를 설정하세요.

## 가장 중요한 채널

| 채널                                                                                                         | 목적                                                                                                                                       |
| ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `team`                                                                                                       | 일반 팀 채팅 (알림 있음)                                                                                                                   |
| `team-async`                                                                                                 | 일반 팀 채팅 (알림 없음, 대부분의 메시지에 적합)                                                                                           |
| `engineering`, `product`, `user-feedback`, `market-insights`, `media-recommendations`, `marketing`, `growth` | 주제별 채널 (알림 없음)                                                                                                                    |
| `incidents`                                                                                                  | 인시던트 커뮤니케이션 (알림 있음)                                                                                                          |
| `memehq`                                                                                                     | 아마도 가장 재미있는 채널                                                                                                                  |
| `ai-trickshots`                                                                                              | AI로 만든 결과물을 자랑하는 채널 (알림 없음)                                                                                               |
| `queue-<name>-x-<name>`                                                                                      | 팀원 간 알림 없는 메시지를 위한 비공개 채널                                                                                                |
| `ext-*`                                                                                                      | 고객 커뮤니케이션. 세일즈/GTM은 대화를 진전시키기 위해 일부 알림을 사용합니다. 나머지 팀원은 Pylon을 통해 관리되므로 음소거할 수 있습니다. |
| `vendor-*`                                                                                                   | 벤더 커뮤니케이션. 자신과 관련된 채널에 참여하세요.                                                                                        |
| `partner-*`                                                                                                  | 파트너 커뮤니케이션. 자신과 관련된 채널에 참여하세요.                                                                                      |

## 외부 채널

일부 우선순위가 높은 고객 그룹에 대해서는 지원 및 세일즈를 위해 공유 Slack Connect 채널을 사용합니다.

### Pylon과 동기화

다음 접두사를 채널에 사용합니다.

- 고객: `ext-` (Pylon에서 동기화됨, Pylon 앱이 자동으로 참여)
- 파트너: `partner-` (Pylon에서 동기화되지 **않음**)
- 벤더: `vendor-` (Pylon에서 동기화되지 **않음**)

`ext-*` 채널에서의 모든 지원/세일즈 커뮤니케이션은 Pylon을 통해 이루어집니다.

### 새 외부 Slack Connect 채널 만들기

- 적절한 채널 접두사를 사용하세요(예: `ext-`, `partner-`, `vendor-`).
- 공유 채널은 고객의 Slack 조직 내에서 검색이 가능하도록 공개 채널이어야 합니다.
- `ext-*` 채널(고객)을 만들 때:
  - 외부 멤버를 초대하세요.
  - 내부 멤버는 명시적으로 초대하고, 모두를 위한 기본 채널로 만들지 마세요. 모든 외부 채널의 멤버가 되기를 원하는 Clemens와 Akio만 초대하세요. 다른 모든 사람은 Pylon 통합을 통해 이 채널에서 도움을 줄 수 있으며, 앱이 자동으로 모든 `ext-*` 채널에 참여합니다.
  - **비공개 채널:** Pylon 앱은 비공개 채널에 자동으로 참여할 수 없습니다. 다음을 수동으로 수행해야 합니다.
    1. Pylon 앱을 채널에 추가하세요(Slack에서 채널을 열고 → 채널 이름 클릭 → _Integrations_ → _Add apps_ → Pylon 검색).
    2. Slack 채널을 Pylon에 계정으로 추가하세요: [app.usepylon.com/customers/accounts/add](https://app.usepylon.com/customers/accounts/add).
  - 선택 사항: 외부 멤버가 참여한 것을 확인하면 채널에 환영 메시지를 공유한 다음 Slack에서 채널을 음소거하세요. 이후부터는 Pylon 통합을 통해 이 채널에서 도움을 줄 수 있습니다.

### 외부 채널을 위한 Slack 설정

- 대부분의 채널과 대부분의 팀원의 경우: 기존 `ext-*` 채널을 음소거하거나 나갈 수 있습니다. Pylon을 통해 스레드에 배정됩니다.
- 고가치 계정의 경우, 관계를 관리하는 담당자는 채널에 남아 있어야 합니다. 이렇게 하면 다이렉트 채널을 활용하여 가장 중요한 고객과 가깝게 지내며 아웃바운드로 소통하기가 더 쉬워집니다.
