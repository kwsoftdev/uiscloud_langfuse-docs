# Pylon 사용하기

이 가이드는 Langfuse에서 지원 요청을 처리하는 프로세스를 설명합니다. 이를 위해 [Pylon](https://app.usepylon.com/) ([문서](https://docs.usepylon.com))을 사용합니다:

- [지원 채널](/support)을 하나의 받은편지함으로 통합
- 에스컬레이션 단계 관리 및 SLA 추적
- 고객 요청 및 보고된 버그를 Linear 이슈에 연결

모든 구성원은 온보딩 과정에서 Pylon에 대한 접근 권한을 받습니다.

## Pylon의 프로세스는 무엇인가요?

1. **통합 받은편지함**: Pylon은 [지원 채널](/support)인 **Slack** Connect 채널, **MS Teams** 채널, **이메일** 지원 주소, **GitHub Discussions**(Support 및 Ideas), **인앱 지원 양식**과 동기화되어 있습니다. 사용자가 하나의 요청에 대해 여러 메시지를 보낸 경우, Pylon은 AI를 사용하여 해당 메시지들을 하나의 지원 티켓으로 병합합니다.
2. **1차 지원**: 받은편지함을 정기적으로 확인하고 질문에 답변합니다
   - 에스컬레이션이 필요한 경우 2차 [담당자(owner)](/handbook/how-we-work/ownership)를 지정합니다
   - 일반적으로 버그 리포트와 기능 요청은 담당자에게 에스컬레이션합니다. 담당자는 이를 Linear 백로그에 연결하는 작업을 처리합니다([product ops](/handbook/product-engineering/how-we-work/product-ops) 참고).
3. **담당자(엔지니어, GTM)**: 이슈를 수정하거나 기록하고, 피드백을 메모하며, 기술적인 조언을 제공합니다.
4. **루프 마무리**: 즉시 해결하거나 이슈를 기록한 뒤, 수정이 완료되면 사용자에게 알리고 문서를 업데이트합니다

## Pylon을 어떻게 사용하나요?

### 핵심 애플리케이션 화면

<Frame>
  ![Pylon Screenshot](/images/handbook/using-pylon/pylon-screenshot.png)
</Frame>

1. 이슈의 상태(Status).
2. 이슈의 담당자(Assignee). 담당자로 지정되면 해당 이슈는 본인 소관입니다.
3. 고객 등급(Customer Tier): 유료 고객인지 확인합니다.
4. 내부 메모를 남기고 사람을 태그합니다.
5. Pylon을 사용해 Slack/MS Teams/이메일/채팅 위젯에서 직접 답장합니다.

### 개인 워크플로

- 1차 지원 당번이 아닌 경우:
  - `My issues (only assigned)` 뷰에 집중하면 됩니다.
  - 하루에 최소 한 번은 이 목록을 모두 처리하는 것이 원칙입니다.

### 권장 알림 설정

- 본인에게 할당된 스레드와 관련된 모든 이벤트를 구독하세요
- 우리는 일반적으로 Slack을 비동기적으로 사용합니다. 팀 대부분의 구성원에게는 고객 커뮤니케이션을 신속히 처리할 수 있도록 Slack에서 Pylon 알림을 기본으로 활성화할 것을 권장합니다.
- 휴가(PTO) 중이라면 `Away Mode`를 켜서 (1) 본인이 새로 응답을 남길 때 스레드 할당을 해제하고 (2) 티켓을 본인에게 자동 할당하는 자동화를 비활성화하시기 바랍니다
