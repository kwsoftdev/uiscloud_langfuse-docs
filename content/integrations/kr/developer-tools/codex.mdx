# OpenAI Codex tracing with Langfuse

> **Codex란 무엇인가요?** [Codex](https://github.com/openai/codex)는 OpenAI의 에이전틱 코딩 도구입니다. 터미널(및 IDE)에서 실행되며, 여러분의 코드베이스를 이해하고, 코드를 편집하고, 명령을 실행하며, 서브에이전트를 생성해 작업을 병렬로 처리할 수 있습니다.

> **Langfuse란 무엇인가요?** [Langfuse](https://langfuse.com/)는 오픈소스 AI 엔지니어링 플랫폼입니다. 팀이 에이전틱 애플리케이션을 추적하고, 문제를 디버깅하고, 품질을 평가하고, 프로덕션에서 비용을 모니터링할 수 있도록 돕습니다.

<Callout type="info" title="Install via the Codex plugin marketplace">
이를 설정하는 가장 쉬운 방법은 [Langfuse Codex Plugin](https://github.com/langfuse/codex-observability-plugin)입니다. 마켓플레이스를 추가하고 플러그인을 활성화하세요:

```bash
codex plugin marketplace add langfuse/codex-observability-plugin
```

그런 다음 `~/.codex/config.toml`에서 플러그인 훅과 트레이싱 플러그인을 활성화하고 Langfuse 키를 설정하세요(전체 단계는 아래에 있습니다). Node.js 22+와 Codex 0.128+가 필요합니다.

</Callout>

## 이 인테그레이션으로 무엇을 추적할 수 있나요?

Codex의 플러그인 훅을 사용해, 이 인테그레이션은 Codex가 각 세션에 대해 작성하는 트랜스크립트를 읽어 Langfuse로 전송합니다. 다음을 모니터링할 수 있습니다:

- **사용자 입력**: Codex로 보내는 모든 프롬프트
- **모델 응답**: 턴 내 각 모델 호출에 대한 어시스턴트 메시지와 추론 요약
- **도구 호출**: 셸 명령(`exec_command`), 파일 편집(`apply_patch`), MCP 도구, 웹 검색, 서브에이전트 호출 — 각각의 입력, 출력, 오류 상태와 함께
- **토큰 사용량**: 모델 호출당 입력, 출력, 캐시, 추론 토큰으로 비용을 모니터링할 수 있습니다
- **서브에이전트**: 서브에이전트 스레드가 각자의 트랜스크립트에서 해석되어 이를 생성한 턴 아래 중첩됩니다
- **세션**: 하나의 Codex 세션에서 나온 모든 턴이 리플레이를 위해 함께 그룹화됩니다
- **타이밍**: 각 단계의 정확한, 소급 적용된 시작 및 종료 시간

## 작동 방식

Codex는 라이프사이클 시점에 커스텀 명령을 실행하는 [훅](https://github.com/openai/codex)이 포함된 플러그인 시스템을 제공합니다. 이 인테그레이션은 Codex가 각 턴을 마칠 때마다 실행되는 **Stop 훅**을 사용합니다.

1. 플러그인은 Codex가 턴을 마칠 때마다 실행되는 `Stop` 훅을 등록합니다.
2. 훅이 Codex의 세션 트랜스크립트(롤아웃 파일)를 읽습니다.
3. 턴이 재구성되어 [Langfuse TypeScript SDK](/docs/observability/sdk/overview)를 사용해 Langfuse 트레이스로 변환됩니다.
4. 같은 세션의 모든 턴은 공유된 [`session_id`](/docs/observability/features/sessions)를 사용해 그룹화됩니다.
5. 작은 사이드카 파일이 이미 업로드된 턴을 기록해, 세션을 재개해도 중복이 생기지 않습니다.

트레이싱은 `TRACE_TO_LANGFUSE` 환경 변수를 통해 **옵트인**됩니다. 훅은 실패해도 열린 상태를 유지합니다: 문제가 생기면 로그를 남기고 Codex 세션을 막지 않은 채 종료됩니다.

## 빠른 시작

<Steps>

### Langfuse 설정하기

1. [Langfuse Cloud](https://cloud.langfuse.com)에 가입하거나 [Langfuse를 직접 호스팅](/self-hosting)하세요.
2. 새 프로젝트를 생성하고 프로젝트 설정에서 API 키를 복사하세요.

### 플러그인 마켓플레이스 추가하기

Codex CLI를 통해 Langfuse 마켓플레이스를 추가하세요:

```bash
codex plugin marketplace add langfuse/codex-observability-plugin
```

### 플러그인 활성화하기

`~/.codex/config.toml`에서 전역으로, 또는 `<project>/.codex/config.toml`에서 특정 프로젝트에 한해서만 플러그인 훅과 트레이싱 플러그인을 활성화하세요:

```toml
[features]
plugin_hooks = true

[plugins."tracing@codex-observability-plugin"]
enabled = true
```

### Langfuse 자격 증명 설정하기 [#set-credentials]

`TRACE_TO_LANGFUSE`가 `"true"`가 되기 전까지 트레이싱은 꺼져 있으므로, 명시적으로 옵트인하게 됩니다. 셸 프로필(`~/.zshrc`, `~/.bashrc`, 또는 `~/.bash_profile`)에 자격 증명을 추가하세요:

```bash
export TRACE_TO_LANGFUSE="true"
export LANGFUSE_PUBLIC_KEY="pk-lf-..."
export LANGFUSE_SECRET_KEY="sk-lf-..."
export LANGFUSE_BASE_URL="https://cloud.langfuse.com" # 🇪🇺 EU region
```

또는 `~/.codex/langfuse.json`(전역) 또는 `<project>/.codex/langfuse.json`(프로젝트별)에 JSON 설정 파일을 생성하세요:

```json
{
  "enabled": true,
  "public_key": "pk-lf-...",
  "secret_key": "sk-lf-...",
  "base_url": "https://cloud.langfuse.com"
}
```

설정은 기본값 → 전역 설정 파일 → 프로젝트 설정 파일 → 환경 변수 순으로 해석되며, 환경 변수가 우선합니다. `LANGFUSE_CODEX_*` 변수는 일치하는 표준 `LANGFUSE_*` 변수보다 우선하므로, Codex에 한정해 자격 증명 범위를 지정할 수 있습니다.

### Codex 사용하기

평소처럼 Codex를 실행하세요. 각 턴 이후 트레이스가 Langfuse로 전송됩니다:

```bash
cd your-project
codex
```

### Langfuse에서 트레이스 확인하기

Langfuse 프로젝트를 열어 캡처된 트레이스를 확인하세요. 구조는 Codex가 실제로 동작하는 방식을 그대로 반영합니다:

- **턴 트레이스** (`Codex Turn`): 여러분의 프롬프트부터 최종 답변까지, 턴당 하나의 트레이스로 [에이전트 관찰](/docs/observability/features/observation-types)로 캡처됩니다.
- **생성**: 턴 내 모델 응답마다 하나씩입니다. 각각 받은 입력, 모델의 추론과 텍스트, 요청한 도구 호출, 토큰 사용량을 보여줍니다.
- **도구 스팬** (`exec_command`, `apply_patch`, `spawn_agent`, …): 트리거한 생성 아래 중첩되며, 입력, 출력, 오류 상태를 포함합니다. 실패한 명령은 오류로 표시됩니다.
- **서브에이전트**: 서브에이전트 스레드는 이를 생성한 턴 아래 중첩되어 병렬 작업을 한곳에서 따라갈 수 있습니다.
- **세션**: 같은 Codex 세션의 모든 턴은 [`session_id`](/docs/observability/features/sessions)를 통해 그룹화됩니다 — Sessions 탭을 열어 전체 실행을 리플레이하세요.

</Steps>

## 환경 변수

| 변수                           | 설명                                                                                                                                                                      | 필수 여부            |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------- |
| `TRACE_TO_LANGFUSE`            | 트레이싱을 활성화하려면 `"true"`로 설정하세요                                                                                                                             | 예                   |
| `LANGFUSE_PUBLIC_KEY`          | Langfuse 공개 키(`pk-lf-...`)                                                                                                                                             | 예                   |
| `LANGFUSE_SECRET_KEY`          | Langfuse 비밀 키(`sk-lf-...`)                                                                                                                                             | 예                   |
| `LANGFUSE_BASE_URL`            | Langfuse 호스트. EU: `https://cloud.langfuse.com`, US: `https://us.cloud.langfuse.com`, Japan: `https://jp.cloud.langfuse.com`, HIPAA: `https://hipaa.cloud.langfuse.com` | 아니요 (기본값은 EU) |
| `LANGFUSE_TRACING_ENVIRONMENT` | 트레이스에 사용할 환경 라벨(예: `production`)                                                                                                                             | 아니요               |
| `LANGFUSE_CODEX_TAGS`          | 모든 트레이스에 사용할 태그(JSON 배열 또는 쉼표로 구분)                                                                                                                   | 아니요               |
| `LANGFUSE_CODEX_METADATA`      | 모든 트레이스에 첨부할 메타데이터의 JSON 객체                                                                                                                             | 아니요               |
| `LANGFUSE_CODEX_MAX_CHARS`     | 이 문자 수를 넘는 입력/출력을 잘라냅니다(기본값 `20000`)                                                                                                                  | 아니요               |
| `LANGFUSE_CODEX_DEBUG`         | stderr로 자세한 로깅을 원하면 `"true"`로 설정하세요                                                                                                                       | 아니요               |

자격 증명 변수는 `LANGFUSE_CODEX_` 접두사도 지원합니다(예: `LANGFUSE_CODEX_PUBLIC_KEY`). 이 접두사가 붙은 변수가 표준 변수보다 우선합니다.

## 문제 해결

### Langfuse에 트레이스가 나타나지 않음

1. **플러그인이 활성화되지 않았습니다.** `~/.codex/config.toml`에서 `plugin_hooks = true`이고 `tracing@codex-observability-plugin` 플러그인이 활성화되어 있는지 확인하세요.
2. **트레이싱이 켜져 있지 않습니다.** `TRACE_TO_LANGFUSE`는 정확히 문자열 `"true"`여야 하며 Codex 프로세스에서 보여야 합니다. 공개 키가 `pk-lf-`로 시작하는지도 확인하세요.
3. **디버그 로깅을 활성화하세요.** `LANGFUSE_CODEX_DEBUG=true`로 설정하면 stderr에 로그가 남아 실제 원인을 파악할 수 있습니다.

### 인증 오류

API 키가 올바른지, `LANGFUSE_BASE_URL`이 여러분의 키가 속한 리전과 일치하는지 확인하세요:

- **EU 리전**: `https://cloud.langfuse.com`
- **US 리전**: `https://us.cloud.langfuse.com`
- **Japan 리전**: `https://jp.cloud.langfuse.com`
- **HIPAA 리전**: `https://hipaa.cloud.langfuse.com`

### 데이터 프라이버시

활성화되면, 플러그인은 완료된 Codex 트랜스크립트 데이터를 Langfuse로 업로드합니다. 여기에는 프롬프트, 어시스턴트 메시지, 추론 요약, 도구 입력 및 출력, 모델 메타데이터, 토큰 사용량이 포함됩니다. Langfuse에 저장하고 싶지 않은 데이터가 포함된 세션에서는 트레이싱을 활성화하지 마세요. 큰 입력과 출력이 얼마나 캡처되는지 제한하려면 `LANGFUSE_CODEX_MAX_CHARS`를 사용하세요.
