# OpenRouter 통합

이 가이드에서는 [Langfuse](/)를 [OpenRouter](https://openrouter.ai/)와 통합하는 방법을 보여드립니다.

> **OpenRouter란?** [OpenRouter](https://openrouter.ai/)는 280개 이상의 언어 모델과 프로바이더에 대해 OpenAI 호환 완성(completion) API를 제공하며, 직접 호출하거나 OpenAI SDK를 사용해 호출할 수 있습니다. 이를 통해 개발자는 하나의 통합된 인터페이스로 다양한 LLM에 접근할 수 있습니다.

> **Langfuse란?** [Langfuse](/)는 팀이 LLM 호출을 트레이싱하고, 성능을 모니터링하며, AI 애플리케이션의 문제를 디버그하도록 돕는 오픈소스 AI 엔지니어링 플랫폼입니다.

Langfuse를 OpenRouter와 통합하는 방법은 두 가지입니다:

1. [**OpenRouter의 Broadcast 기능을 사용**](#broadcast)하여 코드 변경 없이 자동으로 Langfuse에 트레이스를 전송할 수 있습니다.
2. [**Langfuse OpenAI SDK 래퍼를 사용**](#sdk)하여 Langfuse에 수동으로 트레이스를 전송할 수 있습니다. OpenRouter는 OpenAI API 스키마를 사용하므로, [Python](/integrations/model-providers/openai-py)과 [TypeScript](/integrations/model-providers/openai-js) 모두에서 제공되는 Langfuse의 OpenAI SDK 네이티브 통합을 활용할 수 있습니다.

비용 추적이나 Langfuse Python SDK와의 상호운용성, 다음 단계 등에 대한 더 자세한 정보는 아래 가이드에 있습니다.

## 1. OpenRouter Broadcast를 통한 Langfuse 트레이싱 (노코드) [#broadcast]

OpenRouter의 [Broadcast 기능](https://openrouter.ai/docs/guides/features/broadcast/overview)은 코드 변경 없이 자동으로 Langfuse에 트레이스를 전송할 수 있습니다. 이 통합을 활성화하려면 [OpenRouter 설정](https://openrouter.ai/settings)으로 이동해 Langfuse API 키를 연결하세요. 설정이 완료되면 OpenRouter를 통해 처리되는 모든 요청이 트레이싱되어 Langfuse 프로젝트에서 확인할 수 있습니다.

> **참고:** 이 방법은 커스텀 트레이스 메타데이터나 중첩 트레이싱 같은 고급 기능이 필요 없이, 최소한의 설정으로 모든 OpenRouter 요청을 Langfuse에 캡처하고 싶을 때 권장됩니다.

Langfuse API 키 설정이나 사용 가능한 옵션에 대한 자세한 내용은 [Langfuse 시작 가이드](/docs/observability/get-started)를 참고하세요.

## 2. SDK 통합 [#sdk]

중첩 트레이싱, 커스텀 메타데이터, 트레이스 데이터에 대한 완전한 제어 같은 고급 기능을 사용하려면 Langfuse OpenAI SDK 래퍼를 사용하세요. 이 가이드의 나머지 부분에서는 SDK 통합 방법을 다룹니다.

### SDK 통합으로 시작하기

```bash
pip install langfuse openai
```

```python
import os

# Set your OpenRouter API key (OpenRouter uses the 'OPENAI_API_KEY' environment variable)
os.environ["OPENAI_API_KEY"] = "<YOUR_OPENROUTER_API_KEY>"
```

### 예제 1: 단순 LLM 호출

OpenRouter는 OpenAI 호환 API를 제공하므로, [Langfuse OpenAI SDK 래퍼](/integrations/model-providers/openai-py)를 사용해 OpenRouter 호출을 Langfuse의 생성(generation)으로 자동 기록할 수 있습니다.

- `base_url`은 OpenRouter의 API 엔드포인트로 설정됩니다.
- `"qwen/qwen-plus"`를 OpenRouter에서 제공하는 원하는 모델로 바꿀 수 있습니다.
- `default_headers`에는 OpenRouter 문서에 따른 선택적 헤더를 포함할 수 있습니다.
- `extra_body={"usage": {"include": True}}`는 OpenRouter가 반환하는 비용을 포함합니다.

```python
# Import the Langfuse OpenAI SDK wrapper
from langfuse.openai import openai

# Create an OpenAI client with OpenRouter's base URL
client = openai.OpenAI(
    base_url="https://openrouter.ai/api/v1",
    default_headers={
        "HTTP-Referer": "<YOUR_SITE_URL>",  # Optional: Your site URL
        "X-Title": "<YOUR_SITE_NAME>",      # Optional: Your site name
    }
)

# Make a chat completion request
response = client.chat.completions.create(
    model="qwen/qwen-plus",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Tell me a fun fact about space."}
    ],
    extra_body={"usage": {"include": True}},
    name="fun-fact-request"  # Optional: Name of the generation in Langfuse
)

# Print the assistant's reply
print(response.choices[0].message.content)
```

### 예제 2: 중첩된 LLM 호출

`@observe()` 데코레이터를 사용하면, 중첩된 LLM 호출, 입력, 출력, 실행 시간을 포함해 모든 Python 함수의 실행 세부 정보를 캡처할 수 있습니다. 이를 통해 최소한의 코드 변경으로 심층적인 관측성을 확보할 수 있습니다.

- `@observe()` 데코레이터는 함수의 입력, 출력, 실행 세부 정보를 캡처합니다.
- 중첩된 함수 `summarize_text`와 `analyze_sentiment`에도 데코레이터가 적용되어 트레이스 계층 구조가 만들어집니다.
- 함수 내의 각 LLM 호출이 기록되어, 실행 흐름의 상세한 트레이스를 제공합니다.

```python
from langfuse import observe
from langfuse.openai import openai

# Create an OpenAI client with OpenRouter's base URL
client = openai.OpenAI(
    base_url="https://openrouter.ai/api/v1",
)

@observe()  # This decorator enables tracing of the function
def analyze_text(text: str):
    # First LLM call: Summarize the text
    summary_response = summarize_text(text)
    summary = summary_response.choices[0].message.content

    # Second LLM call: Analyze the sentiment of the summary
    sentiment_response = analyze_sentiment(summary)
    sentiment = sentiment_response.choices[0].message.content

    return {
        "summary": summary,
        "sentiment": sentiment
    }

@observe()  # Nested function to be traced
def summarize_text(text: str):
    return client.chat.completions.create(
        model="openai/gpt-3.5-turbo",
        messages=[
            {"role": "system", "content": "You summarize texts in a concise manner."},
            {"role": "user", "content": f"Summarize the following text:\n{text}"}
        ],
        extra_body={"usage": {"include": True}},
        name="summarize-text"
    )

@observe()  # Nested function to be traced
def analyze_sentiment(summary: str):
    return client.chat.completions.create(
        model="openai/gpt-3.5-turbo",
        messages=[
            {"role": "system", "content": "You analyze the sentiment of texts."},
            {"role": "user", "content": f"Analyze the sentiment of the following summary:\n{summary}"}
        ],
        extra_body={"usage": {"include": True}},
        name="analyze-sentiment"
    )

# Example usage
text_to_analyze = "OpenAI's GPT-4 model has significantly advanced the field of AI, setting new standards for language generation."
analyze_text(text_to_analyze)
```

### 비용 추적

Langfuse [토큰 및 비용 추적](/docs/observability/features/token-and-cost-tracking)은 비용을 직접 계산하는 대신 OpenRouter의 비용을 직접 캡처할 수 있습니다. 이는 특히 덜 대중적인 모델에서 비용 추적의 정확도를 높여줍니다.

이를 활성화하려면 다음이 필요합니다:

1. 위에서 설명한 대로 Langfuse OpenAI 통합을 사용합니다
2. OpenRouter에서 [Usage Accounting](https://openrouter.ai/docs/use-cases/usage-accounting)을 활성화합니다. 이렇게 하면 각 호출에 대해 OpenRouter가 비용을 반환합니다
