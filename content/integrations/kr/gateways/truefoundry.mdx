# TrueFoundry AI 게이트웨이 통합

> **Truefoundry란?** [TrueFoundry](https://www.truefoundry.com/ai-gateway)는 하나의 OpenAI 호환 API 뒤에서 모든 LLM 또는 Gen-AI 워크로드를 배포, 거버넌스, 모니터링할 수 있는 엔터프라이즈급 AI Gateway이자 제어 플레인으로, 속도 제한, 비용 통제, 관측성, 온프레미스 지원을 프로덕션 AI 애플리케이션에 제공합니다.

## TrueFoundry가 Langfuse와 통합되는 방식

Truefoundry의 AI Gateway와 Langfuse가 결합되어 몇 분 만에 설정할 수 있는 엔터프라이즈급 관측성, 거버넌스, 모든 LLM 요청에 대한 비용 통제를 제공합니다.

<details>
<summary><strong>통합 OpenAI 호환 엔드포인트</strong></summary>

Langfuse OpenAI 클라이언트를 Truefoundry의 게이트웨이 URL로 지정하세요. Truefoundry는 지원되는 모든 모델(OpenAI, Anthropic, 셀프 호스팅 모델 등)로 라우팅하며, Langfuse는 코드 변경 없이 각 호출을 투명하게 캡처합니다.

</details>

<details>
<summary><strong>엔드투엔드 트레이싱 및 메트릭</strong></summary>

Langfuse는 다음을 제공합니다:

- 시스템 메시지를 포함한 **전체 요청/응답 로그**\
- **토큰 사용량**(프롬프트, 완료, 총합)\
- 호출당 **지연 시간 분석**\
- 모델과 환경별 **비용 분석**\

몇 초 만에 트레이스를 파고들어 성능을 최적화하거나 회귀 문제를 디버그하세요.

</details>

<details>
<summary><strong>프로덕션 준비 완료된 제어 기능</strong></summary>

Truefoundry는 LLM 스택에 다음 기능을 추가합니다:

- 팀 또는 사용자별 **속도 제한 및 쿼터**\
- 초과 지출을 방지하는 **예산 경고 및 지출 상한**\
- 개발/스테이징/프로덕션을 위한 RBAC가 적용된 **범위 지정 API 키**\
- 완전한 데이터 주권을 위한 **온프레미스/VPC 배포**\

</details>

## 3단계: Langfuse OpenAI 드롭인 대체 사용하기

Langfuse의 OpenAI 호환 클라이언트를 사용해 TrueFoundry AI Gateway를 통해 라우팅되는 모든 요청을 캡처하고 트레이싱하세요. 게이트웨이 설정과 가상 LLM 키 생성에 대한 자세한 단계는 [TrueFoundry 문서](https://docs.truefoundry.com/gateway/langfuse)에서 확인할 수 있습니다.

```python
from langfuse.openai import OpenAI
import os

# Initialize OpenAI client with TrueFoundry Gateway
client = OpenAI(
    api_key=os.environ["TRUEFOUNDRY_API_KEY"],
    base_url=os.environ["TRUEFOUNDRY_BASE_URL"]  # Base URL from unified code snippet
)
```

## 4단계: 예제 실행하기

```python
# Make a request through TrueFoundry Gateway with Langfuse tracing
response = client.chat.completions.create(
    model="openai-main/gpt-4o",  # Paste the model ID you copied from TrueFoundry Gateway
    messages=[
        {"role": "system", "content": "You are a helpful AI assistant specialized in explaining AI concepts."},
        {"role": "user", "content": "Why does an AI gateway help enterprises?"},
    ],
    max_tokens=500,
    temperature=0.7
)

print(response.choices[0].message.content)

# Ensure all traces are sent to Langfuse
langfuse = get_client()
langfuse.flush()
```

## Langfuse Python SDK를 활용한 고급 통합

자동 트레이싱을 추가 Langfuse 기능과 결합해 관측성을 강화하세요.

### @observe 데코레이터 사용하기

`@observe()` 데코레이터는 함수를 자동으로 감싸고 트레이스에 커스텀 속성을 추가합니다:

```python
from langfuse import observe, get_client, propagate_attributes

langfuse = get_client()

@observe()
def analyze_customer_query(query, customer_id):
    """Analyze customer query using TrueFoundry Gateway with full observability"""

    # Propagate attributes to all child observations
    with propagate_attributes(
        user_id=customer_id,
        session_id=f"session_{customer_id}",
        tags=["customer-service", "truefoundry-gateway"],
        metadata={"gateway": "truefoundry"},
        version="1.0.0"
    ):

        response = client.chat.completions.create(
            model="openai-main/gpt-4o",
            messages=[
                {"role": "system", "content": "You are a customer service AI assistant."},
                {"role": "user", "content": query},
            ],
            temperature=0.3
        )

        result = response.choices[0].message.content

    # Set input and output on the current observation
    langfuse.update_current_observation(
        input={"query": query, "customer_id": customer_id},
        output={"response": result},
    )

    return result

# Usage
result = analyze_customer_query("How do I reset my password?", "customer_123")
```
