# LiteLLM Proxy 통합

Langfuse는 LiteLLM에 대한 관측성을 제공합니다. LiteLLM Proxy를 통해 라우팅되는 모든 LLM 호출은 요청마다 토큰 사용량, 비용, 지연 시간과 함께 Langfuse에 기록될 수 있습니다. 이 가이드에서는 설정 방법을 보여드립니다.

> **LiteLLM이란?** [LiteLLM](https://github.com/BerriAI/litellm)은 OpenAI 호환 엔드포인트로 수백 개의 다양한 LLM 프로바이더와 모델을 호출하고 관리할 수 있는 단일 통합 API를 제공하는 오픈소스 프록시이자 SDK입니다.

> **Langfuse란?** [Langfuse](https://langfuse.com)는 LLM 애플리케이션을 트레이싱, 모니터링, 디버그하도록 돕는 오픈소스 LLM 관측성 플랫폼입니다.

---

LiteLLM을 Langfuse와 통합하는 방법은 세 가지입니다:

1. LiteLLM Proxy를 통해 로그를 전송하여 프록시를 거치는 모든 LLM 호출을 캡처합니다.
2. LiteLLM SDK를 사용해 LLM 호출을 직접 캡처합니다.
3. 호환되는 프레임워크(OpenAI나 LangChain SDK 등)를 사용해 LLM 호출을 캡처합니다.

<Callout type="info">
  이 통합은 LiteLLM Proxy를 위한 것입니다. LiteLLM SDK 통합을 찾으신다면
  [LiteLLM SDK 통합](/integrations/frameworks/litellm-sdk) 페이지를
  참고하세요.
</Callout>
---

## LiteLLM Proxy

<Tabs items={["Local", "LiteLLM UI"]}>
<Tab>

프록시 설정에 통합을 추가하세요:

**1. 환경 변수에 자격 증명 추가하기**

```bash
export LANGFUSE_PUBLIC_KEY="pk-lf-..."
export LANGFUSE_SECRET_KEY="sk-lf-..."
export LANGFUSE_OTEL_HOST="https://us.cloud.langfuse.com"  # Default US region
# Other Langfuse data regions: https://cloud.langfuse.com (EU), https://jp.cloud.langfuse.com (Japan), https://hipaa.cloud.langfuse.com (HIPAA)
# export LANGFUSE_OTEL_HOST="https://otel.my-langfuse.company.com"  # custom OTEL endpoint
```

**2. litellm_config.yaml 설정하기**

```yaml litellm_config.yaml
model_list:
  - model_name: gpt-5.1
    litellm_params:
      model: gpt-5.1
litellm_settings:
  callbacks: ["langfuse_otel"]
```

**3. 프록시 시작하기**

```bash
litellm --config /path/to/litellm_config.yaml
```

**4. 프록시를 사용해 Langfuse에 트레이스 기록하기**

```bash
curl -X POST "http://0.0.0.0:4000/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-xxxx" \
  -d '{
    "model": "gpt-3.5-turbo",
    "messages": [
      {"role": "system", "content": "You are a very accurate calculator. You output only the result of the calculation."},
      {"role": "user", "content": "1 + 1 = "}
    ]
  }'
```

LiteLLM UI에서 콜백을 Langfuse로 설정하면 모든 프로바이더에 걸친 응답을 즉시 기록할 수 있습니다. Proxy UI 설정 방법에 대한 자세한 내용은 [LiteLLM 문서](https://docs.litellm.ai/docs/proxy/ui)를 참고하세요.

<Callout type="info">

요청을 트레이스로 그룹화하고, userId, 태그, sessionId 등을 추가하기 위해 요청에 추가 Langfuse 속성을 붙일 수 있습니다. 이러한 속성은 LiteLLM Proxy와 SDK 모두에 공통으로 적용되므로, 가능한 모든 옵션을 알아보려면 두 문서 페이지를 모두 참고하세요:

- [LiteLLM Proxy 문서](https://docs.litellm.ai/docs/proxy/logging#langfuse)
- [LiteLLM Python SDK 문서](https://litellm.vercel.app/docs/observability/langfuse_integration#advanced)

</Callout>
