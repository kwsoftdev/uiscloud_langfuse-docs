# Kong API Gateway를 통해 Langfuse로 AI API 트레이싱하기

이 가이드는 애플리케이션 코드를 수정하지 않고도 AI API 호출을 자동으로 모니터링, 디버그, 평가할 수 있도록 **Kong API Gateway**에 **Langfuse를 통합**하는 방법을 보여드립니다.

> **Kong API Gateway란?**: [Kong Gateway](https://konghq.com/)는 API와 마이크로서비스를 관리하는 클라우드 네이티브, 플랫폼 독립적, 확장 가능한 API Gateway입니다. API 트래픽의 중앙 제어 지점 역할을 하며, 인증, 속도 제한, 모니터링 같은 기능을 제공합니다.

> **Langfuse란?**: [Langfuse](https://langfuse.com/)는 AI 에이전트를 위한 오픈소스 관측성 플랫폼입니다. LLM 호출, 도구 사용, 비용, 지연 시간 등을 시각화하고 모니터링하는 데 도움을 줍니다.

## 기능

- **제로코드 계측**: Kong을 통해 프록시되는 AI API 호출에 대한 자동 트레이싱
- **다중 프로바이더 지원**: OpenAI 호환 API, vLLM, 커스텀 프로바이더
- **풍부한 컨텍스트 캡처**: 사용자 세션, 대화, 메타데이터
- **성능 메트릭**: 지연 시간, 처리량, 토큰 단위 분석
- **논블로킹 아키텍처**: 최소한의 오버헤드로 비동기 동작
- **프로덕션 준비 완료**: 오류 복원력과 우아한 성능 저하(graceful degradation)

### 지원되는 AI 프로바이더

| 프로바이더        | 엔드포인트                                                  | 상태 |
| ----------------- | ----------------------------------------------------------- | ---- |
| OpenAI-Compatible | `/v1/chat/completions`, `/v1/completions`, `/v1/embeddings` | ✅   |
| vLLM              | `/generate`, `/v1/completions`                              | ✅   |
| Custom Providers  | Extensible detection framework                              | ✅   |

## 1. Kong 플러그인 설치

아래에서는 사용 가능한 방법 중 하나로 Kong Langfuse Tracing 플러그인을 설치합니다.

### 사전 요구 사항

- Kong Gateway 3.0+ 설치 및 실행 중
- Langfuse 계정 ([가입하기](https://cloud.langfuse.com))
- Kong의 Admin API 접근 권한

### 옵션 1: LuaRocks를 통한 설치 (권장)

```bash
luarocks install kong-plugin-ai-tracing
```

### 옵션 2: 소스에서 설치

```bash
git clone https://github.com/Ramtinboreili/kong-langfuse-tracing.git
cd kong-langfuse-tracing
luarocks make rockspec/kong-plugin-ai-tracing-1.0.0-1.rockspec
```

### 옵션 3: Docker Compose

```yaml
version: '3.8'
services:
  kong:
    image: kong:3.4
    environment:
      KONG_PLUGINS: bundled,ai-tracing
      KONG_LUA_PACKAGE_PATH: /usr/local/kong/plugins/?.lua;;
      KONG_DATABASE: postgres
      KONG_PG_HOST: postgres
      KONG_PG_USER: kong
      KONG_PG_PASSWORD: kong
    volumes:
      - ./plugins/ai-tracing:/usr/local/kong/plugins/ai-tracing
    ports:
      - "8000:8000"
      - "8001:8001"
```

### Kong에서 플러그인 활성화

Kong 설정에 플러그인을 추가합니다:

```bash
# In kong.conf
plugins = bundled,ai-tracing

# Or via environment variable
export KONG_PLUGINS=bundled,ai-tracing
```

플러그인을 로드하려면 Kong Gateway를 재시작하세요.

## 2. Langfuse 자격 증명 설정

다음으로 Langfuse API 키를 설정하세요. 무료 [Langfuse Cloud](https://langfuse.com/cloud) 계정에 가입하거나 [Langfuse를 셀프 호스팅](https://langfuse.com/self-hosting)하면 이 키를 얻을 수 있습니다.

```bash
# Get keys for your project from the project settings page
export LANGFUSE_PUBLIC_KEY="pk-lf-..."
export LANGFUSE_SECRET_KEY="sk-lf-..."
export LANGFUSE_BASE_URL="https://cloud.langfuse.com" # 🇪🇺 EU region
# Other Langfuse data regions include 🇺🇸 US: https://us.cloud.langfuse.com, 🇯🇵 Japan: https://jp.cloud.langfuse.com and ⚕️ HIPAA: https://hipaa.cloud.langfuse.com
```

## 3. Kong Service에서 플러그인 활성화

Kong의 Admin API를 사용해 AI 서비스에 플러그인을 구성합니다:

```bash
curl -X POST http://localhost:8001/services/YOUR_AI_SERVICE/plugins \
  -H "Content-Type: application/json" \
  -d '{
    "name": "ai-tracing",
    "config": {
      "langfuse_enabled": true,
      "langfuse_public_key": "pk-lf-...",
      "langfuse_secret_key": "sk-lf-...",
      "langfuse_endpoint": "https://cloud.langfuse.com/api/public/ingestion",
      "environment": "production"
    }
  }'
```

### 설정 매개변수

| 매개변수              | 타입    | 기본값                                            | 필수 여부 | 설명                                              |
| --------------------- | ------- | ------------------------------------------------- | --------- | ------------------------------------------------- |
| `langfuse_enabled`    | boolean | `false`                                           | Yes       | Langfuse 통합을 활성화/비활성화합니다             |
| `langfuse_public_key` | string  | -                                                 | Yes       | Langfuse public API 키                            |
| `langfuse_secret_key` | string  | -                                                 | Yes       | Langfuse secret API 키                            |
| `langfuse_endpoint`   | string  | `https://cloud.langfuse.com/api/public/ingestion` | No        | Langfuse API 엔드포인트                           |
| `langfuse_timeout`    | number  | `5000`                                            | No        | HTTP 타임아웃(밀리초)                             |
| `environment`         | string  | `production`                                      | No        | 트레이스 필터링을 위한 환경 태그                  |
| `log_level`           | string  | `info`                                            | No        | 로깅 상세 수준 (`debug`, `info`, `warn`, `error`) |

<Callout type="info">
셀프 호스팅 Langfuse 인스턴스를 사용하는 경우, `langfuse_endpoint`를 인스턴스 URL 뒤에 `/api/public/ingestion`을 붙인 값으로 업데이트하세요.
</Callout>

## 4. Hello World 예제

아래에서는 Kong Gateway를 통해 간단한 AI 요청을 보냅니다. 플러그인이 요청을 자동으로 캡처하여 Langfuse에 트레이스를 생성합니다.

```bash
curl -X POST http://kong-gateway:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-User-Id: user-12345" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {"role": "user", "content": "Explain quantum computing"}
    ],
    "temperature": 0.7,
    "max_tokens": 500
  }'
```

## 5. 헤더로 컨텍스트 추가하기

HTTP 헤더를 사용해 사용자 및 세션 컨텍스트로 트레이스를 보강하세요. 이렇게 하면 사용자, 세션, 대화별로 트레이스를 필터링하고 분석할 수 있습니다.

```bash
curl -X POST http://kong-gateway:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-User-Id: user-12345" \
  -H "X-Session-Id: session-abc" \
  -H "X-Chat-Id: chat-789" \
  -H "X-Organization-Id: org-acme" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {"role": "user", "content": "What is machine learning?"}
    ]
  }'
```

### 지원되는 컨텍스트 헤더

| 헤더                | 설명               | 예시          |
| ------------------- | ------------------ | ------------- |
| `X-User-Id`         | 고유 사용자 식별자 | `user-12345`  |
| `X-Session-Id`      | 세션 식별자        | `session-abc` |
| `X-Chat-Id`         | 대화/채팅 ID       | `chat-789`    |
| `X-Message-Id`      | 개별 메시지 ID     | `msg-54321`   |
| `X-Organization-Id` | 조직 컨텍스트      | `org-acme`    |
| `X-Project-Id`      | 프로젝트 식별자    | `project-xyz` |

## 6. 메타데이터 추가하기

더 풍부한 트레이스를 위해 요청 본문에 추가 메타데이터를 포함하세요. 기능, 실험, 사용자별 변수를 추적하는 데 유용합니다.

```bash
curl -X POST http://kong-gateway:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-User-Id: user-12345" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {"role": "user", "content": "Recommend a movie"}
    ],
    "metadata": {
      "user_id": "user-12345",
      "chat_id": "chat-789",
      "project_id": "project-xyz",
      "features": {
        "web_search": true,
        "image_generation": false
      },
      "variables": {
        "user_tier": "premium",
        "language": "en"
      }
    }
  }'
```

### Python / FastAPI 애플리케이션

```python
import httpx
from fastapi import FastAPI

KONG_URL = "http://kong-gateway:8000"

async def chat_with_ai(user_id: str, session_id: str, message: str):
    async with httpx.AsyncClient() as client:
        response = await client.post(
            f"{KONG_URL}/v1/chat/completions",
            json={
                "model": "gpt-4",
                "messages": [{"role": "user", "content": message}],
                "metadata": {
                    "user_id": user_id,
                    "session_id": session_id
                }
            },
            headers={
                "Content-Type": "application/json",
                "X-User-Id": user_id,
                "X-Session-Id": session_id
            }
        )
        return response.json()

# Usage
result = await chat_with_ai("user-123", "session-abc", "Hello!")
print(result)
```

## 캡처되는 데이터

이 플러그인은 다음 구조로 Langfuse에 포괄적인 트레이스를 생성합니다:

```json
{
  "trace": {
    "id": "trace-12345",
    "name": "/v1/chat/completions",
    "userId": "user-12345",
    "sessionId": "session-abc",
    "metadata": {
      "provider": "openai_compatible",
      "model": "gpt-4",
      "status_code": 200,
      "environment": "production",
      "total_duration_ms": 1250,
      "time_per_token_ms": 12.5,
      "throughput_tokens_per_second": 80.0
    }
  },
  "observations": [
    {
      "type": "generation",
      "name": "chat_completion",
      "usage": {
        "promptTokens": 150,
        "completionTokens": 25,
        "totalTokens": 175
      },
      "metadata": {
        "temperature": 0.7,
        "max_tokens": 500,
        "finish_reason": "stop"
      }
    }
  ]
}
```

### 성능 메트릭

- **총 소요 시간(Total Duration)**: 엔드투엔드 요청 처리 시간
- **토큰당 시간(Time per Token)**: 생성된 토큰당 평균 지연 시간
- **처리량(Throughput)**: 초당 처리된 토큰 수

### 토큰 분석

- **프롬프트 토큰(Prompt Tokens)**: 입력 토큰 수
- **완료 토큰(Completion Tokens)**: 출력 토큰 수
- **총 토큰(Total Tokens)**: 결합된 사용량
- **비용 추적(Cost Tracking)**: 요청 전반의 지출 모니터링

  --data "config.langfuse_enabled=true" \
  --data "config.langfuse_public_key=pk-lf-dev-xxx" \
  --data "config.langfuse_secret_key=sk-lf-dev-xxx" \
  --data "config.environment=development" \
  --data "config.log_level=debug"

## 문제 해결

### Langfuse에 데이터가 표시되지 않는 경우

1. **자격 증명 확인**: Langfuse API 키가 올바른지 확인합니다
2. **연결 확인**: Kong이 Langfuse 엔드포인트에 접근할 수 있는지 확인합니다

```bash
# View plugin status
curl http://localhost:8001/services/YOUR_SERVICE/plugins | \
  jq '.data[] | select(.name=="ai-tracing")'

# Check Kong logs
docker-compose logs kong | grep "ai-tracing"

# Test Langfuse endpoint
curl -I https://cloud.langfuse.com/api/public/ingestion
```

3. **로그 검토**: 오류가 있는지 Kong 로그를 확인합니다

### 사용자 컨텍스트 누락

- 요청에 HTTP 헤더가 올바르게 설정되었는지 확인하세요
- 헤더 이름이 예상 형식과 일치하는지 확인하세요(`X-UserId`가 아니라 `X-User-Id`)
- 헤더가 모든 프록시를 통해 전달되는지 확인하세요

### 성능 문제

- 고부하 상황에서 Kong 메트릭을 모니터링하세요
- 타임아웃이 발생하면 `langfuse_timeout`을 조정하세요
- Kong 로그에서 비동기 타이머 성능을 확인하세요

### 디버그 로깅 활성화

```bash
curl -X PATCH http://localhost:8001/plugins/PLUGIN_ID \
  --data "config.log_level=debug"
```

## 고급 사용법

### 커스텀 AI 프로바이더

`handler.lua`의 감지 로직을 수정하여 플러그인이 추가 AI 프로바이더를 지원하도록 확장할 수 있습니다:

```lua
local function detect_ai_provider(path, headers)
  if path:find("/v1/chat/completions") then
    return "openai_compatible"
  elseif path:find("/anthropic") then
    return "anthropic"
  elseif path:find("/cohere") then
    return "cohere"
  else
    return "custom_provider"
  end
end
```

### 다른 Kong 플러그인과의 통합

AI tracing 플러그인은 다른 Kong 플러그인과 함께 동작합니다:

- **Rate Limiting**: 비용 통제를 위해 속도 제한과 결합
- **Authentication**: 사용자 식별을 위해 key-auth나 JWT와 함께 사용
- **Request Transformer**: 트레이싱 전에 요청 수정

<Callout type="warning">
**보안 모범 사례**:
- Langfuse API 키를 안전하게 보관하세요(Kong의 vault 통합이나 환경 변수 사용)
- PII 준수를 위해 내보낸 데이터를 검토하세요
- 프로덕션에서 플러그인 설정 접근을 제한하세요
- Kong과 Langfuse 간 통신에 HTTPS를 사용하세요
- 민감한 콘텐츠에 대한 데이터 보존 정책을 고려하세요
</Callout>
