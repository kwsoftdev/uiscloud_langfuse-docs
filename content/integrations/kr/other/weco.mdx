# Automate Code Optimization with Weco and Langfuse

이 노트북은 Langfuse 데이터셋, 코드 평가자, 관리형 LLM-as-a-Judge 평가자를 사용해 LLM 애플리케이션 코드를 자동으로 최적화하기 위해 **Weco**를 **Langfuse**와 통합하는 방법을 단계별로 안내합니다.

> **Weco란 무엇인가요?** [Weco](https://www.weco.ai/)([GitHub](https://github.com/WecoAI/weco-cli))는 코드 최적화 플랫폼입니다. 소스 파일과 평가 함수가 주어지면 Weco의 최적화기가 코드를 반복적으로 수정하고, 다시 평가하고, 가장 점수가 높은 버전을 유지합니다. 정확도, 지연 시간, 비용 또는 사용자 지정 복합 점수 등 측정 가능한 모든 지표에 대해 작동합니다.

> **Langfuse란 무엇인가요?** [Langfuse](https://langfuse.com)는 오픈 소스 AI 엔지니어링 플랫폼입니다. LLM 애플리케이션을 위한 트레이싱 및 모니터링 기능을 제공합니다. Langfuse는 네이티브 통합, OpenTelemetry, 전용 SDK를 통해 다양한 도구 및 프레임워크와 통합하여 상세한 인사이트를 제공함으로써 개발자가 AI 시스템을 디버깅하고, 분석하고, 최적화할 수 있도록 돕습니다.

## 작동 방식

Weco는 평가 백엔드로 Langfuse에 연결됩니다. 각 최적화 단계마다 Weco는 다음을 수행합니다.

1. 소스 코드(예: 프롬프트, 파싱 로직)를 수정합니다
2. Langfuse **데이터셋**의 모든 항목에 대해 **대상 함수**를 실행합니다
3. Langfuse UI에서 구성된 로컬 **코드 평가자** 및/또는 **관리형 평가자**(LLM-as-a-Judge)로부터 점수를 수집합니다
4. 점수를 하나의 **지표**로 결합하고 가장 성능이 좋은 버전을 유지합니다

각 반복은 Langfuse에 새로운 **실험 실행(experiment run)**을 생성하므로 Langfuse 대시보드에서 모든 변형을 나란히 비교할 수 있습니다.

### Step 3: Langfuse에서 데이터셋 생성하기

Weco는 Langfuse 데이터셋에 대해 코드를 평가합니다. 각 데이터셋 항목에는 `input` 딕셔너리와 선택적인 `expected_output` 딕셔너리가 있습니다. 작은 QA 데이터셋을 만들어 보겠습니다.

```python
dataset = langfuse.create_dataset(name="weco-demo-qa")

qa_pairs = [
    {
        "input": {"question": "What is the capital of France?"},
        "expected_output": {"expected_answer": "Paris"},
    },
    {
        "input": {"question": "What is the largest planet in our solar system?"},
        "expected_output": {"expected_answer": "Jupiter"},
    },
    {
        "input": {"question": "Who wrote Romeo and Juliet?"},
        "expected_output": {"expected_answer": "William Shakespeare"},
    },
    {
        "input": {"question": "What is the boiling point of water in Celsius?"},
        "expected_output": {"expected_answer": "100 degrees Celsius"},
    },
    {
        "input": {"question": "What year did the Berlin Wall fall?"},
        "expected_output": {"expected_answer": "1989"},
    },
]

for pair in qa_pairs:
    langfuse.create_dataset_item(
        dataset_name="weco-demo-qa",
        input=pair["input"],
        expected_output=pair["expected_output"],
    )

langfuse.flush()
print(f"Created dataset 'weco-demo-qa' with {len(qa_pairs)} items.")
```

### Step 4: 대상 함수 작성하기

대상 함수는 Weco가 최적화할 코드입니다. 데이터셋에서 `inputs` 딕셔너리를 받아 출력 딕셔너리를 반환합니다. Langfuse는 각 평가 실행마다 데이터셋 항목당 한 번씩 이 함수를 호출합니다.

작업 디렉터리에 `agent.py`로 저장하세요.

```python
%%writefile agent.py
from openai import OpenAI

client = OpenAI()

SYSTEM_PROMPT = "You are a helpful assistant. Answer the question concisely in one sentence."


def answer_question(inputs: dict) -> dict:
    question = inputs.get("question", "")
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": question},
        ],
        temperature=0.0,
    )
    return {"answer": response.choices[0].message.content}
```

Weco는 평가 지표를 개선하기 위해 `agent.py`를 반복적으로 수정합니다 — `SYSTEM_PROMPT`, 응답 파싱, 또는 다른 로직을 변경합니다.

### Step 5: 평가자와 지표 함수 작성하기

코드 평가자는 각 대상 함수 출력을 채점하는 Python 함수입니다. 키워드 인자를 받아 Langfuse `Evaluation` 객체를 반환합니다.

지표 함수는 모든 평가자 점수를 Weco가 최적화하는 하나의 숫자로 결합합니다.

`evaluators.py`로 저장하세요.

```python
%%writefile evaluators.py
from langfuse import Evaluation


def answer_quality(*, input, output, expected_output=None, **kwargs):
    """Check that the answer is non-empty and reasonably concise."""
    answer = (output or {}).get("answer", "")
    if not answer:
        return Evaluation(name="answer_quality", value=0.0, comment="Empty answer")
    word_count = len(answer.split())
    score = 1.0 if word_count <= 50 else max(0.0, 1.0 - (word_count - 50) / 100)
    return Evaluation(
        name="answer_quality",
        value=score,
        comment=f"{word_count} words",
    )


def qa_metric(scores: dict) -> float:
    """Combine evaluator scores into a single optimization target.

    Multiplies answer_quality by Correctness (a managed evaluator
    configured in the Langfuse UI) so that only correct, concise
    answers score well.
    """
    return scores.get("answer_quality", 0.0) * scores.get("Correctness", 0.0)
```

### Step 6: Langfuse에서 관리형 평가자 구성하기

관리형 평가자는 실험 트레이스에서 서버 측으로 실행되는 LLM-as-a-Judge 평가자입니다. Langfuse UI에서 하나를 설정하세요.

1. [Langfuse](https://cloud.langfuse.com/)에서 프로젝트로 이동합니다
2. **Evaluation** → **LL-as-a-Judge**로 이동합니다
3. **+ Set up evaluator**를 클릭합니다

**Correctness** 평가자를 생성합니다.

- **Name**: `Correctness`
- **Score**: 0 또는 1(이진 사실 정확도)
- **변수 매핑**:
  - `{{input}}` → `$.input.question`
  - `{{output}}` → `$.output.answer`
  - `{{expected_output}}` → `$.expected_output.expected_answer`

> **중요:** 평가자 이름은 대소문자를 구분합니다. Langfuse UI의 이름(예: `Correctness`)은 `--langfuse-managed-evaluators`에 전달되는 이름 및 지표 함수에서 사용되는 키(`scores.get("Correctness")`)와 정확히 일치해야 합니다.

> 관리형 평가자는 각 실험 이후 비동기로 실행됩니다. Weco는 자동으로 점수를 폴링합니다(기본적으로 최대 15분). `--langfuse-managed-evaluator-timeout`으로 타임아웃을 조정할 수 있습니다.

### Step 7: 최적화 실행하기

데이터셋, 대상 함수, 평가자, 지표 함수가 준비되었으니 커맨드라인에서 Weco를 실행하세요. Weco는 Langfuse를 평가 백엔드로 사용하여 각 반복마다 데이터셋에 대해 대상 함수를 실행하고 Langfuse의 실험 실행으로 진행 상황을 추적합니다.

```python
!weco run --source agent.py \
  --eval-backend langfuse \
  --langfuse-dataset weco-demo-qa \
  --langfuse-target agent:answer_question \
  --langfuse-evaluators evaluators:answer_quality \
  --langfuse-managed-evaluators Correctness \
  --langfuse-metric-function evaluators:qa_metric \
  --metric qa_metric --goal maximize --steps 5 \
  --output plain  # For Jupyter-friendly formatting
```

### Step 8: 결과 확인하기

각 최적화 단계는 Langfuse에 새로운 실험 실행을 생성합니다. **Datasets** → **weco-demo-qa** → **Runs**로 이동하여 모든 변형에 대한 입력, 출력, 평가자 점수를 나란히 비교하세요.

**[Weco 대시보드](https://dashboard.weco.ai)에서 반복별 진행 상황을 추적할 수도 있으며, 여기에는 지표 점수와 Weco가 각 단계에서 수행한 정확한 코드 변경 사항이 표시됩니다.** 실행이 완료되면 가장 성능이 좋은 버전을 소스 파일에 적용하라는 메시지가 표시됩니다.

</Steps>

## 문제 해결

**Langfuse에 실험 실행이 나타나지 않습니다**

- `weco`를 실행하는 것과 동일한 셸 세션에서 `LANGFUSE_SECRET_KEY`, `LANGFUSE_PUBLIC_KEY`, `LANGFUSE_BASE_URL`이 export되어 있는지 확인하세요.
- `--langfuse-dataset`에 전달된 데이터셋 이름이 Langfuse 프로젝트에 있는 기존 데이터셋과 일치하는지 확인하세요.

**관리형 평가자 점수가 도착하지 않습니다**

- `--langfuse-managed-evaluators`에 전달된 평가자 이름이 Langfuse UI의 이름과 정확히 일치하는지(대소문자 구분) 확인하세요.
- 평가자 설정에서 과거 트레이스에 대한 실시간 미리보기를 사용해 변수 매핑을 확인하세요.
- 평가자가 느리다면 `--langfuse-managed-evaluator-timeout`으로 폴링 타임아웃을 늘리세요.

**지표가 0으로 유지됩니다**

- 점수 범위와 키 이름을 확인하기 위해 중간 평가자 출력을 출력해 보세요.
- 지표 함수에서 사용되는 키(`scores.get("Correctness")`)가 평가자 이름과 정확히 일치하는지 확인하세요.

**인증 오류**

- API 키를 다시 확인하고 base URL이 리전(EU 대 US)과 일치하는지 확인하세요.
- Langfuse 프로젝트 설정에서 프로젝트 레벨 키 권한을 확인하세요.
