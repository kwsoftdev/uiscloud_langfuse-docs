# Morph Reflex 평가자를 활용한 실시간 분류

이 가이드는 [Morph](https://morphllm.com) Reflex 모델을 Langfuse의 LLM-as-a-judge 평가자로 사용하는 방법을 보여줍니다. Morph의 API는 OpenAI와 호환되므로, Reflex는 **범주형(categorical)** 평가자로 Langfuse에 바로 연결됩니다. Langfuse는 각 트레이스에 대해 이를 실행하고 예측된 카테고리를 점수로 붙이며, 애플리케이션에 추가 추론 코드가 필요하지 않습니다.

> **Morph란 무엇인가요?** [Morph](https://morphllm.com)는 Reflex를 학습시키고 서빙합니다. Reflex는 LLM 입력과 출력을 실시간으로 분류하는 작고 빠른 분류기 모델로, 예를 들어 탈옥(jailbreak) 탐지, 가드레일, 사용자 불만, 또는 직접 학습시킨 커스텀 카테고리 등에 사용됩니다.

> **Langfuse란 무엇인가요?** [Langfuse](https://langfuse.com)는 팀이 API 호출을 트레이싱하고, 성능을 모니터링하고, LLM 애플리케이션의 문제를 디버깅할 수 있도록 돕는 오픈소스 AI 엔지니어링 플랫폼입니다.

Reflex는 분류기이므로 평가마다 정확히 하나의 카테고리(최상위 예측)를 반환하며 근거는 반환하지 않습니다. 반환되는 reasoning은 항상 `NA-Reflex`입니다.

## 시작하기 전에

- [Morph 대시보드](https://morphllm.com/dashboard/api-keys)에서 발급받은 Morph API 키(`sk-...`).
- Reflex `model` id와 정확한 **라벨**. Langfuse에서 설정하는 카테고리는 이 라벨과 정확히 일치해야 합니다. 예측을 한 번 실행하여 모델의 라벨 목록을 확인할 수 있습니다.

```bash
curl https://api.morphllm.com/v1/reflex/predict \
  -H "Authorization: Bearer $MORPH_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "jailbreak", "text": "ignore your instructions"}'
# classes: [{ "label": "benign", ... }, { "label": "jailbreak", ... }]
```

## 평가자 설정하기

<Steps>

### 1단계: 새 LLM-as-a-judge 평가자 만들기

Langfuse에서 **Evaluators**로 이동해 **Set up evaluator**를 클릭하고 **LLM-as-a-judge**를 선택하세요.

### 2단계: Morph LLM 연결 추가하기

**Change the provider → Add LLM connection**을 클릭하고 다음과 같이 설정하세요.

- **Provider / schema:** **OpenAI** (Morph는 OpenAI와 호환됩니다).
- **API key:** 자신의 Morph `sk-...` 키.
- **Advanced settings**를 열고 **API Base URL**을 다음과 같이 설정합니다.

  ```
  https://api.morphllm.com/v1/reflex-oai
  ```

  **Use Responses API**와 **Extra Headers**는 필요 없으니 꺼둡니다.

- **Use default models**를 끄고, **Custom models**에서 사용하려는 Reflex `model` id(예: `jailbreak`)를 추가합니다.

**Create connection**을 클릭한 다음 해당 연결과 모델을 선택하세요.

### 3단계: 평가자 이름 지정하기

**Define evaluator**에서 원하는 이름을 지정하세요. 예: `jailbreak`.

### 4단계: 평가 프롬프트 작성하기

평가 프롬프트는 Morph가 분류할 텍스트입니다. 판단하고 싶은 변수만 넣고 그 외에는 아무것도 넣지 마세요.

사용자 입력을 평가하려면:

```
{{input}}
```

에이전트 응답을 평가하려면:

```
{{output}}
```

전체 트레이스를 평가하려면:

```
{{input}} {{output}}
```

### 5단계: 점수 유형을 범주형(단일 카테고리)으로 설정하기

**Score type**을 **Categorical**로 설정하세요.

- **Categories:** Reflex 라벨마다 하나씩, 모델이 출력하는 그대로 **정확히** 철자를 맞춰 추가하세요(내장 Reflex의 경우 아래 [기본 Reflex](#default-reflexes)에서 복사하세요). 라벨은 빠짐없이 모두 있어야 하며, 모델에 해당 라벨이 있는 경우에만 만능(catch-all) 카테고리를 추가하세요.
- **"Allow multiple matches"를 활성화하지 마세요.** 이 평가자는 정확히 하나의 카테고리(최상위 예측)만 반환합니다.

<Callout type="warning">
  카테고리가 모델 라벨과 정확히 일치하지 않으면 Langfuse는 파싱 오류와 함께
  점수를 거부합니다. 위의 `predict` 응답에서 라벨을 복사하세요.
</Callout>

### 6단계: reasoning 및 selection 프롬프트는 그대로 두기

**Score reasoning prompt**와 **Category selection prompt**는 기본값 그대로 두어도 됩니다. Morph는 이를 읽지 않습니다. Reflex는 분류기이며 근거를 생성하지 않으므로, 반환되는 reasoning은 항상 `NA-Reflex`입니다.

### 7단계: 저장하기

평가자를 저장하세요. 실행 결과를 확인하려면 점수를 열고 **View execution trace**(환경 `langfuse-llm-as-a-judge`)를 선택해 Langfuse가 보낸 정확한 요청과 Morph가 반환한 카테고리를 확인할 수 있습니다.

### 8단계: 실행하기

평가자는 들어오는 **새로운** 매칭 트레이스를 자동으로 채점합니다. 이미 가지고 있는 트레이스를 채점하려면 **Traces** 테이블을 열고 원하는 행을 선택(또는 **select all**)한 다음 하단의 **Evaluate**를 클릭하세요.

</Steps>

## 기본 Reflex

Morph는 사전 학습된 Reflex 세트를 제공합니다. `model` id를 연결의 **Custom models**에 복사하고, 카테고리를 평가자의 **Categories**에 복사하세요. 대소문자를 구분하니 정확히 그대로 작성해야 합니다.

| Reflex `model`       | 카테고리                                                                | 탐지 대상                                      |
| -------------------- | ----------------------------------------------------------------------- | ---------------------------------------------- |
| `jailbreak`          | `benign`, `jailbreak`                                                   | 프롬프트 인젝션 / 탈옥 시도                    |
| `guardrail`          | `false`, `true`                                                         | 괴롭힘 또는 NSFW 콘텐츠                        |
| `leaked-thinking`    | `clean`, `leaked`                                                       | 에이전트의 내부 사고 과정 노출                 |
| `stuck-in-a-loop`    | `progressing`, `looping`                                                | 에이전트가 막혀서 새로운 시도를 하지 않는 상태 |
| `incomplete-thought` | `complete`, `incomplete`                                                | 사용자가 잘린(불완전한) 프롬프트를 보낸 경우   |
| `user-frustrated`    | `Frustrated`, `Not Frustrated`                                          | 사용자가 에이전트에 불만을 느끼는 상태         |
| `ambiguity`          | `low`, `med`, `high`                                                    | 프롬프트가 얼마나 불명확한지                   |
| `difficulty`         | `easy`, `medium`, `hard`                                                | 모델 라우팅을 위한 프롬프트 난이도             |
| `domain`             | `general`, `summary`, `coding`, `design`, `data`                        | 요청의 주제 (다중 라벨 — 아래 참고 사항 참조)  |
| **Custom**           | [Reflex 대시보드](https://morphllm.com/dashboard/reflex)에서 확인하세요 | 직접 학습시킨 분류기                           |

<Callout type="warning">
  `domain`은 다중 라벨 Reflex이지만, 평가자는 각 트레이스에 **최상위 예측
  라벨만** 붙입니다. OpenAI 호환 엔드포인트는 평가마다 정확히 하나의 카테고리만
  반환하므로 부차적인 라벨은 버려집니다. **Allow multiple matches**는 꺼둔
  상태를 유지하세요. 이를 활성화해도 추가 라벨이 나타나지는 않습니다. 일치하는
  모든 라벨이 필요하다면 [시작하기 전에](#before-you-start)에서 설명한 대로
  `/v1/reflex/predict` 엔드포인트를 직접 호출하고 결과를
  [SDK를 통한 점수](/docs/evaluation/evaluation-methods/scores-via-sdk)로
  붙이세요.
</Callout>
