# Langfuse와 Quarkus LangChain4j 통합하기

이 가이드는 OpenTelemetry를 사용하여 [Langfuse](/)를 [Quarkus LangChain4j](https://docs.quarkiverse.io/quarkus-langchain4j/dev/index.html)와 통합하는 방법을 보여줍니다.

**Quarkus LangChain4j**: AI 호출에 대한 내장 OTel 트레이싱을 갖춘 [LangChain4j](https://docs.langchain4j.dev/)의 [Quarkus](https://quarkus.io/) 기반 통합입니다.

**Langfuse**: 옵저버빌리티, 평가, 프롬프트 관리를 위한 오픈소스 AI 엔지니어링 플랫폼입니다.

<Callout type="info" emoji="🐞">
  이 통합에 문제가 있으면 [GitHub](/issues)에 이슈를 등록해 주세요.
</Callout>

## 1단계: Quarkus LangChain4j에서 OpenTelemetry 활성화하기

**Quarkus OpenTelemetry 종속성 추가** (Maven):

Maven의 경우 `pom.xml`에 다음을 추가하세요 (Gradle 사용자는 Gradle에서 동등한 좌표를 포함할 수 있습니다):

```xml
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-opentelemetry</artifactId>
</dependency>
```

**OpenTelemetry 익스포터 구성** (`application.properties`):

```properties
quarkus.otel.exporter.otlp.traces.protocol=http/protobuf
```

이러한 구성과 종속성을 갖추면 Quarkus 애플리케이션은 OpenTelemetry 트레이스를 생성할 준비가 됩니다. Quarkus LangChain4j 내부 호출(예: 채팅 모델을 호출할 때)은 스팬으로 기록됩니다.

각 스팬은 `gen_ai.operation.name`, `gen_ai.system`(제공업체, 예: "openai"), 모델 이름, 토큰 사용량 등의 속성을 가집니다.
프롬프트 및 응답 콘텐츠에 대한 이벤트를 활성화하려면 langchain4j 프롬프트 트레이싱을 활성화해야 합니다.

**Langchain4j 프롬프트 트레이싱 구성** (`application.properties`):

```properties
quarkus.langchain4j.tracing.include-prompt=true
quarkus.langchain4j.tracing.include-completion=true
```

## 2단계: Langfuse 구성하기

이제 Quarkus 애플리케이션이 OpenTelemetry 트레이스 데이터를 내보내고 있으므로 다음 단계는 해당 데이터를 Langfuse로 보내는 것입니다.

이 설정에서 Langfuse는 OpenTelemetry의 "백엔드" 역할을 합니다. 즉, 일반적인 Jaeger/Zipkin/OTel-Collector를 Langfuse의 트레이스 수집 API로 대체하는 것입니다.

**Langfuse 설정**

- [Langfuse Cloud](https://langfuse.com/cloud) 또는 [셀프 호스팅 Langfuse](https://langfuse.com/self-hosting)에 가입하세요.
- OTLP 엔드포인트(예: `https://cloud.langfuse.com/api/public/otel`)와 API 키를 설정하세요.

<Tabs items={["환경 변수", "`application.properties`에서 구성"]}>
<Tab>

이를 환경 변수를 통해 구성합니다:

```bash
QUARKUS_OTEL_EXPORTER_OTLP_ENDPOINT: set this to the Langfuse OTLP URL (e.g. https://cloud.langfuse.com/api/public/otel).
QUARKUS_OTEL_EXPORTER_OTLP_HEADERS: set this to Authorization=Basic <base64 public:secret>,x-langfuse-ingestion-version=4.
```

</Tab>
<Tab>

`application.properties`에서 다음과 같이 구성합니다:

```bash
quarkus.otel.exporter.otlp.headers=Authorization=Basic <base64 of public:key>,x-langfuse-ingestion-version=4
quarkus.otel.exporter.otlp.endpoint=https://cloud.langfuse.com/api/public/otel
quarkus.otel.exporter.otlp.traces.protocol=http/protobuf
```

</Tab>
</Tabs>

<Callout type="info">
  Basic Auth를 통한 인증에 대한 자세한 내용은
  [여기](/docs/opentelemetry/get-started)에서 확인할 수 있습니다.
</Callout>

## 3단계: 테스트 AI 작업 실행하기

Quarkus 애플리케이션을 시작하세요. Quarkus LangChain4j가 처리하는 AI 작업을 트리거하세요 – 예를 들어, `ChatModel`을 사용하여 완성을 생성하는 서비스나 컨트롤러를 호출합니다.

**참고**: 전체 예제는 [여기](https://github.com/langfuse/langfuse-examples/tree/main/applications/quarkus-langchain4j-demo)에서 찾을 수 있습니다.

```java
@RegisterAiService(tools = EmailService.class)
public interface MyAiService {

    /**
     * Ask the LLM to create a poem about the given topic.
     *
     * @param topic the topic of the poem
     * @param lines the number of line of the poem
     * @return the poem
     */
    @SystemMessage("You are a professional poet")
    @UserMessage("""
            Write a single poem about {topic}. The poem should be {lines} lines long and your response should only include the poem itself, nothing else.
            Then send this poem by email. Your response should include the poem.
            """)
    String writeAPoem(String topic, int lines);

}

@Singleton
public class Startup {

    public void writeAPoem(@Observes StartupEvent event, MyAiService service) {
        System.out.println(service.writeAPoem("Langfuse", 4));
    }
}
```

## 문제 해결

**트레이스가 표시되지 않는 경우:**

- 잠재적인 단서를 찾기 위해 애플리케이션의 로그를 확인하세요.
- [문제 해결](/faq/all/self-hosting-missing-events-after-ingestion) 페이지를 확인하세요.
