# 통합: Amazon Bedrock AgentCore

> **Amazon Bedrock AgentCore란 무엇인가요?** [Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agents/)는 프로덕션 환경에서 AI 에이전트를 구축, 배포, 관리할 수 있게 해주는 관리형 서비스입니다. 컨테이너화된 에이전트 런타임을 제공하여 Amazon Bedrock의 파운데이션 모델을 활용하면서 복잡한 워크플로를 실행하고, 도구를 사용하고, 외부 API와 상호작용할 수 있습니다.

> **Langfuse란 무엇인가요?** [Langfuse](https://langfuse.com/)는 LLM 엔지니어링을 위한 오픈 소스 플랫폼입니다. AI 에이전트를 위한 트레이싱 및 모니터링 기능을 제공하여 개발자가 제품을 디버그, 분석, 최적화할 수 있도록 돕습니다. Langfuse는 네이티브 통합, OpenTelemetry, SDK를 통해 다양한 도구 및 프레임워크와 통합됩니다.

## 시작하기

이 가이드는 OpenTelemetry를 사용하여 에이전트 실행을 트레이싱하기 위해 Langfuse와 Amazon Bedrock AgentCore를 통합하는 방법을 보여줍니다.

<Steps>
### 1단계: 의존성 설치

Langfuse 트레이싱과 함께 AgentCore 에이전트를 구축하고 배포하는 데 필요한 Python 패키지를 설치합니다:

```python
pip install bedrock-agentcore-starter-toolkit strands-agents[otel] langfuse boto3 mcp
```

### 2단계: 환경 변수 설정

AWS 및 Langfuse 자격 증명을 구성합니다:

**2.1 Langfuse 자격 증명 및 OTEL Exporter 구성**

```python
import os
import base64

# Get keys for your project from the project settings page: https://cloud.langfuse.com
os.environ.setdefault("LANGFUSE_PUBLIC_KEY", "pk-lf-...")
os.environ.setdefault("LANGFUSE_SECRET_KEY", "sk-lf-...")
os.environ.setdefault("LANGFUSE_BASE_URL", "https://cloud.langfuse.com")  # 🇪🇺 EU region (default)
# Other Langfuse data regions include 🇺🇸 US: https://us.cloud.langfuse.com, 🇯🇵 Japan: https://jp.cloud.langfuse.com and ⚕️ HIPAA: https://hipaa.cloud.langfuse.com

# Build Basic Auth header for OTEL
langfuse_auth = base64.b64encode(
    f"{os.environ['LANGFUSE_PUBLIC_KEY']}:{os.environ['LANGFUSE_SECRET_KEY']}".encode()
).decode()

# Configure OpenTelemetry endpoint & headers
os.environ["OTEL_EXPORTER_OTLP_ENDPOINT"] = os.environ["LANGFUSE_BASE_URL"] + "/api/public/otel"
os.environ["OTEL_EXPORTER_OTLP_HEADERS"] = f"Authorization=Basic {langfuse_auth},x-langfuse-ingestion-version=4"
```

**2.2 AWS 자격 증명 구성**

Bedrock 서비스에 액세스하기 위한 AWS 자격 증명을 설정합니다:

```python
os.environ["AWS_ACCESS_KEY_ID"] = "..."
os.environ["AWS_SECRET_ACCESS_KEY"] = "..."
os.environ["AWS_DEFAULT_REGION"] = "us-west-2"
```

### 3단계: Langfuse 트레이싱이 포함된 에이전트 생성하기

OpenTelemetry를 통해 Langfuse와 통합되는 AgentCore 에이전트를 생성합니다. 이 예제는 MCP 도구와 함께 Strands Agents SDK를 사용합니다. 다른 에이전트 프레임워크도 Langfuse와 함께 사용할 수 있으며, 다른 프레임워크를 계측하는 방법에 대한 가이드는 [통합 페이지](/integrations)를 참조하세요.

```python
from bedrock_agentcore.runtime import BedrockAgentCoreApp
from strands import Agent
from strands.models import BedrockModel
from strands.telemetry import StrandsTelemetry
from mcp.client.streamable_http import streamablehttp_client
from strands.tools.mcp.mcp_client import MCPClient
from langfuse import get_client

# Initialize MCP client for tool access
streamable_http_mcp_client = MCPClient(
    lambda: streamablehttp_client("https://langfuse.com/api/mcp")
)

# Configure Bedrock model
bedrock_model = BedrockModel(
    model_id="us.anthropic.claude-3-7-sonnet-20250219-v1:0",
    region_name="us-west-2",
    temperature=0.0,
    max_tokens=4096
)

# Define system prompt
system_prompt = """You are an experienced agent supporting developers with
questions about Langfuse and LLM observability."""

app = BedrockAgentCoreApp()

@app.entrypoint
def agent_entrypoint(payload):
    """Agent entrypoint with Langfuse tracing"""
    user_input = payload.get("prompt")
    trace_id = payload.get("trace_id")
    parent_obs_id = payload.get("parent_obs_id")

    # Initialize Strands telemetry and setup OTLP exporter
    strands_telemetry = StrandsTelemetry()
    strands_telemetry.setup_otlp_exporter()

    # Create agent with MCP tools
    with streamable_http_mcp_client:
        mcp_tools = streamable_http_mcp_client.list_tools_sync()

        agent = Agent(
            model=bedrock_model,
            system_prompt=system_prompt,
            tools=mcp_tools
        )

        # Execute within Langfuse trace context
        with get_client().start_as_current_observation(
            name='agentcore-agent',
            trace_context={
                "trace_id": trace_id,
                "parent_observation_id": parent_obs_id
            }
        ):
            response = agent(user_input)

    return response.message['content'][0]['text']

if __name__ == "__main__":
    app.run()
```

### 4단계: 에이전트 배포 및 호출

트레이스 컨텍스트와 함께 Amazon Bedrock AgentCore에 에이전트를 배포하고 호출합니다:

```python
from bedrock_agentcore_starter_toolkit import Runtime
from langfuse import get_client
import boto3
import json

# Deploy agent
runtime = Runtime()
runtime.configure(
    entrypoint="./agent.py",
    auto_create_execution_role=True,
    auto_create_ecr=True,
    agent_name="langfuse-traced-agent",
    memory_mode='NO_MEMORY'
)

launch_result = runtime.launch(
    env_vars={
        "DISABLE_ADOT_OBSERVABILITY": "true",  # Required: disable ADOT to use Langfuse
        "BEDROCK_MODEL_ID": "us.anthropic.claude-3-7-sonnet-20250219-v1:0",
        "OTEL_EXPORTER_OTLP_ENDPOINT": os.environ["OTEL_EXPORTER_OTLP_ENDPOINT"],
        "OTEL_EXPORTER_OTLP_HEADERS": os.environ["OTEL_EXPORTER_OTLP_HEADERS"],
        "LANGFUSE_PROJECT_NAME": "your-project-name",
        "SYSTEM_PROMPT": system_prompt
    }
)

# Invoke agent with trace context
client = boto3.client('bedrock-agentcore', region_name='us-west-2')

# Get current trace context from Langfuse
trace_id = get_client().get_current_trace_id()
obs_id = get_client().get_current_observation_id()

payload = json.dumps({
    "prompt": "What is Langfuse and how does it help monitor LLM applications?",
    "trace_id": trace_id,
    "parent_obs_id": obs_id
}).encode()

response = client.invoke_agent_runtime(
    agentRuntimeArn=launch_result.agent_arn,
    runtimeSessionId="session-123",
    payload=payload
)
```

### 5단계: Langfuse에서 트레이스 보기

에이전트를 실행한 후, [Langfuse](https://cloud.langfuse.com)에 로그인하여 생성된 트레이스를 살펴보세요. 다음을 확인할 수 있습니다:

- 전체 에이전트 실행 흐름
- 토큰 수와 비용이 포함된 LLM 호출
- 도구 사용 및 MCP 상호작용
- 각 단계의 지연 시간 메트릭
- 디버깅을 위한 입력/출력 데이터

이 트레이스는 프로덕션 환경에서 에이전트의 동작에 대한 포괄적인 가시성을 제공합니다.

</Steps>

## AgentCore를 다른 프레임워크와 함께 사용하기

AgentCore에서 LangChain, LlamaIndex 또는 다른 프레임워크를 사용하는 경우: Langfuse SDK 콜백만으로는 AgentCore에서 작동하지 않습니다. AgentCore 런타임은 SDK의 HTTP 호출을 우회할 수 있는 수준에서 텔레메트리를 관리합니다.

AgentCore 런타임을 실행할 때 다음 환경 변수를 전달하세요:

```bash
# Disable AWS's built-in tracing
DISABLE_ADOT_OBSERVABILITY=true

# Configure OTEL to export to Langfuse
OTEL_EXPORTER_OTLP_ENDPOINT="https://cloud.langfuse.com/api/public/otel"
OTEL_EXPORTER_OTLP_HEADERS="Authorization=Basic $(echo -n 'pk-xxx:sk-xxx' | base64),x-langfuse-ingestion-version=4"
```

다른 리전의 경우 엔드포인트 호스트를 그에 맞게 교체하세요: `https://us.cloud.langfuse.com/api/public/otel`(US), `https://jp.cloud.langfuse.com/api/public/otel`(일본), 또는 `https://hipaa.cloud.langfuse.com/api/public/otel`(HIPAA).

다음은 필요한 환경 변수와 함께 AgentCore 런타임을 실행하는 방법의 예시입니다:

```python
import base64

# Prepare auth header
langfuse_auth = base64.b64encode(
    f"{LANGFUSE_PUBLIC_KEY}:{LANGFUSE_SECRET_KEY}".encode()
).decode()

# Launch with required env vars
launch_result = agentcore_runtime.launch(
    env_vars={
        "DISABLE_ADOT_OBSERVABILITY": "true",
        "OTEL_EXPORTER_OTLP_ENDPOINT": f"{LANGFUSE_BASE_URL}/api/public/otel",
        "OTEL_EXPORTER_OTLP_HEADERS": f"Authorization=Basic {langfuse_auth},x-langfuse-ingestion-version=4",
    }
)
```

문제 해결에 대한 더 자세한 내용은 [기존 OpenTelemetry 설정과 함께 Langfuse 사용하기](/faq/all/existing-otel-setup#aws-bedrock-agentcore-adot)를 참조하세요.
