# Langfuse로 Restate 에이전트 트레이싱하기

이 가이드는 [Restate](https://restate.dev/) 에이전트와 워크플로우에 Langfuse를 통합하여 LLM 호출, 도구 호출, 지속적(durable) 워크플로우 단계 등 완전한 옵저버빌리티를 하나의 통합된 트레이스로 제공하는 방법을 보여줍니다.

> **Restate란 무엇인가요?** [Restate](https://restate.dev/)는 에이전트와 워크플로우를 재개 가능하고 복원력 있게 만드는 지속적 실행(durable execution) 플랫폼입니다. 비결정적 작업(LLM 호출, 도구 API 호출, MCP 호출)은 모두 지속적 저널에 저장됩니다. 실패 시 Restate는 저널을 재생하여 중단된 지점부터 재개합니다 – 자동 재시도, 복구, 멱등 실행을 포함합니다.

> **Langfuse란 무엇인가요?** [Langfuse](https://langfuse.com/)는 AI 에이전트를 위한 오픈소스 옵저버빌리티 플랫폼입니다. LLM 호출, 도구 사용, 비용, 지연 시간을 모니터링하고 자동화된 평가를 실행하는 데 도움을 줍니다.

## 지원되는 에이전트 프레임워크

Restate는 OpenAI Agents SDK, Pydantic AI, Google ADK, 순수 Restate 에이전트 등 여러 에이전트 프레임워크에 대한 [SDK 통합](https://docs.restate.dev/ai)을 제공합니다.
아래 예제에서는 OpenAI Agents SDK를 사용합니다. Langfuse와 Restate를 다른 프레임워크와 통합하는 방법을 알아보려면 [Restate 문서](https://docs.restate.dev/ai/ecosystem-integrations/langfuse)를 참고하세요.

## 버전 관리

Restate의 [버전 관리 모델](https://docs.restate.dev/operate/versioning)은 새 배포가 최신 버전으로 새 요청을 라우팅하는 동안 진행 중인 실행은 시작된 버전에서 계속되도록 보장합니다. 이는 각 Langfuse 트레이스가 하나의 불변 아티팩트 – 하나의 코드 버전, 하나의 프롬프트 버전, 하나의 실행 이력 – 에 연결됨을 의미하며, 버전 간 품질을 비교하고 회귀를 발견하기 쉽게 만듭니다.

## 1. 종속성 설치

```bash
pip install restate-sdk openai-agents langfuse openinference-instrumentation-openai-agents hypercorn
```

## 2. 환경 구성

API 키를 설정합니다. Langfuse 키는 [Langfuse Cloud](https://langfuse.com/cloud) 또는 [셀프 호스팅 Langfuse](https://langfuse.com/self-hosting)에서 얻을 수 있습니다.

```bash filename=".env"
LANGFUSE_PUBLIC_KEY=pk-lf-...
LANGFUSE_SECRET_KEY=sk-lf-...
LANGFUSE_BASE_URL=https://cloud.langfuse.com
OPENAI_API_KEY=sk-proj-...
```

## 3. 에이전트 정의하기

[Restate의 OpenAI Agents SDK 통합](https://docs.restate.dev/ai/sdk-integrations/openai-agents-sdk)을 사용하여 에이전트 단계를 지속적으로 만듭니다. `DurableRunner`는 각 LLM 호출을 Restate의 저널에 저장하므로 실패한 실행은 처음부터 다시 시작하는 대신 중단된 지점에서 재개됩니다.

```python filename="agent.py"
import restate
from agents import Agent
from restate.ext.openai import restate_context, DurableRunner, durable_function_tool

# Durable tool — executed exactly once, even across retries
@durable_function_tool
async def get_weather(city: str) -> dict:
    """Get the current weather for a given city."""

    # Do durable steps using the Restate context
    async def call_weather_api(city: str) -> dict:
        return {"temperature": 23, "description": "Sunny and warm."}

    return await restate_context().run_typed("Get weather", call_weather_api, city=city)


# AGENT
weather_agent = Agent(
    name="WeatherAgent",
    instructions="You are a helpful agent that provides weather updates.",
    tools=[get_weather],
)


# AGENT SERVICE
agent_service = restate.Service("agent")


@agent_service.handler()
async def run(_ctx: restate.Context, message: str) -> str:
    # Runner that persists the agent execution for recoverability
    result = await DurableRunner.run(weather_agent, message)
    return result.final_output


if __name__ == "__main__":
    import hypercorn
    import asyncio
    from agent import agent_service

    app = restate.app(services=[agent_service])

    conf = hypercorn.Config()
    conf.bind = ["0.0.0.0:9080"]
    asyncio.run(hypercorn.asyncio.serve(app, conf))
```

## 4. Langfuse 트레이싱 활성화하기

Langfuse 클라이언트를 초기화하고 트레이싱 프로세서를 설정합니다. 이는 OpenAI Agents SDK 스팬을 Restate의 실행 트레이스에 연결하여 Langfuse에서 모든 것이 하나의 통합된 트레이스로 나타나게 합니다.

```python filename="agent.py"
from langfuse import get_client
from opentelemetry import trace as trace_api
from openinference.instrumentation import OITracer, TraceConfig
from openinference.instrumentation.openai_agents._processor import OpenInferenceTracingProcessor
from agents import set_trace_processors
from restate.ext.tracing import RestateTracer

# Initialize Langfuse (sets up the global OTel tracer provider + exporter)
langfuse = get_client()
tracer = OITracer(
    RestateTracer(trace_api.get_tracer("openinference.openai_agents")),
    config=TraceConfig(),
)
set_trace_processors([OpenInferenceTracingProcessor(tracer)])
```

`RestateTracer`는 OpenAI Agents SDK 스팬을 Restate의 상위 스팬 아래로 평탄화하므로 Langfuse의 트레이스 계층 구조가 실제 실행 흐름을 그대로 반영합니다.

Restate는 자체 실행 트레이스(워크플로우 단계, 재시도, 복구)도 OpenTelemetry 스팬으로 내보냅니다. Restate의 트레이싱 엔드포인트를 Langfuse로 지정하면 에이전트 스팬과 워크플로우 스팬이 모두 동일한 트레이스에 나타납니다.

## 5. Langfuse에서 트레이스 보기

에이전트를 실행한 후([지침 참조](https://docs.restate.dev/ai/ecosystem-integrations/langfuse)), Langfuse의 트레이스에는 에이전트 단계와 워크플로우 단계가 모두 표시됩니다. LLM 호출의 경우 입력, 프롬프트, 모델 구성, 출력을 검사할 수 있습니다.

## Restate로 프롬프트 관리하기

Restate와 함께 [Langfuse 프롬프트 관리](/docs/prompt-management/overview)를 사용할 수 있습니다. 각 프롬프트 가져오기는 지속적 단계가 되어 – 재시도는 동일한 프롬프트를 재사용하고 새 실행은 업데이트된 버전을 가져옵니다.

```python
from langfuse import get_client

langfuse = get_client()

def fetch_prompt() -> str:
    prompt = langfuse.get_prompt("claim-agent", type="text")
    return prompt.compile()

# Durably journaled — same prompt is used on retries
prompt = await ctx.run_typed("Fetch prompt", fetch_prompt)
```
