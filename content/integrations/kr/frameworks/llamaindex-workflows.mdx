# LlamaIndex Workflows에 대한 관찰 가능성

이 쿡북은 [Langfuse](https://langfuse.com)를 사용해 [LlamaIndex Workflows](https://docs.llamaindex.ai/en/stable/module_guides/workflow/)에 대한 실시간 관찰 가능성을 확보하는 방법을 보여줍니다.

> **LlamaIndex Workflows란 무엇인가요?** [LlamaIndex Workflows](https://docs.llamaindex.ai/en/stable/module_guides/workflow/)는 견고한 AI 에이전트를 구축하기 위해 설계된 유연한 이벤트 기반 프레임워크입니다. LlamaIndex에서 워크플로우는 각각 `@step` 데코레이터로 정의되고 검증되는 여러 단계를 연결하여 만들어집니다. 각 단계는 특정 이벤트 유형을 처리하며, 이를 통해 AI 에이전트 협업, RAG 흐름, 데이터 추출 등과 같은 복잡한 프로세스를 조율할 수 있습니다.

> **Langfuse란 무엇인가요?** [Langfuse](https://langfuse.com)는 오픈소스 AI 엔지니어링 플랫폼입니다. 팀이 협업하여 프롬프트를 관리하고, 애플리케이션을 트레이싱하고, 문제를 디버그하고, 프로덕션에서 LLM 시스템을 평가할 수 있도록 돕습니다.

환경 변수가 설정되면 이제 Langfuse 클라이언트를 초기화할 수 있습니다. `get_client()`는 환경 변수로 제공된 자격 증명을 사용해 Langfuse 클라이언트를 초기화합니다.

```python
from langfuse import get_client

langfuse = get_client()

# Verify connection
if langfuse.auth_check():
    print("Langfuse client is authenticated and ready!")
else:
    print("Authentication failed. Please check your credentials and host.")

```

### 4단계: 간단한 LlamaIndex Workflows 애플리케이션 만들기

LlamaIndex Workflows에서는 `@step` 데코레이터로 단계를 정의하여 이벤트 기반 AI 에이전트를 구축합니다. 각 단계는 이벤트를 처리하고, 적절한 경우 새로운 이벤트를 발생시킵니다. 이 예제에서는 두 단계로 이루어진 간단한 워크플로우를 만듭니다. 하나는 들어오는 이벤트를 전처리하고, 다른 하나는 답변을 생성합니다.

```python
from llama_index.core.llms import ChatMessage
from llama_index.llms.openai import OpenAI
from typing import Annotated

from workflows import Workflow, step
from workflows.events import StartEvent, StopEvent
from workflows.resource import Resource


def get_llm(**kwargs):
    return OpenAI(model="gpt-4.1-mini")


class MyWorkflow(Workflow):
    @step
    async def step1(
        self, ev: StartEvent, llm: Annotated[OpenAI, Resource(get_llm)]
    ) -> StopEvent:
        msg = ChatMessage(role="user", content=ev.get("input"))
        response = await llm.achat([msg])
        return StopEvent(result=response.message.content)


w = MyWorkflow()
```

```python
response = await w.run(input="Hello, what is Langfuse?")
print(response)
```
