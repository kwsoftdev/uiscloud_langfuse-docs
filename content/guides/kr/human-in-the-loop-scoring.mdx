# 휴먼인더루프 점수 워크플로우 구축하기

많은 팀이 LLM 출력을 평가하기 위해 사람을 프로세스에 참여시킵니다(human-in-the-loop). Langfuse는 이를 위한 내장 [주석 큐](/docs/evaluation/evaluation-methods/annotation-queues)를 제공하지만, 이미 자체 내부 리뷰 도구를 보유하고 있거나 리뷰어가 기존 앱 안에서 작업하기를 원하는 경우도 있습니다.

이 가이드는 그러한 커스텀 도구를 Langfuse에 연결하는 방법을 보여줍니다. 리뷰어가 확인해야 할 트레이스를 가져온 다음, SDK나 API를 통해 리뷰어의 판단을 [점수](/docs/evaluation/scores/overview)로 다시 수집합니다. 이러한 점수를 [점수 설정(score config)](/faq/all/manage-score-configs)에 맞춰 표준화하면 이후 분석 시 일관성을 유지할 수 있습니다.

<Callout type="info">
  리뷰 UI를 직접 소유할 필요가 없다면 대신 Langfuse [주석
  큐](/docs/evaluation/evaluation-methods/annotation-queues)를 사용하세요. 별도의
  도구를 만들지 않아도 내장 리뷰어 인터페이스, 큐 관리, 점수 설정 기능을 바로
  제공합니다. 주석 큐에 필요한 기능이 없어서 직접 UI를 만들 생각이라면, 꼭
  알려주시기 바랍니다. GitHub에서 [기능
  요청](https://github.com/orgs/langfuse/discussions/new?category=ideas)을
  열어주세요.
</Callout>

## 사전 준비 사항

- 애플리케이션에서 이미 Langfuse로 트레이스가 전송되고 있어야 합니다. 아직 설정하지 않았다면 [트레이싱](/docs/observability/get-started)을 참고하세요.
- 리뷰어가 트레이스를 확인하고 판단을 제출하는 커스텀 주석 또는 리뷰 UI가 있어야 합니다.
- Langfuse SDK가 설치되어 있고, 환경 변수에 자격 증명(`LANGFUSE_PUBLIC_KEY`, `LANGFUSE_SECRET_KEY`, `LANGFUSE_BASE_URL`)이 설정되어 있어야 합니다.

## 진행 방법

<Steps>

### 점수 설정 정의하기

[점수 설정(score config)](/faq/all/manage-score-configs)은 리뷰어가 점수를 매길 때 사용하는 스키마를 표준화하여, 모든 리뷰어가 동일한 형태의 데이터를 제출하도록 합니다. 이 설정은 먼저 Langfuse에서 생성해야 합니다.

_예를 들어 고객 지원 QA 도구라면 `excellent`, `acceptable`, `poor` 카테고리를 가진 `support_quality`라는 이름의 범주형(categorical) 설정을 정의할 수 있습니다._

설정 방법은 [점수 설정을 생성하고 관리하는 방법](/faq/all/manage-score-configs)을 참고하고, 결과로 생성된 `configId`를 기록해 두세요. 이는 이후 점수를 수집할 때 참조하게 됩니다.

### 리뷰어에게 트레이스 노출하기

리뷰어가 확인해야 할 트레이스를 가져온 다음, 해당 입력과 출력을 도구에 렌더링합니다. SDK 또는 [퍼블릭 API](/docs/api-and-data-platform/features/public-api)를 통해 이를 수행할 수 있으며, 이름, 사용자, 태그, 기간 등으로 필터링하여 리뷰 큐를 구성할 수 있습니다.

<LangTabs items={["Python SDK", "JS/TS SDK", "API"]}>
<Tab>

```python
from langfuse import get_client

langfuse = get_client()

# 리뷰할 최근 트레이스 목록 조회, 예: 특정 이름을 가진 모든 트레이스
response = langfuse.api.trace.list(name="support-conversation", limit=50)

for trace in response.data:
    # 리뷰 도구에서 trace.input / trace.output을 렌더링
    print(trace.id, trace.input)
```

_[Python SDK 레퍼런스](https://python.reference.langfuse.com)를 참고하세요._

</Tab>
<Tab>

```ts
import { LangfuseClient } from "@langfuse/client";

const langfuse = new LangfuseClient();

// 리뷰할 최근 트레이스 목록 조회, 예: 특정 이름을 가진 모든 트레이스
const response = await langfuse.api.trace.list({
  name: "support-conversation",
  limit: 50,
});

for (const trace of response.data) {
  // 리뷰 도구에서 trace.input / trace.output을 렌더링
  console.log(trace.id, trace.input);
}
```

_[JS/TS SDK 레퍼런스](https://js.reference.langfuse.com)를 참고하세요._

</Tab>
<Tab>

```bash
curl -G https://cloud.langfuse.com/api/public/traces \
  -u "pk-lf-...":"sk-lf-..." \
  --data-urlencode "name=support-conversation" \
  --data-urlencode "limit=50"
```

_[API 레퍼런스](https://api.reference.langfuse.com)를 참고하세요._

</Tab>
</LangTabs>

### 리뷰어의 판단을 점수로 수집하기

리뷰어가 판단을 제출하면, 1단계에서 얻은 `configId`를 참조하여 이를 점수로 Langfuse에 다시 기록합니다. 이렇게 하면 값이 스키마에 맞춰 검증됩니다.

리뷰어가 평가하는 대상에 따라 점수를 연결하세요: 단일 응답(`trace_id`, 그리고 트레이스 내 특정 단계를 지정하려면 선택적으로 `observation_id`), 또는 대화 전체를 평가할 때는 [세션](/docs/observability/features/sessions)(`session_id`)에 연결할 수 있습니다.

<LangTabs items={["Python SDK", "JS/TS SDK", "API"]}>
<Tab>

```python
from langfuse import get_client

langfuse = get_client()

# 단일 응답에 점수 매기기
langfuse.create_score(
    trace_id="trace_id_here",
    name="support_quality",
    value="acceptable",  # 설정된 카테고리 중 하나와 일치해야 함
    data_type="CATEGORICAL",
    config_id="your_config_id",  # 설정된 값에 대해 검증 수행
    comment="Resolved the issue but tone was a bit terse.",
)

# 또는 trace_id 대신 session_id를 전달하여 전체 대화에 점수 매기기
langfuse.create_score(
    session_id="session_id_here",
    name="support_quality",
    value="excellent",
    data_type="CATEGORICAL",
    config_id="your_config_id",
)
```

_[Python SDK 레퍼런스](https://python.reference.langfuse.com)를 참고하세요._

</Tab>
<Tab>

```ts
import { LangfuseClient } from "@langfuse/client";

const langfuse = new LangfuseClient();

// 단일 응답에 점수 매기기
langfuse.score.create({
  traceId: "trace_id_here",
  name: "support_quality",
  value: "acceptable", // 설정된 카테고리 중 하나와 일치해야 함
  dataType: "CATEGORICAL",
  configId: "your_config_id", // 설정된 값에 대해 검증 수행
  comment: "Resolved the issue but tone was a bit terse.",
});

// 또는 traceId 대신 sessionId를 전달하여 전체 대화에 점수 매기기
langfuse.score.create({
  sessionId: "session_id_here",
  name: "support_quality",
  value: "excellent",
  dataType: "CATEGORICAL",
  configId: "your_config_id",
});

// 짧게 실행되는 환경에서는 점수를 flush 하세요
await langfuse.flush();
```

_[JS/TS SDK 레퍼런스](https://js.reference.langfuse.com)를 참고하세요._

</Tab>
<Tab>

```bash
curl -X POST https://cloud.langfuse.com/api/public/scores \
  -u "pk-lf-...":"sk-lf-..." \
  -H "Content-Type: application/json" \
  -d '{
    "traceId": "trace_id_here",
    "name": "support_quality",
    "value": "acceptable",
    "dataType": "CATEGORICAL",
    "configId": "your_config_id",
    "comment": "Resolved the issue but tone was a bit terse."
  }'
```

_[API 레퍼런스](https://api.reference.langfuse.com)를 참고하세요._

</Tab>
</LangTabs>

설정(config)을 참조하면 Langfuse는 점수를 저장하기 전에 이를 검증합니다: 이름은 설정과 일치해야 하며, 범주형(categorical) 값은 설정된 카테고리 중 하나에 매핑되어야 하고, 숫자형 값은 설정된 범위 내에 있어야 합니다. 전체 검증 규칙은 [점수 설정 강제 적용하기](/docs/evaluation/evaluation-methods/scores-via-sdk#enforcing-a-score-config)를 참고하세요.

</Steps>

## 점수로 할 수 있는 일

리뷰 결과가 수집되기 시작하면, 해당 점수는 Langfuse UI에서 연결된 트레이스 및 세션에 표시됩니다. 이를 필터링하고 분석하거나, 퍼블릭 API로 가져와 자체 대시보드를 구축할 수 있으며, [점수 분석](/docs/evaluation/scores/score-analytics)을 사용하여 시간에 따른 리뷰어 간 일치도와 품질 추이를 추적할 수 있습니다.
