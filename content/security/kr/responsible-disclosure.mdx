# 책임 있는 공개

저희는 보안 커뮤니티를 소중히 여기며 시스템의 보안을 최우선으로 생각합니다. 사용자와 고객의 보안 및 개인정보를 보호할 수 있도록 보안 취약점의 책임 있는 공개를 권장합니다.

## 취약점 신고

Langfuse에서 보안 취약점을 발견하셨다고 생각되면 [ClickHouse Bugcrowd 프로그램](https://bugcrowd.com/engagements/clickhouse)을 통해 실행 가능한 취약점 보고서를 제출해 주세요.

보고서에는 다음 세부 정보를 포함해 주세요.

- 취약점에 대한 명확한 설명과 잠재적인 영향
- 취약점을 재현하기 위한 단계, 필요한 특정 구성이나 조건 포함
- 취약점을 입증하는 개념 증명(PoC) 코드, 스크립트 또는 스크린샷

저희는 일반적으로 영업일 기준 2일 이내에 보고서 접수를 확인하며, 문제를 파악하고 해결하기 위해 함께 협력하겠습니다.

## Langfuse 테스트하기

보안 연구자는 책임 있는 공개 테스트를 위해 Langfuse Cloud 무료 플랜을 사용할 수 있습니다. 테스트에 무료 플랜보다 더 많은 규모가 필요한 경우 대신 Langfuse OSS를 셀프 호스팅해 주세요. 책임 있는 공개 테스트를 위해 무료 플랜 이상의 추가 Langfuse Cloud 크레딧은 제공하지 않습니다. 현재 플랜 한도 및 포함된 사용량은 [Langfuse Cloud 요금제 페이지](/pricing)를 참고하세요.

## 중점 대상 영역

- 부적절하거나 누락된 권한 부여(authorization) 검사
- 안전하지 않은 역할 할당 또는 권한 상승
- 특히 조직 간에 할당된 권한을 벗어나 리소스에 접근하거나 작업을 수행할 수 있는 능력

## 대상 범위 제외

다음 항목은 대상 범위에서 제외되며 접수되지 않습니다.

- **서비스 거부(DoS) 공격**, 캡처된 프론트엔드 호출을 기반으로 한 볼류메트릭 공격이나 반복적인 API 호출을 포함
- 구체적인 보안 영향을 입증하지 못하는 **속도 제한(rate limiting) 누락**
- 수동 검증이나 실제 작동하는 개념 증명 없이 제출된 **자동화된 스캐너 결과물**
- 실제 공격(exploit) 입증이 없는 **보안 헤더 누락**(예: CSP, X-Frame-Options) 또는 민감하지 않은 쿠키의 Secure/HttpOnly 플래그 누락
- **원치 않는 메시지**: 스팸이나 기타 원치 않는 메시지를 Langfuse 사용자에게 발송하게 되는 테스트
- Langfuse 직원이나 사용자를 대상으로 한 **사회공학(social engineering)** 또는 피싱 공격

## 저희의 약속

- 신고된 취약점을 신속하게 조사합니다.
- 진행 상황을 지속적으로 알려드립니다.
- 확인된 취약점을 해결하기 위해 적절한 조치를 취합니다.
- 원하시는 경우 취약점이 수정된 후 기여 사실을 공개적으로 인정해 드립니다.

## 버그 바운티 프로그램

Langfuse는 [ClickHouse Bugcrowd 프로그램](https://bugcrowd.com/engagements/clickhouse)에 포함되어 있습니다. 테스트나 보고서 제출 전에 자격 요건, 대상 범위, 보상 세부 정보는 Bugcrowd를 확인해 주세요.
