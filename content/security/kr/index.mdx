# 보안 및 컴플라이언스 개요

**Langfuse는 데이터 프라이버시와 보안을 최우선으로 합니다.** 고객이 Langfuse에 맡기는 데이터가 비즈니스에 매우 중요한 자산이라는 것을 잘 알고 있으며, 이를 최대한 신중하게 다루고 있습니다.

Langfuse는 매년 SOC2 Type 2 및 ISO27001 감사와 외부 침투 테스트를 실시하는 등 데이터 보안과 프라이버시에 대한 약속을 적극적으로 실천하고 있습니다. 관련 보고서는 [여기](/request-security-docs)에서 접근 권한을 요청할 수 있습니다.

Langfuse는 다음과 같은 사항에 중점을 두고 엔터프라이즈 요구 사항을 고려하여 구축되었습니다.

- **보안 조치:** 강력한 [암호화](/security/encryption), 접근 통제, 정기적인 [침투 테스트](/security/penetration-testing)를 시행합니다.
- **프라이버시 조치:** [GDPR](/security/gdpr)과 같은 규정에 따라 사용자 데이터를 보호합니다. [DPA](/dpa)를 제공하며, [HIPAA 대응 리전](/security/hipaa)을 지원하고, [개인정보처리방침](/privacy)을 준수합니다.
- **투명성:** 오픈소스 코어와 [소프트웨어 종속성](/security/dependencies)에 대한 명확한 정보를 제공합니다.
- **보고 체계:** [Bugcrowd를 통한 책임 있는 취약점 공개](/security/responsible-disclosure)와 [내부 고발](/security/whistleblowing)을 위한 명확한 채널을 제공합니다.

## Langfuse Cloud 보안 모델

Langfuse Cloud는 완전관리형 멀티테넌트 SaaS 배포입니다. 보안 모델은 세 가지 계층으로 구성됩니다.

- **보안 태세:** 프로덕션 서비스는 동일한 오픈소스 Langfuse 코드베이스를 기반으로 하며, SOC 2 Type II 및 ISO 27001 감사를 받고, 매년 제3자 침투 테스트를 수행합니다. 최신 보고서는 [여기](/request-security-docs)에서 요청할 수 있습니다.
- **테넌트 격리:** 모든 제품 데이터는 프로젝트 단위로 범위가 지정됩니다. 모든 레코드는 `projectId`와 연결되며, API 키는 프로젝트 단위로 범위가 지정되고, 인증된 요청은 쿼리가 실행되기 전에 RBAC를 통해 권한이 검증됩니다. 자세한 내용은 [보안 FAQ](/security/security-faq)와 [RBAC 문서](/docs/administration/rbac)를 참고하세요.
- **고객 제어:** 팀은 [마스킹](/docs/observability/features/masking), [데이터 보존](/docs/administration/data-retention), [데이터 삭제](/docs/administration/data-deletion), 리전 선택, SSO/SCIM, 감사 로그를 통해 Langfuse로 전송되는 데이터의 양과 보관 기간을 줄일 수 있습니다.

Langfuse Cloud는 격리된 리전 환경에서 AWS와 ClickHouse Cloud 위에서 운영됩니다. Postgres, ClickHouse, Redis, S3와 같은 지원 서비스는 동일한 클라우드 보안 프로그램의 적용을 받습니다: 프라이빗 네트워크 배치, 최소 권한 서비스 접근, 저장 및 전송 중 암호화, 모니터링, 벤더/컴플라이언스 검토가 포함됩니다. 자체 계정이나 VPC에서 인프라 수준의 격리가 필요한 경우, [셀프 호스팅 Langfuse](/self-hosting)를 이용하거나 엔터프라이즈 옵션에 대해 문의해 주세요.

## 프라이버시

Langfuse는 [GDPR](/security/gdpr)을 준수하며, [개인 데이터 처리를 관리](/security/manage-personal-data)할 수 있도록 데이터 보존, 데이터 마스킹, 데이터 삭제 기능을 제공합니다. Langfuse와 [DPA](/dpa)를 체결할 수 있습니다.

## Langfuse에 대한 일반 정보

### Langfuse란 무엇인가요?

Langfuse는 트레이싱, 프롬프트 관리, 평가, 메트릭을 제공하여 팀이 LLM 기반 애플리케이션을 디버깅하고 지속적으로 개선할 수 있도록 돕는 **오픈소스 AI 엔지니어링 플랫폼**입니다.

### 어떤 배포 모델을 이용할 수 있나요?

- **Langfuse Cloud** – 미국, EU, 일본, HIPAA 데이터 리전을 지원하는 완전관리형 SaaS(멀티테넌트)
- **셀프 호스팅 OSS** – 자체 인프라에 배포할 수 있는 MIT 라이선스 소프트웨어
- **셀프 호스팅 엔터프라이즈 에디션** – 추가 보안/컴플라이언스 기능과 벤더 지원이 포함된 상용 라이선스

### 어떤 클라우드 제공업체와 리전을 사용하나요?

Langfuse Cloud는 주로 **AWS와 ClickHouse Cloud** 위에서 운영됩니다.

- **미국 및 HIPAA 리전**: us-west-2 (오레곤)
- **EU 리전**: eu-west-1 (아일랜드)
- **일본 리전**: ap-northeast-1 (일본)

셀프 호스팅 고객은 원하는 리전이나 제공업체를 자유롭게 선택할 수 있습니다. Langfuse 셀프 호스팅은 완전히 오프라인/에어갭 환경에서도 운영할 수 있습니다.
