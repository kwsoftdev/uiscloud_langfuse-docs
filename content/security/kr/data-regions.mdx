# 데이터 리전 및 가용성

Langfuse Cloud는 고가용성을 위해 설계되었으며, 다양한 요구 사항을 충족하기 위해 여러 데이터 리전을 제공합니다.

데이터베이스와 애플리케이션은 AWS 인프라에서 실행되며, 일부는 Clickhouse가 관리합니다.

## Langfuse Cloud 리전

모든 데이터, 사용자 계정, 인프라는 리전 간에 완전히 분리되어 있습니다. 각 리전에 별도의 계정을 만들 수 있습니다.

| 리전                                       | URL                                                                    | 위치                        |
| ------------------------------------------ | ---------------------------------------------------------------------- | --------------------------- |
| **US**                                     | [`https://us.cloud.langfuse.com`](https://us.cloud.langfuse.com)       | 오리건 (AWS `us-west-2`)    |
| **EU**                                     | [`https://cloud.langfuse.com`](https://cloud.langfuse.com)             | 아일랜드 (AWS `eu-west-1`)  |
| **일본**                                   | [`https://jp.cloud.langfuse.com`](https://jp.cloud.langfuse.com)       | 도쿄 (AWS `ap-northeast-1`) |
| **HIPAA** ([자세히 보기](/security/hipaa)) | [`https://hipaa.cloud.langfuse.com`](https://hipaa.cloud.langfuse.com) | 오리건 (AWS `us-west-2`)    |

> 모든 데이터와 계정은 리전 간에 완전히 분리되어 있으므로, 리전을 전환하려면 새 계정을 생성하고 데이터를 마이그레이션해야 합니다. 프로젝트 간에 트레이스, 프롬프트, 데이터셋을 이전하는 스크립트는 [데이터 마이그레이션 쿡북](/guides/cookbook/example_data_migration)을 참고하세요.

## 리전에 연결하기

Langfuse SDK를 사용해 특정 데이터 리전에 연결하려면 base URL 환경 변수를 설정하거나 초기화 시 전달해야 합니다.

- **Python:** `LANGFUSE_BASE_URL` 환경 변수를 설정하거나 `base_url` 매개변수를 사용합니다.
- **JS/TS:** `LANGFUSE_BASE_URL` 환경 변수를 설정하거나 `baseUrl` 매개변수를 사용합니다.

Base URL 예시:

- US: `https://us.cloud.langfuse.com`
- EU: `https://cloud.langfuse.com`
- 일본: `https://jp.cloud.langfuse.com`
- HIPAA: `https://hipaa.cloud.langfuse.com`

자세한 설정 방법은 [SDK 문서](/docs/sdk)를 참고하세요.

## 리전 선택하기

데이터 리전을 선택할 때는 다음 요소를 고려하세요.

- **컴플라이언스 및 데이터 프라이버시 요구 사항:** 조직의 데이터 거주지 요구 사항에 맞는 리전을 선택하세요 (예: GDPR은 보통 EU 리전을 선호합니다).
- **프롬프트 관리 지연 시간:** Langfuse [프롬프트 관리](/docs/prompts)를 사용하는 경우, 프롬프트를 가져올 때 지연 시간을 줄이려면 애플리케이션 서버와 가까운 리전을 선택하세요. 프롬프트는 Langfuse SDK에 의해 캐싱되므로 이 차이는 거의 체감되지 않습니다.
- **UI 접근 지연 시간:** Langfuse 웹 인터페이스를 더 빠르게 사용하려면 팀의 위치와 가까운 리전을 선택하세요.

덜 중요한 요소:

- **트레이싱 수집 지연 시간:** 트레이스 데이터는 비동기적으로 배치 전송되므로, 수집 지연 시간이 애플리케이션 성능에 직접적인 영향을 미치는 경우는 적습니다.

## 비즈니스 연속성 및 가용성

| 항목            | 세부 내용                                                                                                                                                                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **고가용성**    | AWS 기반의 다중 AZ 데이터베이스 및 로드밸런싱된 상태 비저장(stateless) 애플리케이션 계층.                                                                                                                                                  |
| **복구 정밀도** | RPO: 10분 (Postgres PITR). RTO: 12시간.                                                                                                                                                                                                    |
| **백업 계층**   | Postgres: 7일 보관. ClickHouse: 6시간마다 백업 (4일 보관). S3: 7일 버저닝 보관.                                                                                                                                                            |
| **내구성**      | 암호화된 백업은 AWS/ClickHouse Cloud의 기본(primary) 리전 내 여러 가용 영역에 저장되며, 최소 연 1회 복원 무결성 테스트를 수행합니다. Postgres 백업은 추가로 보조 AWS 리전에 복제됩니다 (아래 [교차 리전 백업](#cross-region-backup) 참고). |
| **상태 페이지** | [https://status.langfuse.com](https://status.langfuse.com)에서 과거 가동 시간 및 장애 이력을 확인할 수 있습니다.                                                                                                                           |

### 교차 리전 백업 [#cross-region-backup]

AWS 리전이 영구적으로 손실되는 경우에도 복구할 수 있도록, 저장 데이터의 일부는 동일한 법적 관할권 내의 보조 AWS 리전으로 복제됩니다.
이는 재해 복구(disaster-recovery) 통제이며 액티브-액티브 장애 조치(failover)를 제공하지는 않습니다. 리전 전체에 장애가 발생하면 Langfuse는 보조 리전에서 애플리케이션 스택을 재구축하고 복제된 백업으로부터 복원합니다.
예상 재구축 시간은 최대 12시간입니다.

| 데이터 저장소                    | 기본(primary) 리전                                                       | 보조(secondary) 리전                                    | 방식                                             | 보관 기간       |
| -------------------------------- | ------------------------------------------------------------------------ | ------------------------------------------------------- | ------------------------------------------------ | --------------- |
| **Postgres**                     | EU: `eu-west-1` <br/> US / HIPAA: `us-west-2` <br/> JP: `ap-northeast-1` | `eu-central-1` <br/> `us-east-2` <br/> `ap-northeast-3` | AWS Backup 일일 스냅샷 복사, 전용 CMK로 재암호화 | 각 리전에서 7일 |
| **ClickHouse** (트레이싱 데이터) | 위 Cloud 리전과 동일                                                     | 복제되지 않음                                           | —                                                | —               |
| **S3 미디어 버킷**               | 위 Cloud 리전과 동일                                                     | 복제되지 않음                                           | —                                                | —               |

Postgres에는 조직, 프로젝트, 사용자, API 키, 프롬프트, 데이터셋, 주석, 점수 설정 데이터가 저장됩니다.
이를 교차 리전으로 복제함으로써 리전 장애 발생 시 고객의 개입 없이도 프로젝트, 자격 증명, 프롬프트 관리를 복원할 수 있습니다.

ClickHouse에는 과거 트레이스, 관찰(observation), 점수가 저장됩니다.
ClickHouse Cloud는 현재 관리형 교차 리전 백업을 제공하지 않으며, Langfuse도 리전 외부 사본을 별도로 유지하지 않습니다.
**기본 AWS 리전이 영구적으로 손실될 경우, 과거 트레이싱 데이터는 Langfuse에서 복구할 수 없으며, 복원된 환경은 새로운 데이터 수집부터 다시 시작합니다.**
대체 환경이 가동되면 트레이싱 수집이 다시 시작됩니다.

S3 미디어 버킷에는 트레이스에서 참조하는 업로드된 미디어 항목이 저장됩니다. 이 버킷은 교차 리전으로 복제되지 않습니다.
**기본 AWS 리전이 영구적으로 손실될 경우, 참조하는 트레이스가 살아남더라도 업로드된 미디어 항목은 복구할 수 없습니다.**

모든 보조 리전은 해당 기본 리전과 동일한 법적 관할권 내에 있으므로(EU ↔ EU, US ↔ US, 일본 ↔ 일본), 교차 리전 복제는 [DPA](/dpa) 및 [HIPAA 리전 개요](/security/hipaa)에 명시된 데이터 거주지 및 국경 간 이전 정책을 변경하지 않습니다.

<Callout type="info">
  **리전 장애로부터 트레이싱 데이터 보호하기:** ClickHouse 트레이싱 데이터와
  S3 미디어 버킷은 교차 리전으로 복제되지 않으므로, 트레이스와 관찰을
  자체 버킷으로 지속적으로 내보내려면 [블롭 스토리지
  통합](/docs/api-and-data-platform/features/export-to-blob-storage)을
  사용하는 것을 권장합니다. 해당 버킷이 Langfuse Cloud 리전 외부에 있다면,
  트레이싱 데이터는 리전 장애로부터 보호됩니다.
</Callout>

## 셀프 호스팅 인스턴스

데이터와 인프라를 완전히 직접 제어하고 싶다면 Langfuse를 셀프 호스팅할 수도 있습니다. 설치 및 설정 방법은 [셀프 호스팅 가이드](/self-hosting)를 참고하세요.
