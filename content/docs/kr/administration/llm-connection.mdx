# LLM 연결

LLM 연결(LLM connections)은 Langfuse Playground에서 모델을 호출하거나 LLM-as-a-Judge 평가에 사용됩니다.

## 설정

<Tabs items={["UI를 통한 설정", "API를 통한 설정"]}>

<Tab>

1. `Project Settings` > `LLM Connections`로 이동하여 `Add new LLM API key`를 **클릭합니다**.
2. LLM 연결의 이름과 사용하려는 모델의 API 키를 입력합니다.

<Video
  src="https://static.langfuse.com/docs-videos/2026-03-06-add-llm-connection-v3.mp4"
  gifStyle
  className="mt-4"
/>

</Tab>

<Tab>

[API](/docs/api-and-data-platform/features/public-api)를 사용하여 LLM 연결을 관리할 수 있습니다.

```bash
GET /api/public/llm-connections
PUT /api/public/llm-connections
```

</Tab>

</Tabs>

## 고급 구성

### 추가 제공업체 옵션

<Callout type="info">
  제공업체 옵션(Provider options)은 **Project Settings > LLM Connections**
  페이지에서 설정하는 것이 아니라, [Playground](/docs/prompt-management/features/playground)에서
  LLM 연결을 선택할 때 또는 [LLM-as-a-Judge](/docs/evaluation/evaluation-methods/llm-as-a-judge)
  평가기 설정 중에 설정합니다.
</Callout>

생성된 LLM 연결의 LLM 호출은 `temperature`, `top_p`, `max_tokens`와 같은 특정 파라미터 집합으로 구성할 수 있습니다.

하지만 많은 LLM 제공업체는 모델을 호출할 때 `reasoning_effort`, `service_tier` 등을 포함한 추가 파라미터를 허용합니다. 이러한 파라미터는 제공업체마다 다른 경우가 많습니다.

모든 LLM 호출에 대해 추가 구성을 JSON 객체로 제공할 수 있습니다. 모델 파라미터 설정에서 하단에 있는 "provider options" 필드를 찾을 수 있습니다. 이 필드를 통해 사용 중인 LLM 제공업체의 API 엔드포인트가 허용하는 특정 키-값 쌍을 입력할 수 있습니다.

지원되는 추가 필드는 사용 중인 제공업체의 API 레퍼런스를 참조하세요.

- [Anthropic Messages API Reference](https://docs.anthropic.com/en/api/messages)
- [OpenAI Chat Completions API Reference](https://platform.openai.com/docs/api-reference/chat/create)
- [Google Gemini API Reference](https://ai.google.dev/api)
- [Google Vertex AI Gemini API Reference](https://cloud.google.com/vertex-ai/generative-ai/docs/model-reference/inference)

이 기능은 현재 다음 어댑터에서 사용할 수 있습니다.

- Anthropic
- OpenAI
- Google AI Studio
- Google Vertex AI
- AWS (Amazon Bedrock)

OpenAI gpt-5 호출에서 reasoning effort를 `minimal`로 강제하는 예시입니다.

```json
{
  "thinkingLevel": "MEDIUM",
  "thinkingBudget": 2048

```

`thinkingLevel`은 추론 깊이를 제어하고, `thinkingBudget`은 thinking에 사용할 토큰 예산을 설정합니다. 이 옵션은 thinking을 지원하는 Google 모델에서만 유효합니다.

### 게이트웨이를 통한 연결

[LiteLLM](/integrations/gateways/litellm), [OpenRouter](/integrations/gateways/openrouter), [Portkey](/integrations/gateways/portkey)와 같은 OpenAI 호환 게이트웨이를 통해 LLM 호출을 라우팅하는 경우, 이를 Playground 및 LLM-as-a-Judge 평가를 위한 LLM 연결로 사용할 수 있습니다.

설정 방법은 다음과 같습니다.

1. **Project Settings** > **LLM Connections**로 이동하여 **Add new LLM API key**를 클릭합니다.
2. 제공업체로 **OpenAI**를 선택합니다.
3. 게이트웨이의 API 키를 입력합니다.
4. **Advanced Settings**에서 **Base URL**을 게이트웨이의 엔드포인트로 설정합니다. 예: `https://your-litellm-instance.com/v1`.
5. 게이트웨이가 제공하는 **custom model names**를 추가합니다.

<Callout type="info">
  LLM-as-a-Judge 평가가 동작하려면 게이트웨이가 OpenAI 형식의
  [tool calling](https://platform.openai.com/docs/guides/function-calling)을
  지원해야 합니다.
</Callout>

OpenAI 호환 게이트웨이의 경우, Langfuse에서 LLM-as-a-Judge를 지원하려면 게이트웨이가 처리해야 하는 tool calling 요청 예시는 다음과 같습니다.

### OpenAI Responses API 엔드포인트

일부 OpenAI 호환 제공업체는 Chat Completions API 대신 [OpenAI Responses API](https://platform.openai.com/docs/api-reference/responses)를 제공합니다. 이러한 제공업체의 경우, OpenAI LLM 연결에서 **Use Responses API**를 활성화하면 Langfuse가 Playground, LLM-as-a-Judge, 프롬프트 실험 요청을 Responses API를 통해 전송합니다.

예를 들어 Bedrock Mantle 엔드포인트를 통해 Amazon Bedrock에서 OpenAI 모델을 사용하려면 다음과 같이 합니다.

1. **Project Settings** > **LLM Connections**로 이동하여 **Add new LLM API key**를 클릭합니다.
2. 제공업체로 **OpenAI**를 선택합니다.
3. [Amazon Bedrock API 키](https://docs.aws.amazon.com/bedrock/latest/userguide/api-keys.html)를 입력합니다.
4. **Advanced Settings**에서 **Base URL**을 `https://bedrock-mantle.<aws-region>.api.aws/openai/v1`로 설정합니다.
5. **Use Responses API**를 활성화합니다.
6. Bedrock 계정에서 사용 가능한 **custom model names**를 추가합니다. 예: `openai.gpt-5.5` 또는 `openai.gpt-5.4`.
