Langfuse의 감사 로깅 시스템은 모든 시스템 활동을 포괄적으로 추적하며, 누가 어떤 작업을 수행했는지, 언제 발생했는지, 어떤 변경이 이루어졌는지에 대한 상세한 정보를 캡처합니다.
이 기능은 엔터프라이즈 보안, 컴플라이언스 요구 사항, 그리고 사고 조사에 필수적입니다.

## 감사 로그란 무엇인가요?

감사 로그는 조직 및 프로젝트 내 모든 주요 활동에 대한 불변 기록입니다.
감사 로그는 다음을 캡처합니다:

- **누가(Who)**: 작업을 수행한 사용자 또는 API 키
- **무엇을(What)**: 수행된 구체적인 작업(생성, 업데이트, 삭제)
- **언제(When)**: 작업이 발생한 정확한 타임스탬프
- **어디서(Where)**: 조직 및 프로젝트 컨텍스트
- **세부 정보(Details)**: 변경 사항에 대한 변경 전/후 전체 상태

이러한 로그는 보안 모니터링, 컴플라이언스 보고, 포렌식 분석을 위한 완전한 감사 추적을 제공합니다.

### 접근 제어

- `auditLogs:read` 권한을 가진 사용자만 이용할 수 있습니다
- 일반적으로 프로젝트 OWNER 및 ADMIN이 접근할 수 있습니다
- Langfuse의 역할 기반 접근 제어(RBAC) 시스템을 통해 제어됩니다

### 필터링 및 탐색

- **시간 기반 필터링**: 특정 기간의 로그를 확인할 수 있습니다
- **프로젝트 필터링**: 특정 프로젝트의 활동에 집중할 수 있습니다
- **페이지네이션**: 방대한 감사 추적을 효율적으로 탐색할 수 있습니다

## 감사 로그 내보내기

UI에서 직접 감사 로그를 내보낼 수 있습니다. 이를 통해 데이터를 외부에서 분석하거나 중요한 정보의 백업을 생성할 수 있습니다.

**사용 방법:**

- 감사 로그 테이블로 이동합니다
- 우측 상단의 내보내기 버튼을 클릭합니다

### 상태 캡처

업데이트 작업의 경우 Langfuse는 다음을 캡처합니다:

- **변경 전 상태(Before State)**: 수정 이전의 리소스 전체 상태
- **변경 후 상태(After State)**: 수정 이후의 리소스 전체 상태

상태는 JSON 형태로 저장되어, 데이터에 가해진 모든 수정 사항에 대한 완전한 컨텍스트를 제공합니다.

### 사용자 귀속

#### 신원 소스

감사 로그는 서로 다른 유형의 행위자를 구분합니다:

- **사용자 작업**(`USER` 유형): 웹 인터페이스를 통해 인증된 사용자가 수행한 작업
- **API 키 작업**(`API_KEY` 유형): API 키를 통한 프로그래밍 방식의 작업

#### 컨텍스트 정보

각 항목에는 다음이 포함됩니다:

- **사용자 ID**: 사용자가 시작한 작업의 경우
- **API 키 ID**: 프로그래밍 방식의 작업의 경우
- **조직 ID**: 조직 컨텍스트
- **역할 컨텍스트**: 작업 시점의 사용자의 조직 및 프로젝트 역할
