# 데이터 삭제

개발 흐름 중 잘못 생성된 트레이스, PII에 해당하는 사용자 데이터, 또는 프로젝트 전체 등 Langfuse에서 선택한 데이터를 제거해야 하는 경우가 있을 수 있습니다.
최근 데이터만 유지하고자 하는 경우에는 [데이터 보존](/docs/data-retention) 기능을 사용할 수 있습니다.

다음과 같은 방법으로 Langfuse에서 원치 않는 데이터를 삭제할 수 있습니다:

- 단일 트레이스 삭제
- 여러 트레이스를 일괄 삭제
- 쿼리 필터와 일치하는 모든 트레이스 삭제
- 프로젝트 삭제
- 조직 삭제
- 사용자 계정 삭제

아래에서 각 옵션과 그 보장 사항에 대해 살펴보겠습니다.

## 트레이스 삭제

트레이스를 삭제하면 모든 데이터 저장소에서 점수 및 관측값(observation)과 같은 관련 엔티티도 함께 삭제된다는 점에 유의하세요.

### 단일 트레이스

<Tab>

단일 트레이스를 삭제하려면 해당 트레이스의 상세 보기를 열고 `Delete` 버튼을 클릭하세요.
해당 트레이스를 삭제할 것인지 확인합니다.

</Tab>

<Tab>

DELETE /api/public/traces/{traceId}​
```

[레퍼런스](https://api.reference.langfuse.com/#tag/trace/DELETE/api/public/traces/%7BtraceId%7D)를 참고하세요.

</Tab>

</LangTabs>

<LangTabs items={["Langfuse UI", "API"]}>

<Tab>

여러 트레이스를 일괄 삭제하려면 트레이스 목록에서 해당 트레이스들을 선택한 후 `Actions` 드롭다운에서 `Delete`를 선택하세요.

DELETE /api/public/traces
```

</Tab>

</LangTabs>

### 쿼리로 삭제

<LangTabs items={["Langfuse UI"]}>

<Tab>

쿼리 필터와 일치하는 모든 트레이스를 삭제하려면 트레이스 목록에서 원하는 필터를 설정하세요.
현재 페이지의 항목만 선택된 상태에서 상단 바를 통해 전체 항목 선택으로 변경합니다.
그런 다음 `Actions` 드롭다운에서 `Delete`를 선택합니다.

</Frame>

</Tab>

### 제한 사항

Langfuse의 대부분의 삭제 작업은 즉시 처리되지만, 트레이싱 데이터의 삭제는 그렇지 않습니다.
데이터 웨어하우스에서 해당 레코드를 제거하는 작업은 리소스를 많이 소모하는 작업이므로, 특정 시점에 처리할 수 있는 삭제 건수를 제한(rate limit)하고 있습니다.
일반적으로 트레이스 데이터는 삭제 호출 후 15분 이내에 시스템에서 삭제되며 별도의 완료 확인은 제공되지 않습니다. 즉, 데이터가 실제로 삭제되었는지 확인하려면 다시 조회해야 합니다.

오래된 데이터를 정기적으로 정리해야 하는 경우, 지정한 일수보다 오래된 트레이스, 관측값, 점수, 미디어 자산을 자동으로 삭제해 주는 [데이터 보존](/docs/administration/data-retention) 기능을 사용하는 것을 고려해보세요.

## 프로젝트 삭제

<LangTabs items={["Langfuse UI"]}>

<Tab>

프로젝트를 삭제하려면 프로젝트 설정으로 이동한 후 `General` 섹션의 `Danger Zone`으로 스크롤하세요.
프로젝트를 삭제할 것인지 확인합니다.
이 작업을 수행하면 모든 API 키가 즉시 폐기되고 프로젝트 삭제가 예약됩니다.
이후 몇 분 이내에 관련된 모든 데이터가 시스템에서 되돌릴 수 없게 제거됩니다.

<Callout type="warning">
  프로젝트 삭제는 되돌릴 수 없으며 모든 데이터가 제거됩니다. 이 작업을 수행할
  때는 신중해야 합니다. 삭제를 확인한 후 프로젝트가 실제로 삭제되기까지
  최대 5분이 소요될 수 있습니다.
</Callout>

</Tab>

</LangTabs>

## 조직 삭제

<LangTabs items={["Langfuse UI"]}>

<Tab>

조직 내에 남아 있는 프로젝트가 없는 경우, 조직 설정에서 조직을 삭제할 수 있습니다.
조직 설정으로 이동한 후 `General` 섹션의 `Danger Zone`으로 스크롤하세요.
조직을 삭제할 것인지 확인합니다.
조직 및 관련된 모든 사용자 정보가 시스템에서 제거됩니다.

</Tab>

</LangTabs>

## 사용자 계정 삭제 (Cloud)

<LangTabs items={["Langfuse UI"]}>

<Tab>

사용자는 계정 설정 페이지에서 자신의 계정을 삭제할 수 있습니다. 우측 하단의 사용자 메뉴에서 계정 설정으로 이동하세요.

만약 여러분이 어떤 조직의 유일한 소유자(sole owner)인 경우, 계정을 삭제하기 전에 먼저 소유권을 다른 사용자에게 이전하거나 해당 조직을 삭제해야 합니다.

<Frame className="my-10">
  ![사용자 계정 삭제](/images/docs/delete-account-settings.png)
</Frame>

</Tab>

</LangTabs>

## 사용자 계정 삭제 (셀프 호스팅)

`users` 테이블에서 해당 사용자 레코드를 제거하고, cascade를 사용하여 이를 참조하는 모든 외래 키(foreign key)를 함께 삭제하세요.
