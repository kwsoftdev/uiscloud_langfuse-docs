# Langfuse 로드맵

Langfuse는 [오픈 소스](/open-source)이며, 저희는 현재 무엇을 작업하고 있고 다음에 무엇을 할 것인지 완전히 투명하게 공개하고자 합니다. 이 로드맵은 살아있는 문서이며 진행 상황에 따라 계속 업데이트됩니다. 아래에서 [로드맵](#roadmap), [제품 비전](#vision), [최근 릴리스](#releases)에 대한 자세한 내용을 확인하세요.

  무엇인지 이해하는 데 큰 도움이 됩니다.
</Callout>

## 로드맵 [#roadmap]

현재의 초점은 기존 기반을 훌륭하게 다듬고, 에이전트를 위한 지속적인 개선 루프로 각 요소들을 연결하는 것입니다.

### 에이전트 관측성 및 뷰

- v4 관측(observation) 테이블, 필터 사이드바, 저장된 뷰, 기본 뷰를 에이전트 트레이스에 적합하도록 훌륭하게 만듭니다.
- 에이전트별 트레이스, 비용, 지연 시간, 단계, 도구 호출, 그리고 집계된 단계/도구 동작을 위한 에이전트 수준의 뷰를 구축합니다.
- 장시간 실행되는 에이전트 트레이스를 위한 트레이스 상세 페이지를 개선합니다. 여기에는 압축된 표현 방식, 선택된 JSON 경로, 차트에서 하위 스팬으로 이동하는 더 나은 방법이 포함됩니다.
- 전체 텍스트 검색, 메타데이터 필터링, 커스텀 차원, 그리고 대시보드에서 트레이스로 이어지는 워크플로우를 개선하여 팀이 더 적은 노이즈로 관측 데이터를 세분화할 수 있도록 합니다.

### 평가 및 실험

- 실험과 평가기를 위한 공개 API를 출시합니다.
- 평가기 데이터 모델을 확장하고 새로운 평가기 유형을 지원합니다.
- 실험 차트, 비교 흐름, 평가기 관리, 평가기 템플릿 라이브러리를 개선합니다.
- 코드 평가기, 범주형 및 불리언 판정기(judge), 자유 텍스트 점수, 멀티모달 데이터셋, 그리고 트레이스 수준 평가의 폐기(deprecation) 경로를 확장합니다.

### 워크플로우 자동화 및 에이전트

- Langfuse 데이터를 읽고, 화면 컨텍스트를 활용하며, 트레이스 비교와 같은 작업을 지원하는 최초의 제품 내장 Langfuse 에이전트를 구축합니다.
- 스킬, 가이드, 아카데미 콘텐츠를 활용해 제품 외부에서 AI 엔지니어링 워크플로우를 자동화한 뒤, 가장 성과가 좋은 것들을 제품 내에 패키징합니다.
- Langfuse CLI, MCP 인터페이스, 스킬 관리 기능을 개선하여 외부 에이전트가 데이터 구조를 파악하고, Langfuse를 효율적으로 쿼리하며, 일반적인 워크플로우를 실행할 수 있도록 합니다.
- 낮은 점수 분석, 실패 클러스터링, 평가기 설정, 프로덕션에서 데이터셋으로의 갱신, 합성 데이터 생성, 실험 트리거와 같은 반복 가능한 워크플로우를 우선적으로 다룹니다.

### 플랫폼 안정성 및 확장성

- Langfuse Cloud와 셀프 호스팅 배포 전반에 걸쳐 v4 롤아웃을 완료합니다.
- 대규모 에이전트 워크로드를 위한 수집(ingestion) 확장을 계속하고, 필요한 경우 사전 집계를 통해 읽기 경로를 더 빠르게 만듭니다.
- 블롭 내보내기, S3 내보내기, 공개 API, 메트릭, 관측 데이터 접근, CLI와 같은 시스템 통합 지점을 지루할 정도로 안정적으로 만듭니다.

### 알림, 워크플로우, 엔터프라이즈 제어

- Slack, PagerDuty, 웹훅, 이메일 등 다양한 전달 채널을 통해 평가, 메트릭, 운영 임계값에 대한 알림을 출시합니다.
- 관측성 및 평가 이벤트를 위한 웹훅과 자동화를 탐색합니다.
- API 키 범위 지정을 개선하고, 베어러 키 방식으로 전환하며, 엔터프라이즈 배포를 위한 관리자 제어 기능을 확장합니다.
- 셀프 호스팅 및 Helm 차트 경험을 개선하고, ClickHouse Operator를 활용하며, 더 강력한 데이터 격리나 직접적인 ClickHouse 접근이 필요한 고객을 위한 하이브리드 또는 BYOC 배포 모델을 탐색합니다.

### 멀티모달 및 플레이그라운드

- 트레이스, 플레이그라운드, 데이터셋, 평가가 하나의 일관된 시스템처럼 느껴지도록 멀티모달 격차를 해소합니다.

## 제품 비전과 방향성 [#vision]

Langfuse는 사람, 그리고 궁극적으로는 에이전트가 에이전트를 개선하는 데 도움을 주는 개방형 데이터 및 평가 레이어가 되어야 합니다. 저희는 무엇보다 하나의 [제품 루프](/academy)에 집중합니다: 에이전틱 시스템을 추적, 이해, 평가, 개선하는 것입니다.

전략적으로 Langfuse는 실행 레이어에서 중립을 유지하는 선택을 합니다. Langfuse는 특정한 견해를 가진 에이전트 프레임워크나 런타임이 되어서는 안 됩니다. 대신 Langfuse는 에이전틱 소프트웨어를 둘러싼 개선 루프를 소유해야 합니다: 에이전트 동작을 이해하고, 유용한 뷰로 세분화하며, 프로덕션 실패를 데이터셋으로 전환하고, 실험을 실행하며, API, CLI, 스킬, 제품 내장 에이전트를 통해 반복되는 워크플로우를 자동화하는 것입니다.

장기적인 방향은 자가 최적화 에이전트입니다: 트레이싱과 코드 저장소를 연결하면, Langfuse가 여러분을 대신해 에이전트 개선 루프를 관리할 수 있습니다. Langfuse는 여러분의 시스템을 정의하는 지침, 프롬프트, 평가, 스킬 파일을 이해하고, 버전을 관리하며, 평가를 실행하고, 실험을 제안하거나 트리거하며, 가장 중요한 판단이 필요한 순간에는 사람을 계속 참여시킵니다.

### 플랫폼의 기본 단위로서의 뷰

뷰는 관측 데이터를 유용한 제품 표면으로 세분화하는 기본 단위가 되어야 합니다. 뷰는 어떤 관측 데이터가 중요한지, 어떻게 그룹화되는지, 어떤 속성이 표시되는지, 어떤 메트릭과 점수가 중요한지, 그리고 어떤 후속 작업이 가능한지를 정의합니다. 이를 통해 에이전트 개요 대시보드, 기본 템플릿, 시맨틱 클러스터링, 평가 분포 비교, 워크플로우 트리거가 가능해집니다.

### 선호도 레이어

사람의 판단은 여전히 에이전트를 평가하는 데 있어 궁극적인 기준입니다. Langfuse는 명시적인 피드백을 더 쉽게 수집하고, 암묵적인 신호를 도출하며, LLM-as-a-judge 평가기를 사람의 선호도에 맞게 정렬하고, 확신도가 낮은 사례를 다시 사람의 검토로 돌려보내는 데 도움을 주어야 합니다.

### 시맨틱 그룹화

에이전트가 라우팅 기반의 하위 에이전트 시스템에서 더 광범위하고 동적인 에이전트로 발전함에 따라, 고정된 라벨만으로는 충분하지 않습니다. Langfuse는 팀이 필터링된 뷰 내에서 의미 있는 상호작용 그룹을 발견하고, 이러한 그룹 간 점수를 비교하며, 반복되는 실패를 데이터셋이나 실험으로 전환할 수 있도록 도와야 합니다.

### 개선을 이끄는 실험 표면

실험은 프롬프트, 모델, 런타임 변경 사항을 비교하는 대표적인 워크플로우가 되어야 합니다. Langfuse는 베이스라인, 비교 실행, 주석, 메트릭, 다음 작업을 충분히 쉽게 만들어서 팀이 자연스럽게 실험을 에이전트 개선 루프로 활용하도록 해야 합니다.

### 관리형 개선 루프

최종적인 목표는 Langfuse가 에이전트 시스템을 모니터링하고, 실험을 제안하거나 실행하며, 프로덕션에서 테스트 세트를 갱신하고, 사람의 개입이 필요할 때 주석 작업을 할당하며, 시스템이 시간에 따라 어떻게 개선되고 있는지 보고할 수 있게 되는 것입니다.

## 🚀 최근 출시된 기능 [#releases]

import { changelogSource } from "@/lib/source";
import { Link } from "@/components/ui/link";

export const ChangelogList = () => (
  <ul className="list-disc list pl-6">
    {changelogSource
      .getPages()
      .sort(
        (a, b) =>
          new Date(String(b.data.date || 0)).getTime() -
          new Date(String(a.data.date || 0)).getTime(),
      )
      .slice(0, 10)
      .map((page) => (
        <li
          className="my-2"
          id={page.url.replace("/changelog/", "")}
          key={page.url.replace("/changelog/", "")}
        >
          <Link href={page.url} variant="underline">
            <span>{page.data.title}</span>
          </Link>
          <span className="text-primary/60 ml-2 hidden sm:inline">
            {page.data.date
              ? `(${new Date(String(page.data.date)).toLocaleDateString(
                  "en-US",
                  {
                    year: "numeric",
                    month: "short",
                    day: "numeric",
                    timeZone: "UTC",
                  },
                )})`
              : null}
          </span>
        </li>
      ))}
  </ul>
);
