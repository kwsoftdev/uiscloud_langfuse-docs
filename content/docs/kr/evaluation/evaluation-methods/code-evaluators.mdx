코드 평가기는 Langfuse에서 사용자 정의 Python 또는 TypeScript 로직을 실행하여 하나 이상의 [점수](/docs/evaluation/scores/overview)를 반환합니다. 모델 기반 판단보다 코드가 더 신뢰할 수 있는 결정론적이고 객관적인 검사에 사용하세요.

일반적인 예로는 정확 일치 검사, 정규식 검증, JSON 파싱 가능 여부, 스키마 검증, 키워드 검사, 도구 호출(tool-call) 검사, 사용자 정의 비즈니스 규칙 등이 있습니다.

평가가 의미론적 판단, 루브릭 기반 추론, 또는 유용성·어조·답변 품질과 같은 주관적 평가를 필요로 할 때는 대신 [LLM-as-a-Judge](/docs/evaluation/evaluation-methods/llm-as-a-judge)를 사용하세요.

## 코드 평가기 사용 방법

코드 평가기는 두 가지 유형의 데이터에 대해 실행할 수 있습니다: **관측값(Observations)** (실제 프로덕션 트래픽에서 발생하는 개별 작업) 또는 **실험(Experiments)** (통제된 테스트 데이터셋). 어떤 것을 선택할지는 개발 단계에서 테스트하는지 아니면 프로덕션을
모니터링하는지에 따라 달라집니다.

### 각 평가 대상 이해하기

<Tabs items={["실시간 프로덕션 데이터", "오프라인 실험 데이터"]}>
<Tab>

트레이스 내의 개별 관측값, 예를 들어 LLM 호출, 검색 작업, 임베딩 생성, 또는 도구 호출에 대해 평가기를 실행합니다.

**관측값을 대상으로 삼는 이유**

- **작업 단위의 정밀도**: 관측값 유형별로 필터링하여 전체 트레이스가 아닌 중요한 작업만 평가합니다.
- **결정론적 프로덕션 모니터링**: 실시간 트래픽에 대해 JSON 유효성, 스키마 준수 여부, 정확 일치, 또는 비즈니스 규칙을 검사합니다.
- **조합형 평가**: 하나의 트레이스 내에서 서로 다른 작업에 대해 서로 다른 코드 평가기를 실행합니다.
- **결합 필터링**: 관측값 필터를 `userId`, `sessionId`, 태그, 버전, 메타데이터와 같은 트레이스 필터와 함께 적용합니다.

**데이터 흐름**

수집 시점에 각 관측값은 설정한 필터 조건에 따라 평가됩니다. 조건에 일치하는 관측값은 평가 큐에 추가됩니다. 평가 작업은 비동기로 처리되며, 점수는 해당 관측값에 첨부됩니다.

**예시 사용 사례**

- 최종 LLM 응답이 파싱 가능한 JSON인지 검증
- 도구 호출에 필수 인자가 포함되어 있는지 확인
- 선택한 모델 호출에 대해 사용자 정의 비즈니스 규칙을 적용

</Tab>

<Tab>

재현 가능한 환경에서 모델 버전, 프롬프트 변형, 시스템 구성을 비교하기 위해 통제된 테스트 데이터셋에 대해 평가기를 실행합니다.

**실험을 대상으로 삼는 이유**

- 개발 워크플로우를 위한 결정론적 pass/fail 검사가 필요한 경우
- 여러 프롬프트 버전 또는 모델 구성을 비교하고 싶은 경우
- 평가기가 검사해야 할 예상 출력이나 메타데이터가 포함된 데이터셋이 있는 경우

**데이터 흐름**

각 실험 실행은 선택한 평가기로 점수를 매길 수 있는 트레이스와 관측값을 생성합니다. 평가기는 관측값 데이터와 함께 예상 출력, 항목 메타데이터와 같은 실험 항목 컨텍스트를 받습니다.

1. 테스트 입력과, 선택적으로 예상 출력을 포함하는 데이터셋을 생성합니다.
2. UI 또는 SDK를 통해 실험을 실행합니다. [UI로 실험 실행](/docs/evaluation/experiments/experiments-via-ui) 또는 [SDK로 실험 실행](/docs/evaluation/experiments/experiments-via-sdk)을 참고하세요.
3. 생성된 관측값에 점수를 매길 코드 평가기를 선택합니다.
4. 실험 실행 간 결과를 비교하여 데이터 기반 의사 결정을 내립니다.

**예시 사용 사례**

- 지원 질문 데이터셋에서 두 가지 프롬프트 버전을 비교하고 각 응답에 필수 JSON 필드가 포함되어 있는지 확인

## 함수 계약 [#function-contract]

각 평가기는 `evaluate` 함수를 노출합니다. Langfuse는 `EvaluationContext`를 전달하고, 하나 이상의 점수를 담은 `EvaluationResult`를 기대합니다.

<LangTabs items={["Python", "TypeScript"]}>
<Tab>

```python
from dataclasses import dataclass, field
from typing import Any


@dataclass
class ToolCall:
    id: str = ""
    name: str = ""
    arguments: Any = None
    type: str = ""
    index: int = 0


@dataclass
class ObservationContext:
    input: Any = None
    output: Any = None
    metadata: Any = None
    tool_calls: list[ToolCall] = field(default_factory=list)


@dataclass
class ExperimentContext:
    item_expected_output: Any = None
    item_metadata: Any = None


@dataclass
class EvaluationContext:
    observation: ObservationContext
    experiment: ExperimentContext | None = None


@dataclass
class Score:
    name: str
    value: int | float | str | bool
    data_type: str
    comment: str | None = None
    config_id: str | None = None
    metadata: dict[str, Any] | None = None


@dataclass
class EvaluationResult:
    scores: list[Score]


def evaluate(ctx: EvaluationContext) -> EvaluationResult:
    output_present = ctx.observation.output is not None

    return EvaluationResult(
        scores=[
            Score(
                name="Output present",
                value=output_present,
                data_type="BOOLEAN",
                comment=(
                    "Observation output is present."
                    if output_present
                    else "Observation output is missing."
                ),
                metadata={"rule": "output_present"},
            )
        ]
    )
```

</Tab>

type ToolCall = {
  id: string;
  name: string;
  arguments: unknown;
  type: string;
  index: number;
};

type EvaluationContext = {
  observation: {
    input: any;
    output: any;
    metadata: any;
    toolCalls: ToolCall[];
  };
  experiment:
    | {
        itemExpectedOutput: any;
        itemMetadata: any;
      }
    | undefined;
};

type ScoreBase = {
  name: string;
  comment?: string;
  configId?: string | null;
  metadata?: Record<string, unknown>;
};

type NumericScore = ScoreBase & {
  dataType: "NUMERIC";
  value: number;
};

type BooleanScore = ScoreBase & {
  dataType: "BOOLEAN";
  value: boolean;
};

type CategoricalScore = ScoreBase & {
  dataType: "CATEGORICAL";
  value: string;
};

type TextScore = ScoreBase & {
  dataType: "TEXT";
  value: string;
};

type Score = NumericScore | BooleanScore | CategoricalScore | TextScore;

type EvaluationResult = {
  scores: Score[];
};

function evaluate({
  observation: { input, output, metadata, toolCalls },
  experiment,
}: EvaluationContext): EvaluationResult {
  const itemExpectedOutput = experiment?.itemExpectedOutput;
  const itemMetadata = experiment?.itemMetadata;
  const outputPresent = output != null;

  return {
    scores: [
      {
        name: "Output present",
        value: outputPresent,
        dataType: "BOOLEAN",
        comment: outputPresent
          ? "Observation output is present."
          : "Observation output is missing.",
        metadata: {
          rule: "output_present",
          hasInput: input != null,
          hasObservationMetadata: metadata != null,
          toolCallCount: toolCalls.length,
          hasExpectedOutput: itemExpectedOutput != null,
          hasExperimentMetadata: itemMetadata != null,
        },
      },
    ],
  };
}
```

### 점수 필드

| 필드                     | 설명                                                                                                                                                 |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`                   | 필수 항목인 점수 이름입니다.                                                                                                                         |
| `value`                  | 필수 항목인 점수 값입니다.                                                                                                                           |
| `data_type` / `dataType` | 필수 항목인 점수 데이터 타입입니다. 지원되는 값은 [`NUMERIC`, `CATEGORICAL`, `BOOLEAN`, `TEXT`](/docs/evaluation/scores/overview#score-types)입니다. |
| `comment`                | 점수와 함께 저장되는 선택적 근거 또는 설명입니다.                                                                                                    |
| `config_id` / `configId` | 선택적 점수 설정(config) ID입니다. 제공된 경우 점수는 참조된 [점수 설정](/faq/all/manage-score-configs)을 충족해야 합니다.                           |
| `metadata`               | 점수와 함께 저장되는 선택적 메타데이터입니다.                                                                                                        |

이 예시는 관측값 출력이 실험 항목의 예상 출력과 정확히 일치할 때 통과하는 불리언 점수를 반환합니다.

def evaluate(ctx: EvaluationContext) -> EvaluationResult:
    """Evaluates one observation and returns one or more Langfuse scores."""
    expected_output = (
        ctx.experiment.item_expected_output if ctx.experiment is not None else None
    )
    matches_expected_output = (
        expected_output is not None and ctx.observation.output == expected_output
    )

    return EvaluationResult(
        scores=[
            Score(
                name="Exact match",
                value=matches_expected_output,
                data_type="BOOLEAN",
                comment=(
                    "Output exactly matches the expected output."
                    if matches_expected_output
                    else "Output does not match the expected output."
                ),
            )
        ]
    )

/**
 * Evaluates one observation and returns one or more Langfuse scores.
 */
function evaluate({
  observation: { input, output, metadata },
  experiment,
}: EvaluationContext): EvaluationResult {
  const itemExpectedOutput = experiment?.itemExpectedOutput;
  const itemMetadata = experiment?.itemMetadata;
  const matchesExpectedOutput =
    itemExpectedOutput != null && output === itemExpectedOutput;

  return {
    scores: [
      {
        name: "Exact match",
        value: matchesExpectedOutput,
        dataType: "BOOLEAN",
        comment: matchesExpectedOutput
          ? "Output exactly matches the expected output."
          : "Output does not match the expected output.",
        metadata: {
          hasInput: input != null,
          hasObservationMetadata: metadata != null,
          hasExperimentMetadata: itemMetadata != null,
        },
      },
    ],
  };
}
```

</LangTabs>

## 코드 평가기 실행 디버깅 [#debug-code-evaluator-executions]

모든 코드 평가기 실행은 트레이스를 생성하므로 평가 과정을 완전히 파악할 수 있습니다. 이를 통해 선택된 입력과 출력, 실험 컨텍스트, 런타임 지연 시간, 반환된 점수, 로그, 오류를 확인할 수 있습니다.

트레이싱 테이블에서 환경(environment)을 `langfuse-code-eval`로 필터링하여 코드 평가기 실행 트레이스를 표시할 수 있습니다.

<Frame fullWidth>
  ![코드 평가기 실행으로 필터링된 트레이싱
  테이블](/images/docs/code-evaluators/debugging.png)
</Frame>

<Details>
<Summary>코드 평가기 실행 상태</Summary>

- **Completed(완료)**: 평가가 성공적으로 끝나고 유효한 점수를 반환했습니다.
- **Error(오류)**: 평가가 실패했습니다(입력, 출력, 지연 시간, 로그, 오류 세부 정보를 보려면 실행 트레이스 ID를 클릭하세요).
- **Pending(대기)**: 평가가 큐에 있으며 실행을 기다리고 있습니다.

</Details>

새 평가기를 활성화하기 전에 평가기 테스트 실행을 사용하세요. 선택된 관측값 데이터, 실험 컨텍스트, 점수 이름, 점수 값, 점수 데이터 타입을 검증하는 가장 빠른 방법입니다.

## 런타임 제약 조건 [#runtime-constraints]

코드 평가기는 많은 관측값에 대해 빠르고 안전하게 실행될 수 있는 간결하고 결정론적인 검사를 위한 것입니다.

<Callout type="info">
  코드 평가기에 특정 서드파티 라이브러리나 네트워크 액세스가 필요하신가요?
  [GitHub Discussions](https://github.com/orgs/langfuse/discussions)에서
  사용 사례를 공유해 주세요. 여러분의 피드백은 더 광범위한 런타임 지원이 유용한
  지점을 파악하는 데 도움이 됩니다.
</Callout>

| 제약 조건       | 제한 사항 / 안내                                                                                                                                                                                                                                                      |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 언어            | 평가기는 Python 또는 TypeScript로 작성하세요. 셀프 호스팅 배포에서는 Python은 `aws-lambda` 디스패처가 필요하며, `insecure-local`은 TypeScript/JavaScript만 지원합니다.                                                                                                |
| TypeScript 문법 | [erasable TypeScript syntax](https://www.typescriptlang.org/docs/handbook/release-notes/typescript-5-8.html#the---erasablesyntaxonly-option)를 사용하세요. 타입 어노테이션과 인터페이스는 사용 가능하지만, enum, namespace, 데코레이터, 파라미터 프로퍼티는 피하세요. |
| 종속성          | 언어의 표준 라이브러리([Python](https://docs.python.org/3/library/index.html) 및 [TS/JS](https://nodejs.org/download/release/latest-v24.x/docs/api/all.html))를 사용하세요. 서드파티 패키지는 평가기 런타임에서 사용할 수 없습니다.                                   |
| 네트워크 액세스 | 평가기는 네트워크 아웃바운드 없이 실행됩니다. 필요한 모든 데이터는 관측값 또는 실험 컨텍스트 안에 두세요.                                                                                                                                                             |
| 런타임 제한     | 평가기는 2초 이내에 완료되어야 합니다.                                                                                                                                                                                                                                |
| 결과 형태       | `evaluate`에서 최소 하나 이상의 점수를 반환해야 합니다.                                                                                                                                                                                                               |
| 소스 크기       | 평가기 소스 코드는 256KB 미만으로 유지하세요.                                                                                                                                                                                                                         |
| 입력 크기       | 소스 코드와 선택된 변수를 포함한 디스패치 페이로드는 5.5MB 미만으로 유지하세요.                                                                                                                                                                                       |
| 결과 크기       | 평가기 결과는 256KB 미만으로 유지하세요.                                                                                                                                                                                                                              |

## FAQ

<Details>
<Summary>타임아웃 오류는 어떻게 디버깅하나요?</Summary>

타임아웃은 대개 평가기가 2초 런타임 제한에 비해 너무 많은 작업을 수행하거나 네트워크에 접근을 시도하고 있음을 의미합니다. 네트워크 요청은 런타임에 의해 차단되며 타임아웃 오류로 나타날 수 있습니다.

이를 디버깅하려면 작은 샘플 관측값에 대해 평가기를 실행하고, 네트워크 호출을 제거하고, 큰 반복문이나 비용이 큰 파싱을 피하고, 평가기에서 선택하는 입력, 출력, 메타데이터, 실험 컨텍스트의 양을 줄여보세요.

</Details>

<Details>
<Summary>서드파티 패키지를 사용할 수 있나요?</Summary>

사용할 수 없습니다. 코드 평가기는 현재 표준 라이브러리만 지원합니다. 평가에 서드파티 패키지가 필요하다면, 해당 로직은 자체 인프라에서 실행하고 결과는 [SDK를 통한 점수 전송](/docs/evaluation/evaluation-methods/scores-via-sdk)으로 수집하세요.

</Details>

<Details>
<Summary>실험 컨텍스트가 존재하지 않을 때가 있는 이유는 무엇인가요?</Summary>

`ctx.experiment`는 평가기가 실험에 대해 실행될 때만 존재합니다. 실시간 관측값 평가기의 경우, Python에서는 `ctx.experiment`가 `None`일 수 있고 TypeScript에서는 `undefined`일 수 있음을 처리하도록 코드를 작성하세요.

</Details>

<Details>
<Summary>API나 SDK로 코드 평가기를 생성할 수 있나요?</Summary>

가능합니다. Langfuse UI 외에도, unstable 공개 평가기 엔드포인트는 `type: "code"`를 받아 코드 평가기를 생성하고 이를 평가 규칙에서 참조할 수 있게 합니다. [Evaluators API 레퍼런스](https://api.reference.langfuse.com/#tag/unstableevaluators)를 참고하세요. 이 엔드포인트는 unstable 상태이며 변경될 수 있습니다.

자체 애플리케이션이나 CI 파이프라인에서 결정론적 평가 로직을 실행하고 싶다면, [SDK를 통한 점수 전송](/docs/evaluation/evaluation-methods/scores-via-sdk)을 사용하여 결과 점수를 Langfuse로 수집하세요.

</Details>

<Details>
<Summary>코드 평가기 실행 트레이스를 찾을 수 없는 이유는 무엇인가요?</Summary>

코드 평가기 실행은 내부 환경(environment)인 `langfuse-code-eval`을 사용합니다. 내부 환경은 기본 트레이싱 뷰에서 숨겨지므로, 트레이싱 테이블을 `environment = langfuse-code-eval`로 필터링하거나 관련 점수 또는 평가기 로그에서 실행 트레이스를 여세요.

</Details>

<Details>
<Summary>셀프 호스팅 Langfuse에서 코드 평가기는 어떻게 구성하나요?</Summary>

셀프 호스팅 배포에서는 [코드 평가기](/self-hosting/configuration/code-evaluators)에서 코드 평가기 디스패처와 실행 워커를 구성하세요.

유일한 SDK 요구 사항은 OpenTelemetry 기반 수집입니다.

- Python SDK v3 이상(OTel 기반). Python SDK v2를 사용 중이라면 [Python v2 → v3 마이그레이션 가이드](/docs/observability/sdk/upgrade-path/python-v2-to-v3)를 참고하세요.
- JS/TS SDK v4 이상(OTel 기반). JS/TS SDK v3를 사용 중이라면 [JS/TS v3 → v4 마이그레이션 가이드](/docs/observability/sdk/upgrade-path/js-v3-to-v4)를 참고하세요.

</Details>
