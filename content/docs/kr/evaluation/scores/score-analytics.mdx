# 점수 분석

점수 분석(Score Analytics)은 평가 데이터를 별도의 설정 없이 즉시 분석할 수 있는 경량 기능을 제공합니다. 서로 다른 LLM 평가기(judge)가 일관된 결과를 내는지 검증하거나, 사람이 작성한 주석(annotation)이 자동 평가와 일치하는지 확인하거나, 점수 분포와 추세를 탐색하는 등 어떤 목적이든 점수 분석은 평가 프로세스에 대한 확신을 구축하는 데 도움을 줍니다.

## 점수 분석을 사용하는 이유

점수 분석은 Langfuse의 [실험 SDK](/docs/evaluation/overview) 및 [셀프 서브 대시보드](/docs/metrics/features/custom-dashboards)를 보완하여, 즉각적이고 설정이 필요 없는 점수 분석을 제공합니다.

- **경량 설정**: 별도 설정 없이 점수가 수집되는 즉시 분석을 시작할 수 있습니다
- **빠른 검증**: 서로 다른 소스(예: 평가기로서의 GPT-4와 Gemini)의 점수를 비교하여 일치도를 측정하고 신뢰성을 확인합니다
- **즉시 사용 가능한 인사이트**: 별도의 대시보드 설정 없이 분포를 시각화하고, 추세를 추적하며, 상관관계를 발견합니다
- **통계적 엄밀성**: Pearson 상관계수, Cohen's Kappa, F1 점수와 같은 메트릭을 내장된 해석과 함께 확인할 수 있습니다

커스텀 메트릭이나 복잡한 비교가 필요한 고급 분석의 경우, 더 깊이 있는 조사를 위해 [실험 SDK](/docs/evaluation/overview)를 사용하세요.

## 시작하기

<Steps>

### 사전 준비 사항

Langfuse 프로젝트에 다음과 같은 평가 방법으로 생성된 [점수 데이터](/docs/evaluation/overview)가 있는지 확인하세요.

- [사람에 의한 주석](/docs/evaluation/evaluation-methods/annotation)
- [LLM-as-a-Judge 평가](/docs/evaluation/evaluation-methods/llm-as-a-judge)
- [코드 평가기](/docs/evaluation/evaluation-methods/code-evaluators)
- [SDK](/docs/evaluation/evaluation-methods/scores-via-sdk) 또는 [API](/docs/api)를 통해 수집된 커스텀 점수

### 점수 분석으로 이동

1. Langfuse에서 프로젝트로 **이동합니다**
2. 내비게이션 메뉴에서 `Scores`를 **클릭합니다**
3. `Analytics` 탭을 **선택합니다**

### 단일 점수 분석

1. 첫 번째 드롭다운 메뉴에서 **점수를 선택합니다**
2. 분석할 객체 유형(트레이스, 관측(Observations), 세션, 또는 데이터셋 실행 항목)을 **선택합니다**
3. 날짜 선택기를 사용해 **기간을 설정합니다**(예: 지난 90일)
4. 총 개수, 평균/최빈값, 표준편차를 보여주는 Statistics 카드를 **확인합니다**
5. 점수 값이 어떻게 분포되어 있는지 보여주는 Distribution 차트를 **살펴봅니다**
6. 시간적 패턴을 추적하는 Trend Over Time 차트를 **검토합니다**

<Frame fullWidth>
  ![단일 점수 분석](/images/docs/score-analytics-boolean-single.png)
</Frame>

### 두 점수 비교

1. 두 번째 드롭다운 메뉴에서 **두 번째 점수를 선택합니다**(동일한 데이터 유형이어야 합니다)
2. Statistics 카드에서 비교 메트릭을 **확인합니다**.
   - 일치 개수(동일한 상위 객체에 연결된 점수)
   - 상관관계 메트릭(Pearson, Spearman)
   - 오차 메트릭(숫자형 점수의 경우 MAE, RMSE)
   - 일치도 메트릭(범주형/불리언 점수의 경우 Cohen's Kappa, F1, 전체 일치도)
3. Score Comparison 히트맵을 **살펴봅니다**.
   - 강한 대각선 패턴은 높은 일치도를 나타냅니다
   - 역대각선 패턴은 음의 상관관계를 나타냅니다
   - 산발적인 패턴은 낮은 정합성을 시사합니다
4. matched 탭과 all 탭에서 분포를 **비교합니다**
5. 시간에 따라 두 점수가 함께 어떻게 변화하는지 **추적합니다**

  ![불리언 점수 비교](/images/docs/score-analytics-boolean-compare.png)
</Frame>

</Steps>

## 주요 기능

### 다중 데이터 유형 지원

점수 분석은 점수 데이터 유형에 따라 시각화와 메트릭을 자동으로 조정합니다.

<Callout type="info">
[텍스트 점수](/docs/evaluation/scores/overview#score-types)는 자유 형식 텍스트를 의미 있게 집계하거나 비교할 수 없기 때문에 점수 분석에서 지원되지 않습니다.
</Callout>

**숫자형 점수** (1~10 등급과 같은 연속 값)

- **분포**: 값 범위를 보여주는 10구간 히스토그램
- **비교**: 상관관계 패턴을 보여주는 10×10 히트맵
- **메트릭**: Pearson 상관계수, Spearman 상관계수, MAE(평균 절대 오차), RMSE(제곱근 평균 제곱 오차)

**범주형 점수** ("good/bad/neutral"과 같은 이산 범주)

- **분포**: 범주별 개수를 보여주는 막대 차트
- **비교**: 범주가 어떻게 일치하는지 보여주는 N×M 혼동 행렬(confusion matrix)
- **메트릭**: Cohen's Kappa, F1 점수, 전체 일치도

**불리언 점수** (true/false 이진 값)

- **분포**: 2개 범주로 구성된 막대 차트
- **비교**: 2×2 혼동 행렬
- **메트릭**: Cohen's Kappa, F1 점수, 전체 일치도

### Matched 데이터와 All 데이터 분석

점수 분석은 데이터를 이해하기 위한 두 가지 보기를 제공합니다.

**Matched 데이터** (기본 탭)

- 선택한 두 점수가 모두 연결된 상위 객체(트레이스, 관측, 세션, 또는 데이터셋 실행 항목)만 표시합니다
- 평가 방법 간의 유효한 비교를 가능하게 합니다
- 두 점수가 동일한 상위 객체와 연관될 때 일치(match)가 존재합니다
- 일치도와 상관관계를 측정할 때 이 보기를 사용하세요

**All 데이터** (개별 점수 탭)

- 각 점수의 전체 분포를 독립적으로 보여줍니다
- 평가 커버리지(각 점수를 가진 상위 객체 수)를 드러냅니다
- 평가 전략의 공백을 식별하는 데 도움이 됩니다

### 시간 기반 분석

Trend Over Time 차트는 다음과 같은 기능으로 점수 패턴을 모니터링하는 데 도움을 줍니다.

- **조정 가능한 간격**: 분 단위부터 연 단위까지(5m, 30m, 1h, 3h, 1d, 7d, 30d, 90d, 1y)
- **자동 간격 선택**: 선택한 기간에 따른 스마트 기본값
- **공백 채우기**: 일관된 시각화를 위해 누락된 기간은 0으로 채워집니다
- **평균 계산**: 부제목에 해당 기간의 전체 평균이 표시됩니다

### 통계 메트릭

점수 분석은 업계 표준 통계 메트릭을 해석 가이드와 함께 제공합니다.

**상관관계 메트릭** (숫자형 점수용)

<Callout type="info">
**Pearson 상관계수**: 점수 간의 선형 관계를 측정합니다. 값의 범위는 -1(완전한 음의 상관관계)부터 1(완전한 양의 상관관계)까지입니다.
- 0.9~1.0: 매우 강한 상관관계
- 0.7~0.9: 강한 상관관계
- 0.5~0.7: 중간 정도의 상관관계
- 0.5 미만: 약한 상관관계

**Spearman 상관계수**: 순위 기반의 단조 관계(monotonic relationship)를 측정합니다. Pearson보다 이상치(outlier)에 더 강건합니다.

</Callout>

**오차 메트릭** (숫자형 점수용)

<Callout type="info">
**MAE(평균 절대 오차)**: 점수 간 절대 차이의 평균입니다. 값이 낮을수록 좋습니다.

**RMSE(제곱근 평균 제곱 오차)**: 제곱 차이 평균의 제곱근입니다. MAE보다 큰 오차에 더 큰 페널티를 부여합니다.

</Callout>

**일치도 메트릭** (범주형/불리언 점수용)

<Callout type="info">
**Cohen's Kappa**: 우연에 의한 일치를 보정한 일치도를 측정합니다. 값의 범위는 -1부터 1까지입니다.
- 0.81~1.0: 거의 완벽한 일치
- 0.61~0.80: 상당한 일치
- 0.41~0.60: 중간 정도의 일치
- 0.41 미만: 약간의 일치

**F1 점수**: 정밀도(precision)와 재현율(recall)의 조화 평균입니다. 값의 범위는 0부터 1까지이며, 1이 완벽한 값입니다.

**전체 일치도**: 일치하는 분류의 단순 비율입니다. 우연에 의한 일치는 보정되지 않습니다.

</Callout>

## 사용 사례 예시

### LLM 평가기 신뢰성 검증 [#validate-llm-judge-reliability]

**시나리오**: GPT-4와 Gemini를 모두 사용하여 유용성(helpfulness)을 평가합니다. 두 평가기가 일관된 결과를 내고 있을까요?

**워크플로우**:

1. "helpfulness_gpt4-NUMERIC-EVAL"을 점수 1로 선택합니다
2. "helpfulness_gemini-NUMERIC-EVAL"을 점수 2로 선택합니다
3. Statistics 카드를 확인합니다. "Very Strong" 배지와 함께 Pearson 상관계수 0.984가 표시됩니다
4. 히트맵을 살펴봅니다. 강한 대각선 패턴이 정합성을 확인해 줍니다
5. **결과**: 두 평가기가 강하게 일치하며, 평가가 신뢰할 만합니다

### 사람과 AI 주석 간 일치도

**시나리오**: 품질에 대한 사람의 주석과 AI 평가가 모두 있습니다. AI를 신뢰해도 될까요?

**워크플로우**:

1. "quality-CATEGORICAL-ANNOTATION"을 점수 1로 선택합니다
2. "quality-CATEGORICAL-EVAL"을 점수 2로 선택합니다
3. 혼동 행렬을 확인합니다. 강한 대각선은 높은 일치도를 나타냅니다
4. Cohen's Kappa를 검토합니다. 0.85는 "거의 완벽한" 일치도를 보여줍니다
5. **결과**: AI 평가가 사람의 판단과 잘 일치합니다

### 음의 상관관계 식별

**시나리오**: 서로 다른 애플리케이션 동작 간의 관계를 이해합니다

**워크플로우**:

1. "has_tool_use-BOOLEAN-EVAL"을 점수 1로 선택합니다
2. "has_hallucination-BOOLEAN-EVAL"을 점수 2로 선택합니다
3. 혼동 행렬에서 역대각선 패턴을 관찰합니다
4. **결과**: 에이전트가 도구를 사용할 때 환각(hallucination)이 더 적게 발생합니다

### 평가 커버리지 추적

**시나리오**: 평가 데이터가 얼마나 완전한가요?

**워크플로우**:

1. 아무 점수나 선택합니다
2. Distribution에서 "all" 탭과 "matched" 탭을 비교합니다
3. 총 개수를 확인합니다. 개별 점수 1의 1,143건 대 567건의 일치 쌍
4. **결과**: 상위 객체의 약 50%만 두 점수를 모두 가지고 있음을 확인합니다

### 품질 저하 감지

**시나리오**: 최근 배포 이후 모델 품질이 저하되었나요?

**워크플로우**:

1. 품질 또는 성능 점수를 선택합니다
2. 배포 전후 기간을 포함하도록 기간을 설정합니다
3. Trend Over Time 차트에서 하락이나 변화가 있는지 검토합니다
4. **결과**: 품질 저하를 빠르게 발견하고 근본 원인을 조사합니다

## 현재 제한 사항

<Callout type="warning">
**베타 기능**: 점수 분석은 현재 베타 단계입니다. 문제나 피드백이 있으면 알려주세요.

**현재 제약 사항**:

- **최대 두 개의 점수**: 현재는 한 번에 최대 두 개의 점수만 비교할 수 있습니다. 다자간 비교가 필요한 경우 쌍별(pairwise) 분석을 수행하세요.
- **동일한 데이터 유형만 가능**: 동일한 데이터 유형의 점수끼리만 비교할 수 있습니다(숫자형은 숫자형끼리, 범주형은 범주형끼리, 불리언은 불리언끼리).
- **샘플링**: 성능 최적화를 위해 점수1 또는 점수2 중 하나라도 10만 건을 초과하는 쿼리는 자동으로 무작위 샘플링을 적용합니다. 이 샘플링은 실제 무작위 샘플링에 근사하며 데이터의 통계적 속성을 유지합니다. 샘플링이 활성화되면 화면에 표시되며, 전체 데이터셋이 필요한 경우 기간 또는 객체 유형 필터를 사용해 분석 범위를 좁힐 수 있습니다.

</Callout>

## 팁과 모범 사례

**비교할 점수 선택하기**

- 동일한 데이터 유형의 점수만 비교할 수 있습니다
- 척도가 다른 점수도 비교할 수 있지만, 오차 메트릭(MAE, RMSE)은 척도 차이의 영향을 받습니다
- 의미 있는 비교를 위해 유사한 차원을 평가하는 점수를 선택하세요

**히트맵 해석하기**

- **대각선 패턴**: 일치도를 나타냅니다(두 점수가 유사한 값을 부여함)
- **역대각선 패턴**: 음의 상관관계를 나타냅니다(한 점수의 높은 값이 다른 점수의 낮은 값에 대응함)
- **산발적인 패턴**: 낮은 상관관계 또는 노이즈가 많은 데이터를 나타냅니다
- **셀의 진하기**: 셀 색이 진할수록 해당 구간 조합에 더 많은 데이터 포인트가 있음을 의미합니다

**Matched 데이터 이해하기**

- 점수는 항상 하나의 상위 객체(트레이스, 관측, 세션, 또는 데이터셋 실행 항목)에 연결됩니다
- 두 점수가 동일한 상위 객체와 연관될 때 두 점수 간의 일치가 존재합니다
- 일치 개수가 개별 개수보다 훨씬 적다면 커버리지 공백이 있는 것입니다
- 일부 평가 방법은 선택적으로 적용될 수 있습니다(예: 엣지 케이스에만 주석을 다는 경우)

## 관련 자료

- 다른 메트릭과 함께 점수 추세를 시각화하려면 [커스텀 대시보드](/docs/metrics/features/custom-dashboards)를 구축하세요.
- 외부 모니터링을 위해 집계된 점수 데이터를 프로그래밍 방식으로 쿼리하려면 [Metrics API](/docs/metrics/features/metrics-api)를 사용하세요.
