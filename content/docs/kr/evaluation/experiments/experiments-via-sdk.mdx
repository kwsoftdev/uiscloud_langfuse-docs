# SDK를 통한 실험

SDK를 통한 실험은 애플리케이션이나 프롬프트를 데이터셋을 통해 프로그래밍 방식으로 반복 실행하고, 선택적으로 결과에 평가 방법을 적용하는 데 사용됩니다. Langfuse에 호스팅된 데이터셋 또는 로컬 데이터셋을 실험의 기반으로 사용할 수 있습니다.

SDK를 통해 실험을 실행하는 방법에 대한 자세한 내용은 [JS/TS SDK 레퍼런스](https://js.reference.langfuse.com/classes/_langfuse_client.ExperimentManager.html)와 [Python SDK 레퍼런스](https://python.reference.langfuse.com/langfuse#Langfuse.run_experiment)를 참조하세요.

## SDK를 통한 실험을 사용하는 이유

- 자체 애플리케이션 로직을 사용할 수 있는 완전한 유연성
- 개별 항목의 출력과 전체 실행 결과를 평가하기 위한 커스텀 채점 함수 사용
- Langfuse UI에서 결정론적인 Python 또는 TypeScript 검사를 작성하고 관측(observation)과 실험 전반에서 재사용하고 싶다면 [코드 평가기](/docs/evaluation/evaluation-methods/code-evaluators) 사용
- 동일한 데이터셋에서 여러 실험을 병렬로 실행
- 기존 평가 인프라와 쉽게 통합
- 배포 전에 회귀를 발견하기 위해 [CI/CD에서 실험 실행](/docs/evaluation/experiments/experiments-ci-cd)

## 실험 러너 SDK

Python과 JS/TS SDK 모두 데이터셋에 대한 실험을 실행하기 위한 고수준 추상화를 제공합니다. 데이터셋은 로컬에 있거나 Langfuse에 호스팅될 수 있습니다. 실험 러너를 사용하는 것이 SDK로 데이터셋에 대한 실험을 실행하는 데 권장되는 방법입니다.

실험 러너는 다음을 자동으로 처리합니다.

- 설정 가능한 제한을 가진 작업의 **동시 실행**
- 관측성을 위한 모든 실행의 **자동 트레이싱**
- 항목 수준 및 실행 수준 평가기를 모두 사용하는 **유연한 평가**
- 개별 실패가 실험을 중단시키지 않도록 하는 **오류 격리**
- 손쉬운 비교 및 추적을 위한 **데이터셋 통합**

실험 러너 SDK는 Langfuse에 호스팅된 데이터셋과 로컬에 호스팅된 데이터셋을 모두 지원합니다. 실험에 Langfuse에 호스팅된 데이터셋을 사용하는 경우, SDK는 Langfuse UI에서 확인하고 비교할 수 있는 데이터셋 실행(dataset run)을 자동으로 생성합니다. Langfuse에 없는 로컬 호스팅 데이터셋의 경우, 트레이스와 (평가를 사용하는 경우) 점수만 Langfuse에 기록됩니다.

### 기본 사용법

로컬 데이터에서 작업(task) 함수를 테스트하는 가장 간단한 실험부터 시작하겠습니다. 이미 Langfuse에 데이터셋이 있다면 [여기를 참조하세요](#usage-with-langfuse-datasets).

```python
from langfuse import get_client
from langfuse.openai import OpenAI

# 클라이언트 초기화
langfuse = get_client()

# 작업 함수 정의
def my_task(*, item, **kwargs):
    question = item["input"]
    response = OpenAI().chat.completions.create(
        model="gpt-4.1", messages=[{"role": "user", "content": question}]
    )

    return response.choices[0].message.content

local_data = [
    {"input": "What is the capital of France?", "expected_output": "Paris"},
    {"input": "What is the capital of Germany?", "expected_output": "Berlin"},
]

result = langfuse.run_experiment(
    name="Geography Quiz",
    description="Testing basic functionality",
    data=local_data,
    task=my_task,
)

# format 메서드를 사용해 결과 출력
print(result.format())
```

트레이스가 Langfuse로 전송되려면 OpenTelemetry가 올바르게 설정되어 있어야 합니다. 구성에 대한 자세한 내용은 [트레이싱 설정 문서](/docs/observability/sdk/overview#initialize-tracing)를 참조하세요. 모든 트레이스가 전송되도록 실행이 끝날 때 항상 스팬 프로세서를 플러시하세요.

```typescript
import { OpenAI } from "openai";
import { NodeSDK } from "@opentelemetry/sdk-node";

import {
  LangfuseClient,
  ExperimentTask,
  ExperimentItem,
} from "@langfuse/client";
import { observeOpenAI } from "@langfuse/openai";
import { LangfuseSpanProcessor } from "@langfuse/otel";

// OpenTelemetry 초기화
const otelSdk = new NodeSDK({ spanProcessors: [new LangfuseSpanProcessor()] });
otelSdk.start();

// 클라이언트 초기화
const langfuse = new LangfuseClient();

// 로컬 데이터에서 실험 실행
const localData: ExperimentItem[] = [
  { input: "What is the capital of France?", expectedOutput: "Paris" },
  { input: "What is the capital of Germany?", expectedOutput: "Berlin" },
];

// 작업 함수 정의
const myTask: ExperimentTask = async (item) => {
  const question = item.input;

  const response = await observeOpenAI(new OpenAI()).chat.completions.create({
    model: "gpt-4.1",
    messages: [
      {
        role: "user",
        content: question,
      },
    ],
  });

  return response;
};

// 실험 실행
const result = await langfuse.experiment.run({
  name: "Geography Quiz",
  description: "Testing basic functionality",
  data: localData,
  task: myTask,
});

// 포맷된 결과 출력
console.log(await result.format());

// 중요: 트레이스가 전송되도록 OTEL SDK를 종료
await otelSdk.shutdown();
```

**JS/TS SDK 참고 사항**: 트레이스가 Langfuse로 전송되려면 OpenTelemetry가 올바르게 설정되어 있어야 합니다. 구성에 대한 자세한 내용은 [트레이싱 설정 문서](/docs/observability/sdk/overview#initialize-tracing)를 참조하세요. 모든 트레이스가 전송되도록 실행이 끝날 때 항상 스팬 프로세서를 플러시하세요.

### 멀티모달 실험 [#multimodal-experiments]

SDK 기반 실험은 `input`, `expectedOutput`, `metadata`에 미디어 첨부 파일이 포함된 데이터셋에서도 실행할 수 있습니다. SDK를 통해 데이터셋을 가져오면 각 미디어 토큰은 기본적으로 서명된 `LangfuseMediaReference`로 하이드레이션됩니다.

<Callout type="info">
  멀티모달 데이터셋은 Python SDK `>= 4.10.0`과 JS/TS SDK `@langfuse/client
  >= 5.6.0`에서 SDK 기반 실험에 대해 지원됩니다. UI 기반 실험은 아직 미디어
  첨부 파일이 있는 데이터셋 항목을 지원하지 않습니다.
</Callout>

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab>
{/* PYTHON SDK */}

```python
from langfuse import get_client
from langfuse.media import LangfuseMediaReference

langfuse = get_client()

dataset = langfuse.get_dataset("visual-qa")

def my_multi_modal_task(*, item, **kwargs):

    assert isinstance(image, LangfuseMediaReference)

    # 여기서 멀티모달 애플리케이션을 호출하세요.
    return run_visual_qa(
        question=item.input["question"],

result = dataset.run_experiment(
    name="Visual QA",
    task=my_multi_modal_task,
)
```

`LangfuseMediaReference`는 미디어를 원시 바이트, 원시 base64, 또는 데이터 URI로 가져올 수 있는 헬퍼를 제공합니다.

| SDK    | Bytes           | Base64           | Data URI           |
| ------ | --------------- | ---------------- | ------------------ |
| Python | `fetch_bytes()` | `fetch_base64()` | `fetch_data_uri()` |
| JS/TS  | `fetchBytes()`  | `fetchBase64()`  | `fetchDataUri()`   |

해결된 URL은 서명되어 있으며 만료됩니다. 실험에서 사용하기 전에 URL이 만료되면 데이터셋을 다시 가져와 최신 미디어 참조를 받으세요.

### 고급 기능

평가기와 고급 구성 옵션으로 실험을 향상시킬 수 있습니다.

#### 평가기 [#evaluators]

평가기는 항목 수준에서 작업 출력의 품질을 평가합니다. 각 항목의 input, metadata, output, expected output을 받아 평가 메트릭을 반환하며, 이는 Langfuse에서 트레이스에 대한 점수로 보고됩니다.

이러한 SDK 평가기 함수는 실험 프로세스 내에서 실행됩니다. Langfuse UI에서 결정론적인 평가기 코드를 작성하고 Langfuse가 이를 실험 관측(observation)에서 실행하도록 하려면 [코드 평가기](/docs/evaluation/evaluation-methods/code-evaluators)를 사용하세요.

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab>
{/* PYTHON SDK */}

```python
from langfuse import Evaluation

# 평가 함수 정의
def accuracy_evaluator(*, input, output, expected_output, metadata, **kwargs):
    if expected_output and expected_output.lower() in output.lower():
        return Evaluation(name="accuracy", value=1.0, comment="Correct answer found")

    return Evaluation(name="accuracy", value=0.0, comment="Incorrect answer")

def length_evaluator(*, input, output, **kwargs):
    return Evaluation(name="response_length", value=len(output), comment=f"Response has {len(output)} characters")

# 여러 평가기 사용
result = langfuse.run_experiment(
    name="Multi-metric Evaluation",
    data=test_data,
    task=my_task,
    evaluators=[accuracy_evaluator, length_evaluator]
)

print(result.format())
```

#### 실행 수준 평가기

실행 수준 평가기는 전체 실험 결과를 평가하고 집계 메트릭을 계산합니다. Langfuse 데이터셋에서 실행할 경우, 이러한 점수는 전체 실험 성능을 추적하기 위해 전체 데이터셋 실행에 첨부됩니다.

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab>
{/* PYTHON SDK */}

```python
from langfuse import Evaluation

def average_accuracy(*, item_results, **kwargs):
    """모든 항목의 평균 정확도를 계산합니다"""
    accuracies = [
        eval.value for result in item_results
        for eval in result.evaluations
        if eval.name == "accuracy"
    ]

    if not accuracies:
        return Evaluation(name="avg_accuracy", value=None)

    avg = sum(accuracies) / len(accuracies)

    return Evaluation(name="avg_accuracy", value=avg, comment=f"Average accuracy: {avg:.2%}")

result = langfuse.run_experiment(
    name="Comprehensive Analysis",
    data=test_data,
    task=my_task,
    evaluators=[accuracy_evaluator],
    run_evaluators=[average_accuracy]
)

print(result.format())
```

</Tab>
<Tab>
{/* JS/TS SDK */}

```typescript
const averageAccuracy = async ({ itemResults }) => {
  // 모든 항목의 평균 정확도를 계산합니다
  const accuracies = itemResults
    .flatMap((result) => result.evaluations)
    .filter((evaluation) => evaluation.name === "accuracy")
    .map((evaluation) => evaluation.value as number);

  if (accuracies.length === 0) {
    return { name: "avg_accuracy", value: null };
  }

  const avg = accuracies.reduce((sum, val) => sum + val, 0) / accuracies.length;

  return {
    name: "avg_accuracy",
    value: avg,
    comment: `Average accuracy: ${(avg * 100).toFixed(1)}%`,
  };
};

const result = await langfuse.experiment.run({
  name: "Comprehensive Analysis",
  data: testData,
  task: myTask,
  evaluators: [accuracyEvaluator],
  runEvaluators: [averageAccuracy],
});

console.log(await result.format());
```

### Autoevals 통합

[autoevals 라이브러리](https://github.com/braintrustdata/autoevals) 통합을 통해 사전 구축된 평가 함수를 사용할 수 있습니다.

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab>
{/* PYTHON SDK */}

Python SDK는 직접 통합을 통해 AutoEvals 평가기를 지원합니다.

```python
from langfuse.experiment import create_evaluator_from_autoevals
from autoevals.llm import Factuality

evaluator = create_evaluator_from_autoevals(Factuality())

result = langfuse.run_experiment(
    name="Autoevals Integration Test",
    data=test_data,
    task=my_task,
    evaluators=[evaluator]
)

print(result.format())
```

</Tab>
<Tab>
{/* JS/TS SDK */}

JS SDK는 사전 구축된 평가 함수를 위해 AutoEvals 라이브러리와 원활하게 통합됩니다.

```typescript
import { Factuality, Levenshtein } from "autoevals";
import { createEvaluatorFromAutoevals } from "@langfuse/client";

// AutoEvals 평가기를 Langfuse 호환 형식으로 변환
const factualityEvaluator = createEvaluatorFromAutoevals(Factuality());
const levenshteinEvaluator = createEvaluatorFromAutoevals(Levenshtein());

// 추가 매개변수와 함께 사용
const customFactualityEvaluator = createEvaluatorFromAutoevals(
  Factuality,
  { model: "gpt-4o" } // 추가 AutoEvals 매개변수
);

const result = await langfuse.experiment.run({
  name: "AutoEvals Integration Test",
  data: testDataset,
  task: myTask,
  evaluators: [
    factualityEvaluator,
    levenshteinEvaluator,
    customFactualityEvaluator,
  ],
});

console.log(await result.format());
```

</Tab>
</LangTabs>

## 선택 사항: UI에서 SDK 실험 트리거하기

SDK를 통한 실험을 설정할 때, Langfuse UI에서 실험 실행을 트리거할 수 있도록 하면 유용할 수 있습니다.

Langfuse로부터 트리거 요청을 받기 위해 웹훅을 설정해야 합니다.

<Steps>

### 데이터셋으로 이동

- **다음으로 이동**: `Your Project` > `Datasets`
- 원격 실험 트리거를 설정할 데이터셋을 **클릭**

<Frame className="max-w-lg">![New Experiment Button](/images/docs/navigate-to-dataset.png)</Frame>

### 설정 페이지 열기

`Start Experiment`를 **클릭**하여 설정 페이지를 엽니다.

<Frame className="max-w-lg">![New Experiment Button](/images/docs/trigger-process.png)</Frame>

`Custom Experiment` 아래의 `⚡`를 **클릭**합니다.

<Frame className="max-w-lg">
  ![New Experiment Button](/images/docs/trigger-remote-experiment-1.png)
</Frame>

### 웹훅 구성하기 [#configure-webhook]

실험이 트리거될 때 웹훅을 수신할 외부 평가 서비스의 URL을 **입력**하세요.
웹훅으로 전송될 기본 구성을 **지정**하세요. 사용자는 실험을 트리거할 때 이를 수정할 수 있습니다.

<Frame className="max-w-lg">
  ![New Experiment Button](/images/docs/trigger-remote-experiment-2.png)
</Frame>

### 실험 트리거하기

구성이 완료되면 팀원들은 **Custom Experiment** 옵션 아래의 `Run` 버튼을 통해 원격 실험을 트리거할 수 있습니다. Langfuse는 데이터셋 메타데이터(ID와 이름)와 커스텀 구성을 웹훅으로 전송합니다.

<Frame className="max-w-lg">
  ![New Experiment Button](/images/docs/trigger-remote-experiment-3.png)
</Frame>

</Steps>

**일반적인 워크플로**: 웹훅이 요청을 수신하고, Langfuse에서 데이터셋을 가져오고, 데이터셋 항목에 대해 애플리케이션을 실행하고, 결과를 평가한 다음, 점수를 새로운 실험 실행(Experiment run)으로 Langfuse에 다시 수집합니다.
