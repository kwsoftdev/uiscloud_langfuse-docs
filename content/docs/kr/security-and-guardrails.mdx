# LLM 보안 및 가드레일

LLM 기반 애플리케이션에는 다양한 잠재적 안전 위험이 존재합니다. 여기에는 프롬프트 인젝션, 개인 식별 정보(PII) 유출, 유해한 프롬프트 등이 포함됩니다. Langfuse를 사용하면 이러한 보안 위험을 모니터링하고 방지할 수 있으며, 사고가 발생했을 때 이를 조사할 수 있습니다.

## LLM 보안이란?

LLM 보안은 LLM과 그 인프라를 무단 접근, 오용, 적대적 공격으로부터 보호하기 위한 보호 조치를 구현하여 모델과 데이터의 무결성 및 기밀성을 보장하는 것을 말합니다. 이는 AI/ML 시스템에서 윤리적인 사용을 유지하고, 프롬프트 인젝션과 같은 보안 위험을 방지하며, 안전한 조건에서 신뢰할 수 있는 운영을 보장하는 데 매우 중요합니다.

## LLM 보안은 어떻게 작동하나요?

<Callout type="info">

LLM 보안은 다음 두 가지를 조합하여 다룰 수 있습니다.

- 런타임 보안 조치를 위한 LLM 보안 라이브러리
- 이러한 조치의 효과를 사후 평가하기 위한 Langfuse

</Callout>

### 1. 런타임 보안 조치

LLM 기반 애플리케이션의 보안 위험을 완화하는 데 사용할 수 있는 널리 알려진 보안 라이브러리들이 있습니다. 여기에는 [LLM Guard](https://llm-guard.com), [Prompt Armor](https://promptarmor.com), [NeMo Guardrails](https://github.com/NVIDIA/NeMo-Guardrails), [Microsoft Azure AI Content Safety](https://azure.microsoft.com/en-us/products/ai-services/ai-content-safety), [Lakera](https://www.lakera.ai)가 포함됩니다. 이러한 라이브러리는 다음과 같은 방식으로 보안 조치를 지원합니다.

1. 잠재적으로 유해하거나 부적절한 프롬프트를 모델에 전송하기 전에 감지하여 차단합니다.
2. 민감한 PII를 모델에 전송하기 전에 마스킹하고, 응답에서 이를 다시 복원(un-redact)합니다.
3. 프롬프트와 완료 결과의 유해성, 관련성, 민감한 자료 여부를 런타임에 평가하고, 필요한 경우 응답을 차단합니다.

### 2. Langfuse를 통한 보안 조치 모니터링 및 평가

Langfuse [트레이싱](/docs/observability/overview)을 사용하면 보안 메커니즘의 각 단계에 대한 가시성과 신뢰도를 확보할 수 있습니다. 다음은 일반적인 워크플로입니다.

1. 트레이스를 수동으로 확인하여 보안 이슈를 조사합니다.
2. Langfuse 대시보드에서 시간에 따른 보안 점수를 모니터링합니다.
3. 보안 검사를 검증합니다. Langfuse [점수](/docs/evaluation/core-concepts)를 사용하여 보안 도구의 효과를 평가할 수 있습니다. Langfuse를 팀의 워크플로에 통합하면 어떤 보안 위험이 가장 흔하게 발생하는지 파악하고, 해당 이슈에 특화된 더욱 견고한 도구를 구축하는 데 도움이 됩니다. 고려할 수 있는 두 가지 주요 워크플로는 다음과 같습니다.
   - [주석(UI에서)](/docs/evaluation/evaluation-methods/annotation-queues). 프로덕션 트레이스의 일부에 주석을 달아 기준선을 설정하면, 보안 도구가 반환한 보안 점수를 이러한 주석과 비교할 수 있습니다.
   - [자동 평가](/docs/evaluation/evaluation-methods/llm-as-a-judge). Langfuse의 모델 기반 평가는 비동기적으로 실행되며, 트레이스를 스캔하여 유해성이나 민감성 등을 확인하고 잠재적 위험을 표시하며 LLM 보안 설정의 허점을 식별할 수 있습니다. 이러한 평가를 설정하는 방법에 대해 더 알아보려면 문서를 확인하세요.
4. 지연 시간을 추적합니다. 일부 LLM 보안 검사는 모델 호출 전에 완료를 기다려야 하고, 일부는 사용자에게 전달되는 응답을 차단합니다. 따라서 이러한 검사는 LLM 애플리케이션 전체 지연 시간에 중요한 요소가 됩니다. Langfuse는 트레이스 내에서 이러한 검사들의 지연 시간을 세분화하여, 검사가 기다릴 만한 가치가 있는지 파악하는 데 도움을 줍니다.

## 시작하기

> 예시: 개인 식별 정보(PII) 익명화

LLM에 PII를 노출하는 것은 계약상 의무나 규제 준수 요건 위반, 또는 데이터 유출이나 데이터 침해의 위험을 초래하는 등 심각한 보안 및 개인정보 보호 위험을 야기할 수 있습니다.

개인 식별 정보(PII)에는 다음이 포함됩니다.

- 신용카드 번호
- 성명
- 전화번호
- 이메일 주소
- 사회보장번호
- IP 주소

아래 예시는 주어진 법정 속기록을 요약하는 간단한 애플리케이션을 보여줍니다. 개인정보 보호를 위해 이 애플리케이션은 정보를 모델에 입력하기 전에 PII를 익명화하고, 이후 응답에서 이를 다시 복원(un-redact)하여 일관된 요약을 생성하고자 합니다.

프롬프트 인젝션, 금지된 주제, 악성 URL 등 다른 보안 위험에 대해 더 알아보려면 각 라이브러리의 문서를 확인하거나, 더 많은 예시가 포함된 [보안 쿡북](/docs/security/example-python)을 참고하세요.

<Steps>

### 패키지 설치

이 예시에서는 런타임 보안 검사를 위해 오픈소스 라이브러리인 [LLM Guard](https://protectai.github.io/llm-guard/)를 사용합니다. 이 예시들은 [Prompt Armor](https://promptarmor.com), [NeMo Guardrails](https://github.com/NVIDIA/NeMo-Guardrails), [Microsoft Azure AI Content Safety](https://azure.microsoft.com/en-us/products/ai-services/ai-content-safety), [Lakera](https://www.lakera.ai)와 같은 다른 라이브러리에도 쉽게 적용할 수 있습니다.

먼저 보안 패키지와 Langfuse 도구를 임포트합니다.

```bash
pip install llm-guard langfuse openai
```

```python
from llm_guard.input_scanners import Anonymize
from llm_guard.input_scanners.anonymize_helpers import BERT_LARGE_NER_CONF
from langfuse.openai import openai # OpenAI 통합
from langfuse import observe
from llm_guard.output_scanners import Deanonymize
from llm_guard.vault import Vault
```

### PII 익명화 및 복원 후 Langfuse로 트레이싱

프로세스의 각 단계를 개별 함수로 분리하여 Langfuse에서 각 단계를 별도로 추적할 수 있도록 합니다.

함수에 `@observe()` 데코레이터를 적용하면 프로세스의 각 단계를 추적하고 보안 도구가 반환한 위험 점수를 모니터링할 수 있습니다. 이를 통해 보안 도구가 얼마나 잘 작동하는지, 그리고 예상대로 PII를 포착하고 있는지 확인할 수 있습니다.

```python
vault = Vault()

@observe()
def anonymize(input: str):
  scanner = Anonymize(vault, preamble="Insert before prompt", allowed_names=["John Doe"], hidden_names=["Test LLC"],
                    recognizer_conf=BERT_LARGE_NER_CONF, language="en")
  sanitized_prompt, is_valid, risk_score = scanner.scan(prompt)
  return sanitized_prompt

@observe()
def deanonymize(sanitized_prompt: str, answer: str):
  scanner = Deanonymize(vault)
  sanitized_model_output, is_valid, risk_score = scanner.scan(sanitized_prompt, answer)

  return sanitized_model_output
```

### LLM 호출 계측

이 예시에서는 네이티브 OpenAI SDK 통합을 사용하여 LLM 호출을 계측합니다. 이를 통해 토큰 수, 모델 파라미터, 모델에 전송된 정확한 프롬프트를 자동으로 수집할 수 있습니다.

참고: Langfuse는 여러 프레임워크(예: LlamaIndex, LangChain, Haystack 등)와 [네이티브로 통합](/integrations)되며, [SDK](/docs/sdk)를 통해 어떤 LLM이든 쉽게 계측할 수 있습니다.

```python
@observe()
def summarize_transcript(prompt: str):
  sanitized_prompt = anonymize(prompt)

  answer = openai.chat.completions.create(
        model="gpt-4o-mini",
        max_tokens=100,
        messages=[
          {"role": "system", "content": "Summarize the given court transcript."},
          {"role": "user", "content": sanitized_prompt}
        ],
    ).choices[0].message.content

  sanitized_model_output = deanonymize(sanitized_prompt, answer)

  return sanitized_model_output
```

### 애플리케이션 실행

함수를 실행합니다. 이 예시에서는 법정 속기록의 일부를 입력합니다. 민감한 정보를 다루는 애플리케이션은 HIPAA나 GDPR과 같은 개인정보 보호 정책을 준수하기 위해 익명화 및 복원 기능을 사용해야 하는 경우가 많습니다.

```python
prompt = """
Plaintiff, Jane Doe, by and through her attorneys, files this complaint
against Defendant, Big Corporation, and alleges upon information and belief,
except for those allegations pertaining to personal knowledge, that on or about
July 15, 2023, at the Defendant's manufacturing facility located at 123 Industrial Way, Springfield, Illinois, Defendant negligently failed to maintain safe working conditions,
leading to Plaintiff suffering severe and permanent injuries. As a direct and proximate
result of Defendant's negligence, Plaintiff has endured significant physical pain, emotional distress, and financial hardship due to medical expenses and loss of income. Plaintiff seeks compensatory damages, punitive damages, and any other relief the Court deems just and proper.
"""
summarize_transcript(prompt)
```

## FAQ

<Details>
<Summary>LLM Guard란 무엇인가요?</Summary>

[LLM Guard](https://llm-guard.com)는 Protect AI에서 제공하는 오픈소스 라이브러리로, LLM 애플리케이션을 위한 보안 가드레일을 제공합니다. 여기에는 입력 스캐너(프롬프트 인젝션 탐지, PII 익명화, 유해성 탐지, 주제 차단)와 출력 스캐너(콘텐츠 검열, 편향 탐지, 악성 URL 탐지, 복원(deanonymization))가 포함됩니다. LLM Guard는 Langfuse와 통합하여 각 보안 검사의 효과를 추적하고 모니터링할 수 있습니다.

</Details>

<Details>
<Summary>LLM용 보안 가드레일이란 무엇인가요?</Summary>

보안 가드레일은 LLM 애플리케이션의 입력과 출력을 가로채고 필터링하여 오용과 피해를 방지하는 보호 조치입니다. 여기에는 입력 가드레일(프롬프트 인젝션 및 악성 입력 차단), 출력 가드레일(유해하거나 위험한 응답 필터링), PII 탐지 및 마스킹, 콘텐츠 필터링, 주제 제한이 포함됩니다. 가드레일은 애플리케이션 계층에서 작동하며 모델 수준의 안전성 학습을 보완합니다.

</Details>

<Details>
<Summary>LLM 애플리케이션에서 프롬프트 인젝션을 어떻게 방지하나요?</Summary>

프롬프트 인젝션은 여러 계층에서 완화할 수 있습니다. (1) LLM Guard나 Lakera와 같은 가드레일 라이브러리를 사용하여 인젝션 시도가 모델에 도달하기 전에 탐지하고 차단합니다. (2) 시스템 지침과 사용자 입력을 명확히 분리합니다. (3) Langfuse 트레이싱을 사용하여 프로덕션 환경에서 인젝션 시도를 모니터링합니다. (4) 의심스러운 패턴을 표시하는 자동 평가를 설정합니다. 어떤 단일 방법도 100% 효과적이지 않으므로, 모니터링을 포함한 다층 방어(defense-in-depth) 전략을 권장합니다.

</Details>

<Details>
<Summary>프로덕션 환경에서 LLM 보안을 어떻게 모니터링하나요?</Summary>

Langfuse를 사용하여 LLM 보안 상태를 모니터링할 수 있습니다. (1) 가드레일 파이프라인을 통과하는 모든 요청을 트레이싱하여 어떤 검사가 트리거되었는지 확인합니다. (2) LLM-as-a-Judge 평가기를 사용하여 유해성, PII 유출, 프롬프트 인젝션에 대해 트레이스를 자동으로 채점합니다. (3) 커스텀 대시보드를 설정하여 시간에 따른 보안 점수를 추적합니다. (4) 주석 큐를 사용하여 플래그가 지정된 대화를 사람이 검토합니다. (5) 보안 검사의 지연 시간 영향을 추적하여 보호와 사용자 경험 사이의 균형을 맞춥니다.

</Details>
