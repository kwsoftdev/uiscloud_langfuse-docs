# 웹훅 & Slack 통합

웹훅을 사용하면 Langfuse에서 프롬프트 버전이 생성, 수정, 삭제될 때마다 실시간 알림을 받을 수 있습니다. 이를 통해 API를 폴링하지 않고도 CI/CD 파이프라인을 트리거하거나, 프롬프트 카탈로그를 동기화하거나, 변경 사항을 감사할 수 있습니다.

## 웹훅을 사용하는 이유

- **프로덕션 모니터링**: 프로덕션 프롬프트가 업데이트될 때 알림을 받습니다
- **팀 협업**: 프롬프트 변경 사항을 팀 전체에 공유합니다
- **동기화**: 프롬프트 카탈로그를 다른 시스템과 동기화합니다

## 시작하기

`Prompts`로 이동하여 `Automations`를 클릭합니다.

감시할 이벤트를 선택합니다.

<Frame className="max-w-md">

</Frame>

웹훅을 트리거할 프롬프트 버전 작업을 선택합니다.

- **Created:** 새 버전이 추가된 경우.
- **Updated:** 레이블 또는 태그가 변경된 경우 (레이블/태그를 얻은 버전과 잃은 버전 각각에 대해 이벤트가 하나씩 발생하여 총 두 개의 이벤트가 발생합니다).
- **Deleted:** 버전이 삭제된 경우.

(선택 사항) 특정 프롬프트에서만 트리거되도록 필터링할 수 있습니다.

### 요청 구성

<Frame className="max-w-md">
  ![요청 구성](/images/docs/webhook-action.png)
</Frame>

- **URL**: POST 요청을 받는 HTTPS 엔드포인트입니다.
- **Headers**: 기본 헤더는 다음과 같습니다:
  - `Content-Type: application/json`
  - `User-Agent: Langfuse/1.0`
  - `x-langfuse-signature: <sig>` (아래 HMAC 서명 검증 관련 참고 사항 참조)
- **필요한 경우 사용자 지정 정적 헤더를 추가합니다.**

### 전달 확인 응답

핸들러는 다음 조건을 충족해야 합니다:

- 수신을 확인하기 위해 HTTP 2xx 상태를 반환해야 합니다.
- 멱등성을 가져야 합니다—Langfuse는 성공 응답을 받을 때까지 지수 백오프 방식으로 재시도할 수 있습니다.

### 신뢰성 검증 (권장)

각 요청에는 `x-langfuse-signature`에 HMAC SHA-256 서명이 포함되어 있습니다.
웹훅을 생성할 때 시크릿을 받아둡니다 (나중에 다시 생성할 수도 있습니다).

```python
import hmac
import hashlib
from typing import Optional


def verify_langfuse_signature(
    raw_body: str,
    signature_header: str,
    secret: str,
) -> bool:
    """
    Validate a Langfuse webhook/event signature.

    Parameters
    ----------
    raw_body : str
        The request body exactly as received (no decoding or reformatting).
    signature_header : str
        The value of the `Langfuse-Signature` header, e.g. "t=1720701136,s=0123abcd...".
    secret : str
        Your Langfuse signing secret.

    Returns
    -------
    bool
        True if the signature is valid, otherwise False.
    """
    # "t=timestamp,s=signature"를 두 개의 키/값 조각으로 분리합니다
    try:
        ts_pair, sig_pair = signature_header.split(",", 1)
    except ValueError:  # 형식이 잘못되었거나 쉼표가 없는 경우
        return False

    # 값 추출 (첫 번째 "=" 이후의 모든 부분)
    if "=" not in ts_pair or "=" not in sig_pair:
        return False
    timestamp = ts_pair.split("=", 1)[1]
    received_sig_hex = sig_pair.split("=", 1)[1]

    # 메시지를 재구성하고 예상되는 HMAC-SHA256 16진수 다이제스트를 계산합니다
    message = f"{timestamp}.{raw_body}".encode("utf-8")
    expected_sig_hex = hmac.new(
        secret.encode("utf-8"), message, hashlib.sha256
    ).hexdigest()

    # *디코딩된* 바이트 문자열에 대해 상수 시간 비교를 사용합니다
    try:
        return hmac.compare_digest(
            bytes.fromhex(received_sig_hex), bytes.fromhex(expected_sig_hex)
        )
    except ValueError:  # received_sig_hex가 유효한 16진수가 아닌 경우
        return False
```

```ts
import crypto from "crypto";

export function verifyLangfuseSignature(
  rawBody: string,
  signatureHeader: string,
  secret: string
): boolean {
  const [tsPair, sigPair] = signatureHeader.split(",");
  if (!tsPair || !sigPair) return false;

  const timestamp = tsPair.split("=")[1];
  const receivedSig = sigPair.split("=")[1];
  const expectedSig = crypto
    .createHmac("sha256", secret)
    .update(`${timestamp}.${rawBody}`, "utf8")
    .digest("hex");

  return crypto.timingSafeEqual(
    Buffer.from(receivedSig, "hex"),
    Buffer.from(expectedSig, "hex")
  );
}
```
