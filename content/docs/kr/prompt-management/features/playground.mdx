# LLM Playground

Langfuse Prompt Playground에서 프롬프트를 직접 테스트하고 반복 개선할 수 있습니다. 프롬프트와 모델 매개변수를 조정하여 입력 변경에 따라 다양한 모델이 어떻게 응답하는지 확인해 보세요. 이를 통해 다른 도구로 전환하거나 코드를 작성할 필요 없이 LLM 앱에서 최상의 결과를 얻기 위해 프롬프트를 빠르게 반복 개선하고 최적화할 수 있습니다.

<Frame fullWidth>![LLM Playground](/images/docs/playground-overview.png)</Frame>

## 핵심 기능

### 나란히 비교 보기

여러 프롬프트 변형을 나란히 비교할 수 있습니다. 모든 변형을 한 번에 실행하거나 단일 변형에 집중할 수 있습니다. 각 변형은 자체적인 LLM 설정, 변수, 도구 정의, 플레이스홀더를 유지하므로 모든 변경 사항의 영향을 즉시 확인할 수 있습니다.

### 프롬프트를 Playground에서 열기

[Langfuse 프롬프트 관리](/docs/prompt-management/get-started)로 생성한 프롬프트를 Playground에서 열 수 있습니다.

<Video
  src="https://static.langfuse.com/docs-videos/playground-open-prompt.mp4"
  gifStyle
  aspectRatio={16 / 9}

### 프롬프트를 프롬프트 관리에 저장

프롬프트가 만족스러우면 저장 버튼을 클릭하여 프롬프트 관리에 저장할 수 있습니다.

<Video
  src="https://static.langfuse.com/docs-videos/playground-save-prompt.mp4"

### 생성 결과를 Playground에서 열기

생성 상세 페이지에서 `Open in Playground` 버튼을 클릭하여 [Langfuse 관측성](/docs/observability)의 생성 결과를 Playground에서 열 수 있습니다.

### 도구 호출 및 구조화된 출력

Langfuse Playground는 도구 호출과 구조화된 출력 스키마를 지원하여, 도구 호출에 의존하고 특정 응답 형식을 강제하는 LLM 실행을 정의, 테스트, 검증할 수 있습니다.

<Callout type="info">
  현재 Langfuse는 OpenAI ChatML 형식으로 되어 있는 tool 타입 관측(observation)만
  Playground에서 열 수 있도록 지원합니다. 추가 형식 지원을 원하시면 [퍼블릭
  로드맵](https://langfuse.com/ideas)에 요청을 자유롭게 추가해 주세요.
</Callout>

**도구 호출**

- JSON 스키마 정의로 커스텀 도구 정의
- 도구 응답을 모킹하여 도구에 의존하는 프롬프트를 실시간으로 테스트
- 도구 정의를 프로젝트에 저장

**구조화된 출력**

- JSON 스키마를 사용하여 응답 형식 강제
- 스키마를 프로젝트에 저장
- 구조화된 출력을 사용하는 OpenAI 생성 결과에서 Playground로 바로 이동

### 프롬프트 변수 추가

Playground에서 프롬프트 변수를 추가하여 프롬프트에 대한 다양한 입력을 시뮬레이션할 수 있습니다.

<Frame fullWidth>
  ![프롬프트 변수 추가](/images/docs/playground-variables.png)
</Frame>

### 원하는 모델 사용하기

Langfuse 프로젝트 설정에 사용하려는 모델의 API 키를 추가하여 원하는 모델을 사용할 수 있습니다. LLM 연결 설정 방법은 [여기](/docs/administration/llm-connection)에서 확인할 수 있습니다.

<Frame fullWidth>
  ![사용할 모델
  선택하기](/images/docs/playground-model-selection.png)
</Frame>

선택적으로, 많은 LLM 제공업체는 모델 호출 시 추가 매개변수를 지원합니다. 모델 선택 드롭다운에서 "Additional Options"를 토글하면 Playground에서 이러한 매개변수를 전달할 수 있습니다. 자세한 내용은 [추가 제공업체 옵션에 대한 문서](/docs/administration/llm-connection#advanced-configurations)를 참고하세요.

## 관련 자료

- 데이터셋에서 프롬프트 및 모델 변형에 대한 구조화된 회귀 테스트를 수행하려면 [실험](/docs/evaluation/core-concepts#experiments)을 사용하세요.
