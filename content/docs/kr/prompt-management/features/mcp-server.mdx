# Langfuse MCP 서버를 통한 프롬프트 관리

Langfuse [Model Context Protocol](https://modelcontextprotocol.io)(MCP) 서버는 AI 어시스턴트와 에이전트가 [프롬프트 관리](/docs/prompt-management/overview) 시스템과 상호작용할 수 있도록 합니다. 이 서버는 `/api/public/mcp`(streamableHttp)에 Langfuse에 직접 내장되어 있으며, 별도의 외부 설정이나 빌드 단계가 필요하지 않습니다.

<Callout type="info">

[MCP 레퍼런스](https://mcp.reference.langfuse.com)는 현재의 Langfuse MCP 서버, 설정 스니펫, 도구, 입력 스키마, 생성된 요청 예시에 대한 표준 출처입니다.

</Callout>

## 설정

인증 및 클라이언트 구성에 대한 전체 안내는 [MCP 서버 설정 가이드](/docs/api-and-data-platform/features/mcp-server#set-up)를 참조하세요.

## 예시 워크플로우

### 새 프롬프트 생성

AI 에이전트에게 다음과 같이 요청합니다.

> "이름과 상품 변수를 포함하는 친근한 인사 템플릿으로 'customer-email'이라는 새 텍스트 프롬프트를 만들어줘. 'draft'로 태그를 지정하고 production 레이블을 추가해줘."

에이전트는 `createTextPrompt`를 사용하여 프롬프트 버전을 생성합니다.

### 프롬프트를 프로덕션으로 승격

> "customer-email 프롬프트의 production 레이블을 버전 2에서 버전 3으로 옮겨줘"

에이전트는 `updatePromptLabels`를 사용하여 레이블 지정을 업데이트합니다.

### 채팅 프롬프트 반복 작업

> "code-review 프롬프트의 새 버전을 만들되 시스템 지침을 개선하고, 리뷰 형식을 보여주는 어시스턴트 메시지 예시를 추가해줘"

에이전트는 `getPrompt`로 프롬프트를 가져온 뒤 `createChatPrompt`를 사용하여 개선된 콘텐츠로 새 버전을 생성합니다.
