# 프롬프트 관리

프롬프트 관리는 LLM 애플리케이션에서 사용하는 프롬프트를 체계적으로 저장, 버전 관리, 조회하는 방식입니다. 애플리케이션 코드에 프롬프트를 하드코딩하는 대신, Langfuse에서 중앙 집중식으로 관리할 수 있습니다.

### 프롬프트 업데이트와 코드 배포 분리

대부분의 LLM 애플리케이션에서 **프롬프트 반복 작업과 코드 배포**는 **서로 다른 담당자**가 수행합니다. 제품 관리자와 도메인 전문가는 프롬프트를 반복적으로 개선하고, 엔지니어는 배포를 관리합니다.

프롬프트가 코드에 포함되어 있으면, 간단한 텍스트 변경조차 엔지니어링 참여, 코드 리뷰, 전체 배포 주기를 거쳐야 하므로 2분이면 끝날 업데이트가 몇 시간 또는 며칠씩 걸리게 됩니다.

프롬프트가 Langfuse에 있으면, 비기술 팀원이 UI에서 프롬프트를 직접 업데이트하는 동안 애플리케이션은 최신 버전을 자동으로 가져옵니다. 이러한 **관심사 분리** 덕분에 엔지니어링 참여나 별도 배포 없이도 **프롬프트 업데이트가 즉시 반영**됩니다.

### 지연 시간 없음, 가용성 위험 없음

**Langfuse 프롬프트 관리는 애플리케이션에 어떠한 지연 시간도 추가하지 않습니다.** 프롬프트는 SDK에 의해 클라이언트 측에서 캐싱되므로, 조회 속도는 메모리에서 읽는 것만큼 빠릅니다. 자세한 내용은 [캐싱 문서 페이지](/docs/prompt-management/features/caching)를 참고하세요.

## 시작하기

먼저 [첫 번째 프롬프트를 추가](/docs/prompt-management/get-started)하여 Langfuse에 등록하고, 이를 애플리케이션과 연결해보세요. UI에서 처음부터 프롬프트를 새로 만들 수도 있고, 애플리케이션에 이미 있는 프롬프트를 가져올 수도 있습니다.

핵심 개념인 [프롬프트 유형, 버전 관리, 라벨, 설정](/docs/prompt-management/data-model)을 잠시 시간을 들여 이해해보세요.

Langfuse에 프롬프트를 등록하고 애플리케이션에서 사용하기 시작했다면, Langfuse 프롬프트 관리를 최대한 활용하기 위해 다음을 시도해볼 수 있습니다:

- 프롬프트 버전별 성능을 분석하려면 [프롬프트를 트레이스에 연결](/docs/prompt-management/features/link-to-traces)하세요
- 환경별 배포를 관리하려면 [버전 관리와 라벨을 활용](/docs/prompt-management/features/prompt-version-control)하세요

특정 주제를 찾고 계신가요? _Features_ 섹션에서 세부 주제별 가이드를 확인해보세요.
