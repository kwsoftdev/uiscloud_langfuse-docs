# 커스텀 대시보드

Langfuse 커스텀 대시보드로 LLM 애플리케이션 데이터를 실행 가능한 인사이트로 전환하세요.
지연 시간과 비용 최적화부터 품질 모니터링, 사용자 행동 분석에 이르기까지 팀에게 가장 중요한 메트릭을 추적하는 개인화된 뷰를 만들 수 있습니다.

커스텀 대시보드는 [트레이싱 데이터](/docs/tracing-data-model) 전반에 걸쳐 다단계 집계를 지원하는 강력한 쿼리 엔진을 기반으로 한 유연한 셀프 서비스 분석 솔루션을 제공합니다.
프로덕션 성능을 모니터링하든, 사용자 피드백 트렌드를 분석하든, 비용과 품질 메트릭을 상호 연관시키든, 대시보드는 데이터 기반 의사 결정을 내릴 수 있는 시각화 도구를 제공합니다.

<Callout type="info">
[Fast Preview](/docs/v4)에서는 일부 대시보드 동작이 약간 변경되었습니다. 전체 참고 사항은 [Fast Preview의 대시보드 변경 사항](/faq/all/dashboard-changes-in-v4)을 참조하세요.
</Callout>

## 주요 기능

- **유연한 쿼리 엔진**: [Langfuse 데이터 모델](/docs/observability/data-model)을 기반으로 하며, 트레이스, 관찰, 사용자, 세션, 점수에 걸친 복잡한 집계를 지원합니다
- **다양한 시각화 옵션**: 커스터마이즈 가능한 레이아웃을 갖춘 라인 차트, 바 차트, 타임 시리즈 등 여러 차트 유형
- **고급 필터링**: 메타데이터, 타임스탬프, 사용자 속성, 모델 파라미터 등으로 필터링
- **다단계 집계**: 트레이스, 사용자, 세션 수준에서 데이터를 집계하여 복잡한 분석 질문에 답변
- **실시간 업데이트**: 대시보드는 LLM 애플리케이션의 실시간 데이터를 반영합니다
- **팀 협업**: 통합된 모니터링과 인사이트를 위해 프로젝트 전반에 걸쳐 대시보드를 공유
- **Langfuse 큐레이션 대시보드**: 지연 시간, 비용, Langfuse 사용량에 초점을 맞춘 사전 구축된 대시보드 세트로 빠르게 시작할 수 있습니다

## 빠른 시작

두 가지 간단한 단계로 커스텀 대시보드를 시작하거나, Langfuse의 큐레이션 대시보드를 바로 사용해보세요.

<Steps>

### 첫 번째 위젯 만들기

위젯은 LLM 애플리케이션 데이터의 특정 메트릭을 표시하는 개별 시각화 구성 요소입니다.

1. Langfuse 프로젝트에서 **Dashboards** 섹션으로 이동합니다
2. **Widgets** 탭을 선택합니다
3. **New Widget**을 클릭합니다
4. 위젯을 구성합니다:
   - **Data Source**: 트레이스, 관찰, 평가 점수 중에서 선택
   - **Metrics**: 측정할 항목 선택 (개수, 지연 시간, 비용, 점수 등)
   - **Dimensions**: 사용자, 모델, 시간, 트레이스 이름 등으로 그룹화
   - **Filters**: 특정 데이터 하위 집합으로 좁히기
   - **Chart Type**: 데이터에 가장 적합한 시각화 방식 선택
5. **Save**를 클릭하여 위젯을 저장합니다

<Video
  src="https://static.langfuse.com/docs-videos/create-widget.mov.mp4"
  gifStyle
  aspectRatio={16 / 9}
/>

### 대시보드 구성하기

여러 위젯을 결합하여 LLM 애플리케이션 성능에 대한 이야기를 전달하는 종합적인 대시보드를 만듭니다.

1. **Dashboards** 탭으로 이동합니다
2. **New Dashboard**를 클릭합니다
3. 대시보드에 설명적인 이름을 지정합니다 (예: "Production Monitoring", "Cost Analysis", "Quality Metrics")
4. 기존 위젯을 선택하거나 새로 만들어 위젯을 추가합니다
5. 드래그 앤 드롭 인터페이스를 사용해 위젯을 배치합니다
6. 중요한 메트릭을 강조하기 위해 위젯 크기를 조정합니다

  src="https://static.langfuse.com/docs-videos/create-dashboard.mov.mp4"
  gifStyle

/>

</Steps>

### 큐레이션 대시보드 활용하기

일반적인 LLM 애플리케이션 모니터링 요구 사항에 초점을 맞춘 Langfuse 큐레이션 대시보드로 분석을 빠르게 시작하세요:

- **Latency Dashboard**: 모델과 사용자 세그먼트별 응답 시간 모니터링
- **Cost Dashboard**: 시간에 따른 토큰 사용량 및 관련 비용 추적
- **Usage Dashboard**: Langfuse 플랫폼 활용도 파악

이러한 사전 구축된 대시보드는 그대로 사용하거나 자유롭게 편집할 수 있습니다. Langfuse가 관리하는 대시보드와 위젯도 모든 편집 컨트롤(드래그, 크기 조정, 삭제, 추가, 편집)을 그대로 유지합니다. 첫 변경을 시도하면 대화 상자가 열려 대시보드를 프로젝트에 복제하고 변경 사항을 그 복사본에 적용하므로, 원본을 건드리지 않고도 관리형 대시보드를 커스터마이즈할 수 있습니다.

## 홈 대시보드

프로젝트의 **Home** 페이지는 하나의 대시보드입니다. 기본적으로 Langfuse 큐레이션 기본값에서 렌더링됩니다: 클래식 개요(트레이스, 모델 비용, 지연 시간 백분위수, 점수, 모델 사용량)가 이제 일반 위젯으로 구성되어 있습니다. 각 프로젝트는 Home에 표시할 대시보드를 선택할 수 있습니다:

1. **Home**에서 헤더의 **대시보드 선택기**를 열어 원하는 대시보드를 선택하면 그 자리에서 미리 볼 수 있습니다. 선택 사항은 URL에 반영되므로 정확한 뷰를 공유할 수 있습니다.
2. **Set as default**를 클릭하면 프로젝트의 Home으로 설정됩니다. 이는 프로젝트의 모든 사용자에게 페이지를 변경하는 작업이므로, 적용 전 확인을 거칩니다.

Home도 일반 대시보드이기 때문에 모든 타일은 일반 위젯입니다. 클래식 Home 위젯(시간에 따른 트레이스, 모델 사용량, 지연 시간 백분위수, 점수 분석)을 원하는 커스텀 대시보드에 추가하거나, 처음부터 나만의 Home을 만들 수 있습니다.

## 위젯 관리 및 공유

대시보드 타일과 위젯 라이브러리 모두에서 각 위젯에는 **⋯ 메뉴**가 있습니다:

- **Copy**는 위젯을 클립보드에 복사합니다. **⌘/Ctrl+V**로 어떤 대시보드에든 붙여넣으면 위젯이 추가되고 화면에 스크롤되어 표시됩니다. 붙여넣기는 클립보드에 실제로 Langfuse 위젯이 있을 때만 활성화됩니다.
- **Duplicate**는 현재 프로젝트에 위젯의 복사본을 만듭니다.
- **Download JSON**은 위젯을 이동 가능한 파일로 내보냅니다.

위젯과 대시보드는 **버전이 지정된 JSON 형식**을 사용합니다. 위젯은 `{"$langfuseWidget": true, "version": 1}`를 가지며, 대시보드는 참조하는 각 위젯의 구성이 인라인으로 포함된 일치하는 `$langfuseDashboard` 봉투(envelope)를 가지므로 데이터베이스 ID 없이도 이동할 수 있습니다. 이 덕분에 프로젝트와 인스턴스 간에 이식이 가능합니다:

- 단일 위젯이나 전체 대시보드를 JSON 파일로 **내보내기(Export)**할 수 있습니다.
- JSON 파일을 대시보드에 드래그하여 **가져오기(Import)**할 수 있습니다. Langfuse는 파일이 Langfuse 위젯 또는 대시보드인지 검증한 후 프로젝트에 위젯을 다시 생성합니다.

## 고급 기능

### 고급 필터링 및 그룹화

Langfuse의 강력한 필터링 기능을 사용해 정확한 데이터 뷰를 만드세요:

- **메타데이터 필터**: 트레이스와 관찰에 첨부된 커스텀 메타데이터로 필터링
- **시간 기반 필터**: 특정 기간을 분석하거나 시간 범위를 비교
- **사용자 속성**: 사용자 특성 및 행동 패턴별로 세그먼트화
- **모델 파라미터**: 특정 모델 구성 또는 버전으로 필터링
- **태그 및 레이블**: 분류 필터링을 위해 [트레이스 태그](/docs/observability/features/tags) 사용
- **점수 임계값**: 품질 점수 범위나 피드백 등급으로 필터링

### 차트 유형 및 시각화

데이터에 적합한 시각화 방식을 선택하세요:

- **라인 차트**: 시간에 따른 트렌드(지연 시간, 비용, 사용량) 추적에 적합
- **바 차트**: 카테고리(모델, 사용자, 기능) 간 값 비교
- **타임 시리즈**: 시간적 세분화를 통한 실시간 메트릭 모니터링
- **파이 차트**: 카테고리형 데이터의 비율 표시 (예: 피드백 등급)

### 동적 레이아웃 및 반응형 디자인

- **드래그 앤 드롭 인터페이스**: 논리적인 그룹을 만들기 위해 위젯을 손쉽게 재배치
- **반응형 디자인**: 대시보드가 다양한 화면 크기와 기기에 맞춰 조정됨
- **위젯 크기 조정**: 더 큰 시각화로 중요한 메트릭을 강조
- **그리드 시스템**: 깔끔하고 정돈된 레이아웃을 자동으로 유지

추가 분석이나 외부 도구와의 통합을 위해 대시보드 데이터를 내보내세요. 다음을 포함한 다양한 내보내기 옵션은 [데이터 내보내기](/docs/api-and-data-platform/overview) 가이드를 참조하세요:

- 대시보드 데이터의 CSV 내보내기

### API, CLI, MCP를 통한 대시보드 관리 [#manage-dashboards-via-api]

UI의 대시보드 편집기 외에도, 대시보드와 타일 배치, 재사용 가능한 위젯을 프로그래매틱하게 생성하고 관리할 수 있습니다:

- **공개 API**: `/api/public/unstable` 하위의 엔드포인트가 대시보드, 배치, 위젯을 다룹니다. [대시보드](https://api.reference.langfuse.com/#tag/unstabledashboards) 및 [대시보드 위젯](https://api.reference.langfuse.com/#tag/unstabledashboardwidgets) API 참조를 확인하세요.
- **Langfuse CLI**: [CLI](/docs/api-and-data-platform/features/cli)는 이 엔드포인트들을 자동으로 인식하므로, 대시보드 정의를 버전 관리 시스템에 유지하고 여러 프로젝트와 환경에 적용할 수 있습니다.
- **MCP 서버**: [Langfuse MCP 서버](/docs/api-and-data-platform/features/mcp-server)는 동일한 작업을 도구로 노출하여, AI 에이전트가 변경 사항에 대한 휴먼 인 더 루프 승인을 거쳐 대시보드를 만들고 편집할 수 있게 합니다. 동일한 도구가 UI의 [Langfuse Assistant](/docs/langfuse-assistant)를 구동합니다.

<Callout type="info">
  이 엔드포인트와 MCP 도구는 **불안정(unstable)** 상태이며, 대시보드 및 위젯 계약이
  최종 확정되는 동안 변경될 수 있습니다.
</Callout>

## 사용 사례 및 예시

### 프로덕션 모니터링 대시보드

LLM 애플리케이션의 상태와 성능을 실시간으로 모니터링하세요:

- **오류율 추적**: 실패한 요청과 오류 패턴 모니터링
- **지연 시간 분석**: 서로 다른 엔드포인트의 P95, P99 응답 시간 추적
- **처리량 모니터링**: 요청량과 용량 활용도 시각화
- **모델 성능**: 모델 버전 간 정확도와 품질 메트릭 비교
- **도구 사용량 및 지연 시간**: 각 외부 도구(예: API 호출, 데이터베이스 쿼리)가 호출되는 빈도와 지연 시간 추적

### 비용 최적화 대시보드

LLM 사용 비용을 파악하고 최적화하세요:

- **토큰 사용량 트렌드**: 시간에 따른 입력/출력 토큰 소비량 추적
- **사용자별 비용**: 사용량이 많은 사용자를 식별하고 가격 전략을 최적화
- **모델 비용 비교**: 서로 다른 LLM 제공업체 및 모델 간 비용 비교
- **기능별 비용 분석**: 어떤 애플리케이션 기능이 가장 높은 비용을 유발하는지 파악

### 품질 및 사용자 경험 대시보드

LLM 애플리케이션의 품질과 사용자 만족도를 모니터링하세요:

- **사용자 피드백 트렌드**: 좋아요/싫어요 평가와 상세 피드백 추적
- **점수 분포**: 시간에 따른 품질 점수 분포 시각화
- **사용자 행동 분석**: 사용자가 다양한 기능과 어떻게 상호작용하는지 파악
- **A/B 테스트 결과**: 서로 다른 모델 버전이나 프롬프트 간 품질 메트릭 비교
