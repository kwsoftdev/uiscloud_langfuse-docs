# MCP 트레이싱

[Model Context Protocol (MCP)](https://modelcontextprotocol.io/)는 AI 에이전트가 외부 도구 및 데이터 소스와 상호작용할 수 있게 해줍니다. MCP 애플리케이션을 트레이싱할 때 클라이언트와 서버 작업은 기본적으로 별도의 트레이스를 생성하며, 이는 서비스 경계를 명확히 하는 데 유용할 수 있습니다.

하지만 트레이스 메타데이터를 클라이언트에서 서버로 전파함으로써 이러한 트레이스들을 연결하여 전체 요청 흐름에 대한 통합된 뷰를 만들 수 있습니다.

## 별도 트레이스 vs. 연결된 트레이스

**별도 트레이스**: MCP 클라이언트와 서버가 독립적인 트레이스를 생성합니다. 명확한 서비스 경계가 필요하거나 클라이언트와 서버가 서로 다른 팀에 의해 관리되는 경우에 유용합니다.

**연결된 트레이스**: MCP의 `_meta` 필드를 사용하여 클라이언트에서 서버로 트레이스 컨텍스트를 전파합니다. 이를 통해 클라이언트에서 서버를 거쳐 외부 API에 이르는 전체 요청 흐름을 보여주는 단일하고 연결된 트레이스가 생성됩니다.

## 트레이스 컨텍스트 전파하기

MCP는 `_meta` 필드 컨벤션을 통해 컨텍스트 전파를 지원합니다. 도구 호출에 OpenTelemetry 컨텍스트(W3C Trace Context 형식)를 주입함으로써 클라이언트와 서버 트레이스를 연결할 수 있습니다.

1. 클라이언트 측에서 현재 트레이스 컨텍스트를 추출합니다
2. 이를 MCP 도구 호출의 `_meta` 필드에 주입합니다
3. 서버 측에서 컨텍스트를 추출하고 복원합니다
4. 모든 서버 작업이 클라이언트의 트레이스 컨텍스트를 상속받습니다
