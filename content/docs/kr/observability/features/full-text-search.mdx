# 전문 검색

전문 검색(full-text search)을 사용하면 트레이스와 관측의 입력, 출력, 메타데이터 전반에서 특정 키워드나 문구가 등장하는 모든 위치를 찾을 수 있습니다. 이는 복잡한 애플리케이션을 디버깅할 때, 콘텐츠 내용은 기억나지만 어떤 트레이스에 속하는지 모를 경우 특히 유용합니다.

## UI에서 검색하기

트레이스 및 관측 테이블 상단의 검색창을 사용하여 `input`과 `output` 콘텐츠 전체를 검색할 수 있습니다. 일치하는 트레이스와 관측이 반환되므로, 찾고 있는 실행(run)을 빠르게 찾아내고 기존 필터 및 시간 범위 선택기와 검색을 결합할 수 있습니다.

v4의 Observations 및 Traces 테이블에서는 전문 검색이 [필터 검색창(Filter Search Bar)](/docs/observability/features/filter-search-bar)에서도 `level:ERROR`, `latency:>2`와 같은 구조화된 필터와 함께 인라인으로 제공됩니다.

## 성능 [#performance]

검색은 [ClickHouse 전문 검색](https://clickhouse.com/docs/engines/table-engines/mergetree-family/textindexes)을 사용합니다. 텍스트 인덱스를 사용하면 Langfuse가 전체 관측 페이로드를 읽기 전에 쿼리와 일치할 수 없는 데이터를 건너뛸 수 있으므로, 트레이스 양이 많은 대규모 프로젝트에서도 검색 속도가 빠르게 유지됩니다. 자세한 내용은 [ClickHouse GA 발표](https://clickhouse.com/blog/full-text-search-ga-release)에서 확인할 수 있습니다.

이 인덱스는 토큰 기반이기 때문에 쿼리는 단어의 일부(substring)가 아니라 단어 전체와 일치합니다 — `error`는 `error`와 일치하지만 `errors`와는 일치하지 않습니다 — 그리고 여러 단어로 이루어진 쿼리는 연속된 구(phrase)로 일치해야 합니다.

## API를 통한 검색 [#api]

[Observations API v2](/docs/api-and-data-platform/features/observations-api#v2)는 `input`, `output`, 문자열 `metadata` 필터에 대해 토큰 기반 전문 검색을 위한 `matches` 연산자를 지원합니다.

`/api/public/v2/observations`에서 필터를 작성할 때, 정확한 값을 알고 있다면 `=`를 사용하고, 토큰 기반 검색을 원한다면 `matches`를 사용하세요.

- `matches`는 `input`과 `output`에 대해 **대소문자를 구분하지 않으므로**, `refund failed`가 `Refund Failed`와 일치할 수 있습니다.
- 메타데이터 필터의 경우 `matches`는 **대소문자를 구분**하며, 선택한 키의 문자열 메타데이터 값에 적용됩니다.
- `contains`, `starts with`, `ends with`와 같은 부분 문자열(substring) 연산자는 `input`과 `output` 필터에서 느린 전체 콘텐츠 스캔이 필요하기 때문에 `400` 오류로 거부됩니다. 이러한 필드에서 토큰 검색을 하려면 대신 `matches`를 사용하세요.

관측 출력에 대한 `matches` 필터 예시:

```json
[
  {
    "type": "string",
    "column": "output",
    "operator": "matches",
    "value": "refund failed"
  }
]
```

정확한 메타데이터 필터 예시:

```json
[
  {
    "type": "stringObject",
    "column": "metadata",
    "key": "environment",
    "operator": "=",
    "value": "production"
  }
]
```

`GET /api/public/v2/observations`에서 JSON 배열을 URL 인코딩된 `filter` 쿼리 매개변수로 전달하세요. 전체 필터 스키마는 [Observations API v2 문서](/docs/api-and-data-platform/features/observations-api#v2)와 [API 레퍼런스](https://api.reference.langfuse.com/#tag/observations/GET/api/public/v2/observations)를 참고하세요.
