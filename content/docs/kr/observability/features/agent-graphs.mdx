# 에이전트 그래프

Langfuse의 에이전트 그래프는 복잡한 AI 에이전트 워크플로우를 시각적으로 표현하여 다단계 추론 과정과 에이전트 상호작용을 이해하고 디버깅할 수 있도록 도와줍니다.

## 시작하기

<Callout type="info">

그래프 뷰는 현재 베타 버전입니다. 피드백을 자유롭게 공유해 주세요.

</Callout>

트레이스에 그래프가 나타나는 방식은 두 가지입니다:

1. **관측값에서 추론.** 트레이스에 `span`, `event`, `generation`을 제외한 관측 유형을 가진 관측값이 있으면, Langfuse는 이를 에이전틱 트레이스로 해석하고 관측값의 타이밍과 중첩 구조로부터 자동으로 추론된 그래프를 보여줍니다.
2. **LangGraph 통합에서.** LangGraph 통합을 사용하면 그래프가 자동으로 표시됩니다.

**관측 유형**: 사용 가능한 모든 [관측 유형](/docs/observability/features/observation-types)과 설정 방법을 확인하세요.
**LangGraph**: LangGraph를 Langfuse와 네이티브로 통합하여 LLM 에이전트 트레이싱을 수행하는 엔드투엔드 예제는 [LangGraph 통합 가이드](/guides/cookbook/integration_langgraph)를 참고하세요.

## 뷰 모드: 집계형과 확장형

동일한 트레이스는 두 가지 방식으로 그려질 수 있습니다. 그래프 왼쪽 상단의 **집계형 / 확장형** 토글로 전환할 수 있으며, 선택한 값은 트레이스 간에 유지됩니다.

두 방식의 차이는 반복되는 단계를 어떻게 처리하는지에 있습니다 — **단계 이름당** 하나의 노드로 볼지, **호출당** 하나의 노드로 볼지입니다:

|               | **집계형** (기본값)                   | **확장형** ("실행된 그대로")       |
| ------------- | ------------------------------------- | ---------------------------------- |
| 노드란…       | 하나의 **고유한 단계 이름**           | 하나의 **개별 호출**               |
| 반복 호출     | 카운터가 있는 단일 노드로 축소        | 별도의 노드로 표시                 |
| 루프 / 사이클 | **사이클**로 그려짐 (엣지가 되돌아감) | **비순환 그래프(DAG)로 펼쳐짐**    |
| 보여주는 것   | 에이전트의 전반적인 _구조_            | 정확한 _실행_, 단계별로            |
| 적합한 상황   | 한눈에 구조와 복잡도를 파악할 때      | 특정 실행을 추적하거나 디버깅할 때 |

### 집계형

**집계형** 뷰는 간결한 개요를 제공합니다. 같은 이름을 공유하는 단계는 카운터가 있는 단일 노드로 병합됩니다 — `retrieve_docs (3/3)`은 해당 단계가 세 번 실행되었음을 의미합니다 — 그리고 같은 도구를 반복 호출하는 루프는 긴 체인이 아니라 사이클로 그려집니다. 이렇게 하면 바쁘게 동작하는 에이전트라도 가독성이 유지됩니다. 개별 호출을 화면에 모두 표시하지 않고도 _서로 다른_ 단계가 몇 개인지, 그리고 어떻게 연결되는지 파악할 수 있습니다. "이 에이전트는 무엇을 하며, 얼마나 복잡한가?"라는 질문에 가장 빠르게 답할 수 있는 방법입니다.

### 확장형

**확장형** 뷰는 **실행된 그대로**의 전체 프로세스를 보여줍니다. 모든 호출이 각각 하나의 노드이므로 `litellm_request` 호출이 세 번 있었다면 세 개의 개별 노드로 표시되며, 루프는 실행 순서에 따라 방향성 비순환 그래프로 펼쳐집니다. 이 뷰는 더 문자 그대로이며 트레이스 트리에 더 가깝습니다 — 특정 실행을 처음부터 끝까지 추적하거나 무언가 발생한 정확한 위치를 짚어내고 싶을 때 적합한 뷰입니다. 대규모 트레이스일수록 이 뷰에서는 집계형 뷰보다 자연스럽게 더 많은 노드가 생성됩니다.

어느 쪽도 "정답"은 아닙니다 — 서로 다른 질문에 답할 뿐입니다. 에이전트의 구조를 이해하려면 **집계형**을, 단일 실행을 따라가려면 **확장형**을 사용하세요.
