# 댓글

댓글 기능을 사용하면 팀이 Langfuse 내의 트레이스, 관측, 세션, 프롬프트에 직접 컨텍스트 노트와 논의를 추가할 수 있습니다. 이 기능은 팀원들이 다음과 같은 방식으로 협업할 수 있도록 지원합니다:

- 특정 트레이스에서 발생한 이슈나 이상 징후 표시
- 특정 모델 출력에 대한 인사이트 공유
- 엣지 케이스 및 디버깅 노트 문서화
- 프롬프트 개선 사항 조율
- 개발 및 리뷰 과정에서 피드백 남기기

## 지원되는 객체 유형

댓글은 다음 Langfuse 객체에 추가할 수 있습니다:

- **트레이스** - 전체 실행 흐름에 댓글 작성
- **관측** - 특정 LLM 호출, 스팬, 이벤트에 노트 추가
- **세션** - 사용자 상호작용 패턴에 대해 논의
- **프롬프트** - 프롬프트 버전 및 개선 사항에 대해 협업

## 댓글 추가하기

<LangTabs items={["Langfuse UI", "API"]}>

<Tab>

지원되는 각 객체 페이지에는 인터페이스에 댓글 버튼이 표시됩니다. 이 버튼은 다음을 나타냅니다:

- 현재 댓글 수 (가독성을 위해 "99+"로 상한 표시)
- 읽기 권한이 없는 경우 비활성화 상태
- 댓글을 볼 수 있거나 작성할 수 있는 경우 활성화 상태

댓글 버튼을 클릭하면 다음을 포함하는 사이드 드로어가 열립니다:

1. **댓글 스레드** - 기존의 모든 댓글을 시간순으로 표시
2. **작성기** - 새 댓글을 작성하기 위한 텍스트 영역 (쓰기 권한이 있는 경우)
3. **마크다운** - 기본적인 마크다운 서식 지원
4. **@멘션** - @멘션을 사용해 댓글에서 팀원 태그하기
5. **반응** - 댓글에 이모지 반응 추가하기

<Callout type="info">
  댓글 작성자는 본인의 댓글만 삭제할 수 있습니다. 프로젝트 관리자라도 UI를
  통해 다른 사용자의 댓글을 삭제할 수 없습니다.
</Callout>

</Tab>

<Tab>

댓글 API를 사용하면 프로그래밍 방식으로 댓글을 생성하고 조회할 수 있습니다. 모든 엔드포인트는 표준 Langfuse API 패턴을 따릅니다.

```http
GET /api/public/comments
GET /api/public/comments/{commentId}
POST /api/public/comments
```

**[API 레퍼런스](https://api.reference.langfuse.com/#tag/comments/post/api/public/comments)**

## @멘션

@멘션을 사용하여 댓글에서 팀원을 태그함으로써 중요한 발견 사항이나 논의를 알릴 수 있습니다. 이는 특정 트레이스, 관측, 이슈에 대해 누군가의 주의를 끌어야 할 때 특히 유용합니다.

### @멘션 사용 방법

1. 댓글 작성기에서 `@`를 입력하기 시작합니다
2. 프로젝트의 모든 멤버를 보여주는 자동완성 메뉴가 나타납니다
3. 목록에서 팀원을 선택하거나 계속 입력하여 필터링합니다
4. 멘션이 클릭 가능한 배지 형태로 댓글에 삽입됩니다

### 이메일 알림

댓글에서 누군가를 멘션하면:

- 해당 사용자는 댓글 내용과 컨텍스트가 담긴 이메일 알림을 받습니다
- 이메일에는 해당 객체(트레이스, 관측, 세션, 프롬프트)로 바로 이동하는 링크가 포함됩니다
- 사용자는 프로젝트별로 알림 설정을 관리할 수 있습니다

### 알림 설정 관리

팀원은 멘션에 대한 이메일 알림 수신 여부를 다음과 같이 제어할 수 있습니다:

- 프로젝트 설정으로 이동하여 알림 설정을 구성합니다
- 프로젝트별로 멘션 알림을 활성화하거나 비활성화할 수 있습니다
- 설정은 해당 프로젝트의 향후 모든 멘션에 적용됩니다

<Callout type="info">
  프로젝트 멤버만 댓글에서 멘션할 수 있습니다. 자동완성 메뉴는 현재
  프로젝트에 접근 권한이 있는 사용자만 자동으로 필터링하여 보여줍니다.
</Callout>

## 반응

전체 응답을 작성하지 않고도 빠른 확인을 위해 댓글에 이모지 반응을 추가할 수 있습니다. 반응은 동의, 감사를 표현하거나 단순히 댓글을 확인했음을 알리는 가벼운 방법입니다.

### 반응 추가 방법

- 댓글 위에 마우스를 올리면 반응 버튼이 나타납니다
- 클릭하여 반응 선택기에서 이모지를 선택합니다
- 반응은 사용자 이름과 함께 댓글 옆에 표시됩니다
- 기존 반응을 다시 클릭하면 제거됩니다

반응 기능은 댓글 스레드를 간결하게 유지하면서도 팀원들이 빠른 피드백을 제공하고 논의에 대한 참여를 표시할 수 있게 해줍니다.

## 특정 텍스트에 대한 댓글 작성

Google Docs와 유사하게, 트레이스 및 관측의 input, output, metadata 필드 내 특정 텍스트에 고정된 댓글을 추가할 수도 있습니다.

1. 트레이스나 관측의 "JSON Beta" 뷰를 사용합니다
2. 댓글을 남기고 싶은 텍스트를 선택합니다
3. 나타나는 댓글 버튼을 클릭합니다
4. 댓글이 해당 선택 영역에 정확히 고정되어 마우스를 올렸을 때 표시됩니다

이를 통해 팀원들과 LLM 응답의 특정 부분에 대해 논의하거나 정확한 이슈를 표시하기가 더 쉬워집니다.

참고: 댓글이 생성된 이후 트레이스나 관측 데이터가 업데이트되면, 해당 댓글은 참조가 변경되었을 수 있음을 나타내는 시각적 표시와 함께 "분리됨(detached)" 상태가 됩니다.
