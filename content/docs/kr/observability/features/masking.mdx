# 민감한 LLM 데이터 마스킹하기

마스킹을 사용하면 Langfuse로 전송되는 [트레이싱](/docs/tracing/overview) 데이터를 제어할 수 있습니다. 마스킹 함수를 사용하여 트레이스 데이터가 애플리케이션을 벗어나기 전에 민감한 정보를 삭제할 수 있으며, 예를 들어 다음과 같은 용도로 활용할 수 있습니다.

1. 트레이스 또는 관찰(observation)의 입력, 출력, 메타데이터에서 민감한 정보를 삭제합니다.
2. 내보내기 전에 OpenTelemetry 스팬 속성을 변환합니다.
3. 컴플라이언스나 개인정보 보호 요구사항을 위한 세밀한 데이터 필터링을 구현합니다.

저장된 데이터와 관련된 Langfuse의 데이터 보안 및 개인정보 보호 조치에 대해서는 [보안 및 컴플라이언스 개요](/security)에서 자세히 알아보세요.

## 마스킹 구성

<LangTabs items={["Python SDK", "JS/TS SDK", "LangChain (JS/TS)"]}>
<Tab title="Python SDK">

Python SDK는 두 가지 마스킹 훅을 지원합니다. 새로운 Python SDK 설정에서는 `mask_otel_spans`를 사용하는 것이 좋습니다.

| 메서드            | 상태   | 실행 시점                                                                                                                   | 적용 범위                                                                                                            |
| ----------------- | ------ | --------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| `mask_otel_spans` | 권장   | 내보내기(export) 단계에서, Langfuse가 이 클라이언트가 내보낼 OpenTelemetry 스팬을 결정하고 Langfuse 미디어 처리를 마친 이후 | 이 Langfuse 클라이언트가 내보내는 Langfuse SDK 스팬 및 서드파티 계측(instrumentation)의 원시 OpenTelemetry 스팬 속성 |
| `mask`            | 레거시 | Langfuse SDK 속성이 생성될 때 동기적으로 실행                                                                               | `start_observation()`, `update()`, `set_trace_io()` 같은 Langfuse SDK API를 통해 설정된 데이터                       |

`mask_otel_spans`를 사용하면 Langfuse 클라이언트가 OpenTelemetry 스팬 속성을 내보내기 전에 패치를 적용할 수 있습니다. 이 훅은 하나의 OpenTelemetry 내보내기 배치에 대한 읽기 전용 스냅샷을 받아, 변경이 필요한 스팬에 대해 희소(sparse) 패치를 반환합니다.

```python
from typing import Optional

from langfuse import Langfuse
from langfuse.types import (
    MaskOtelSpansParams,
    MaskOtelSpansResult,
    OtelSpanPatch,
)


def mask_otel_spans(
    *, params: MaskOtelSpansParams
) -> Optional[MaskOtelSpansResult]:
    patches = {}

    for identifier, span in params.spans.items():
        if span.instrumentation_scope_name == "openai":
            patches[identifier] = OtelSpanPatch(
                delete_attributes=(
                    "gen_ai.prompt.0.content",
                    "gen_ai.completion.0.content",
                ),
                set_attributes={"masking.applied": True},
            )

    return MaskOtelSpansResult(span_patches=patches)


langfuse = Langfuse(mask_otel_spans=mask_otel_spans)
```

### `mask_otel_spans` 동작 방식

`mask_otel_spans`는 공개된 Python SDK 타입 계약을 따릅니다.

- `params.spans`로 하나의 OpenTelemetry 내보내기 배치를 받습니다. 이 배치가 완전한 트레이스, 요청, 또는 Langfuse 관찰 트리를 포함한다는 보장은 없습니다.
- 각 키는 `OtelSpanIdentifier(trace_id, span_id)`입니다. 패치를 반환할 때는 `params.spans`에서 가져온 식별자 객체를 그대로 재사용하세요.
- 각 값은 `should_export_span` 필터링과 내보내기 단계의 미디어 처리를 거친 이후의 `OtelSpanData` 스냅샷입니다. 이 값의 `attributes`와 `resource_attributes` 매핑은 읽기 전용입니다.
- `None`을 반환하면 배치 전체가 변경되지 않은 채로 유지됩니다.
- `MaskOtelSpansResult(span_patches=...)`를 반환하면 선택한 스팬의 속성을 삭제하거나 교체할 수 있습니다.
- 패치는 희소(sparse)합니다. 변경이 필요 없는 스팬은 생략하면 됩니다.
- `OtelSpanPatch`는 먼저 `delete_attributes`를 삭제한 다음 `set_attributes`를 적용하므로, 동일한 키가 둘 다에 있으면 `set_attributes`가 우선합니다.
- `set_attributes` 값은 유효한 OpenTelemetry 속성 값이어야 합니다: 문자열, 불리언, 정수, 실수, 또는 이러한 스칼라 타입들로 구성된 동질적인(homogeneous) 시퀀스입니다.
- 이 훅은 스팬 속성만 변경할 수 있습니다. 스팬 이름, ID, 부모 관계, 리소스 속성, 이벤트, 링크, 계측 스코프는 변경할 수 없습니다.
- 이 훅은 이 Langfuse 클라이언트가 내보내는 스팬에만 영향을 미칩니다. 동일한 OpenTelemetry 스팬이 다른 익스포터로도 전송되는 경우, 해당 익스포터는 수정되지 않은 자체 사본을 받습니다.

<Callout type="info">

`mask_otel_spans`는 Langfuse Python SDK의 스팬 프로세서를 거쳐 이 Langfuse 클라이언트가 내보내는 스팬에만 영향을 미칩니다. 별도의 OpenTelemetry 스팬 프로세서나 익스포터를 통해 다른 관측성(observability) 백엔드로도 텔레메트리를 전송하는 경우, 해당 백엔드는 수정되지 않은 자체 사본의 스팬을 받습니다. Langfuse가 아닌 익스포터에 대해서는 마스킹을 별도로 구성해야 합니다.

</Callout>

<Callout type="warning">

`mask_otel_spans`는 동기적으로 실행됩니다. 일반적으로 OpenTelemetry 배치 스팬 프로세서의 워커 스레드에서 실행되므로, 애플리케이션의 메인 호출 스레드를 차단해서는 안 됩니다. `flush()` 및 종료(shutdown) 중에는 호출 스레드에서 실행될 수 있습니다.

함수를 결정적(deterministic)이고 빠르게 유지하세요. 네트워크 호출은 가능하지만, 마스킹이 느리면 OpenTelemetry 내보내기 큐가 밀려 스팬 내보내기가 지연됩니다. 마스킹 함수 내에서 오래 걸리는 작업, 무제한 재시도, 요청별 상태(request-local state), 현재 활성 스팬, 비동기 I/O는 피하세요.

</Callout>

`mask_otel_spans`가 예외를 발생시키거나 유효하지 않은 `MaskOtelSpansResult`를 반환하면, Langfuse는 내보내기 배치 전체를 폐기합니다. 개별 `OtelSpanPatch`가 유효하지 않은 경우에는 Langfuse 내보내기에서 해당 스팬만 폐기합니다. 반환된 속성 값이 유효하지 않은 경우에는 해당 속성만 삭제됩니다.

### 레거시 `mask` 동작 방식

`mask` 매개변수는 레거시 Python SDK 마스킹 훅입니다. Langfuse SDK 속성이 생성될 때 동기적으로 실행되며, `start_observation()`, `update()`, `set_trace_io()` 같은 Langfuse SDK API를 통해 설정된 데이터에만 적용됩니다. 서드파티 계측에서 나온 최종 원시 OpenTelemetry 스팬 속성은 검사하지 않습니다.

`mask`는 Langfuse SDK 속성 생성 시점에 데이터를 변환해야 하는 특별한 경우에만 사용하세요.

```python
from typing import Any

from langfuse import Langfuse


def masking_function(*, data: Any, **kwargs: Any) -> Any:
    if isinstance(data, str) and data.startswith("SECRET_"):
        return "REDACTED"

    if isinstance(data, dict):
        return {key: masking_function(data=value) for key, value in data.items()}

    if isinstance(data, list):
        return [masking_function(data=item) for item in data]

    return data


langfuse = Langfuse(mask=masking_function)
```

새로운 Python SDK 마스킹 설정에서는 `mask_otel_spans`를 사용하는 것이 좋습니다.

### 예시

#### `mask_otel_spans`로 신용카드 번호 삭제하기

이 예시는 내보내지는 OpenTelemetry 문자열 속성에서 신용카드와 유사한 패턴을 검사하여, 스팬이 Langfuse로 내보내지기 전에 일치하는 부분을 대체합니다.

```python
import re
from typing import Optional

from langfuse import Langfuse, observe
from langfuse.types import (
    MaskOtelSpansParams,
    MaskOtelSpansResult,
    OtelSpanPatch,
)

credit_card_pattern = re.compile(r"\b(?:\d[ -]*?){13,19}\b")


def mask_otel_spans(
    *, params: MaskOtelSpansParams
) -> Optional[MaskOtelSpansResult]:
    patches = {}

    for identifier, span in params.spans.items():
        replacements = {}

        for key, value in span.attributes.items():
            if isinstance(value, str):
                masked_value = credit_card_pattern.sub(
                    "[REDACTED CREDIT CARD]", value
                )

                if masked_value != value:
                    replacements[key] = masked_value

        if replacements:
            patches[identifier] = OtelSpanPatch(set_attributes=replacements)

    return MaskOtelSpansResult(span_patches=patches)


langfuse = Langfuse(mask_otel_spans=mask_otel_spans)


@observe()
def process_payment():
    return "Customer paid with card number 4111 1111 1111 1111."


result = process_payment()

print(result)
# 출력: Customer paid with card number 4111 1111 1111 1111.

# 짧게 실행되는 애플리케이션에서는 스팬을 flush 하세요.
langfuse.flush()
```

애플리케이션에서 출력되는 함수 결과 자체는 변경되지 않습니다. Langfuse로 전송되는 내보내기 스팬 속성에는 삭제(마스킹) 처리된 값이 담깁니다.

#### 이메일 주소와 전화번호 삭제하기

```python
import re
from typing import Optional

from langfuse import Langfuse
from langfuse.types import (
    MaskOtelSpansParams,
    MaskOtelSpansResult,
    OtelSpanPatch,
)

email_pattern = re.compile(r"\b[\w.-]+?@[\w.-]+?\.\w+?\b")
phone_pattern = re.compile(r"\b\d{3}[-. ]?\d{3}[-. ]?\d{4}\b")


def mask_otel_spans(
    *, params: MaskOtelSpansParams
) -> Optional[MaskOtelSpansResult]:
    patches = {}

    for identifier, span in params.spans.items():
        replacements = {}

        for key, value in span.attributes.items():
            if isinstance(value, str):
                masked_value = email_pattern.sub("[REDACTED EMAIL]", value)
                masked_value = phone_pattern.sub("[REDACTED PHONE]", masked_value)

                if masked_value != value:
                    replacements[key] = masked_value

        if replacements:
            patches[identifier] = OtelSpanPatch(set_attributes=replacements)

    return MaskOtelSpansResult(span_patches=patches)


langfuse = Langfuse(mask_otel_spans=mask_otel_spans)
```

민감한 데이터가 Langfuse로 전송되지 않도록 하려면 `LangfuseSpanProcessor`에 `mask` 함수를 제공하면 됩니다. 이 함수는 모든 관찰(observation)의 `input`, `output`, `metadata`에 적용됩니다.

이 함수는 `{ data }` 객체를 받으며, 여기서 `data`는 속성 값을 문자열화한 JSON입니다. 마스킹 처리된 데이터를 반환해야 합니다.

```ts filename="instrumentation.ts" /mask:/
import { NodeSDK } from "@opentelemetry/sdk-node";
import { LangfuseSpanProcessor } from "@langfuse/otel";

const spanProcessor = new LangfuseSpanProcessor({
  mask: ({ data }) => {
    const maskedData = data.replace(
      /\b\d{4}[- ]?\d{4}[- ]?\d{4}[- ]?\d{4}\b/g,
      "***MASKED_CREDIT_CARD***",
    );

    return maskedData;
  },
});

const sdk = new NodeSDK({
  spanProcessors: [spanProcessor],
});

sdk.start();
```

자세한 내용은 [JS/TS SDK 문서](/docs/sdk/typescript/guide)를 참고하세요.

</Tab>
<Tab title="LangChain (JS/TS)">

마스킹은 `LangfuseSpanProcessor`에서 구성합니다 — LangChain [CallbackHandler](/integrations/frameworks/langchain)가 생성하는 스팬은 자동으로 이를 거치므로, LangChain 전용 마스킹 설정은 필요하지 않습니다.

```typescript
import { NodeSDK } from "@opentelemetry/sdk-node";
import { LangfuseSpanProcessor } from "@langfuse/otel";
import { CallbackHandler } from "@langfuse/langchain";

const spanProcessor = new LangfuseSpanProcessor({
  mask: ({ data }) => {
    if (typeof data === "string" && data.startsWith("SECRET_")) {
      return "REDACTED";
    }

    return data;
  },
});

const sdk = new NodeSDK({ spanProcessors: [spanProcessor] });
sdk.start();

const handler = new CallbackHandler();
```
