# 멀티모달리티와 첨부 파일

Langfuse는 **텍스트, 이미지, 오디오, 기타 첨부 파일**을 포함한 멀티모달 트레이스를 지원합니다.

기본적으로 **[base64로 인코딩된 데이터 URI](https://developer.mozilla.org/en-US/docs/Web/URI/Schemes/data#syntax)는 Langfuse SDK에 의해 자동으로 처리됩니다**. 멀티모달 LLM에서 흔히 사용되는 페이로드에서 추출되어 Langfuse의 오브젝트 스토리지에 업로드되고 트레이스에 연결됩니다.

다음과 같은 경우에도 동작합니다.

1. 외부 URL을 통해 미디어 파일을 참조하는 경우
2. `LangfuseMedia` 클래스를 통해 SDK에서 미디어 파일 처리를 커스터마이즈하는 경우
3. Langfuse API를 직접 통합하는 경우

시작하는 방법과 내부적으로 어떻게 동작하는지 아래에서 자세히 알아보세요.

_예시_

## 제공 범위

### Langfuse Cloud

Langfuse Cloud에서 멀티모달 첨부 파일은 현재 무료로 제공됩니다. 대용량 멀티모달 트레이스와 관련된 추가 스토리지 및 컴퓨팅 비용을 반영하기 위해 가까운 미래에 새로운 가격 책정 지표를 도입할 수 있는 옵션을 보유하고 있습니다.

### 셀프 호스팅

멀티모달 첨부 파일은 현재 사용 가능합니다. Langfuse 환경 변수(`LANGFUSE_S3_MEDIA_UPLOAD_*`)를 통해 자체 오브젝트 스토리지 버킷을 구성해야 합니다. 이 환경 변수에 대한 자세한 내용은 셀프 호스팅 문서를 참조하세요. S3 호환 API는 모든 주요 클라우드 제공업체에서 지원되며 minio를 통해 셀프 호스팅할 수도 있습니다. 구성된 스토리지 버킷은 SDK를 통한 직접 업로드와 브라우저에서 직접 미디어 자산을 가져오는 것을 지원하기 위해 공개적으로 확인 가능한 호스트 이름을 가져야 합니다.

## 지원되는 미디어 형식

Langfuse는 다음을 포함한 다양한 미디어 유형을 지원합니다.

- **이미지**: .png, .jpg, .webp, .gif
- **오디오**: .mp3, .wav, .ogg
- **비디오**: .mp4, .webm, .mov
- **텍스트 및 코드**: .txt, .md, .html, .csv
- **문서**: .pdf, .docx, .xlsx, .pptx
- **데이터 및 압축 파일**: .json, .xml, .zip

## 시작하기

### Base64 데이터 URI로 인코딩된 미디어

LLM 애플리케이션에서 base64로 인코딩된 이미지, 오디오, 기타 파일을 사용한다면 최신 버전의 Langfuse SDK로 업그레이드하세요. Langfuse SDK는 base64로 인코딩된 미디어를 자동으로 감지하고 처리하며, 이를 추출하여 Langfuse 미디어 파일로 별도 업로드한 후 트레이스에 참조를 포함합니다.

이는 표준 Data URI([MDN](https://developer.mozilla.org/en-US/docs/Web/URI/Schemes/data#syntax)) 형식의 미디어(OpenAI 및 기타 LLM에서 사용하는 형식 등)와 함께 동작합니다.

이 [노트북](/guides/cookbook/example_multi_modal_traces)에는 OpenAI SDK와 LangChain을 사용하는 몇 가지 예시가 포함되어 있습니다.

### 외부 미디어 (URL)

Langfuse는 일반적인 형식을 따르는 URL을 통한 미디어 파일의 인라인 렌더링을 지원합니다. 이 경우 미디어 파일은 Langfuse의 오브젝트 스토리지에 업로드되지 않고 원본 소스에서 UI로 직접 렌더링됩니다.

지원되는 형식:

### 커스텀 첨부 파일

더 세밀한 제어가 필요하거나 미디어가 base64로 인코딩되어 있지 않은 경우, 새로운 `LangfuseMedia` 클래스를 사용하여 SDK를 통해 임의의 미디어 첨부 파일을 Langfuse에 업로드할 수 있습니다. 트레이스의 입력, 출력, 메타데이터 또는 데이터셋 항목에 포함하기 전에 미디어를 LangfuseMedia로 감싸세요. 예시는 멀티모달 문서를 참조하세요.

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab title="Python SDK">

```python
from langfuse import get_client, observe, propagate_attributes
from langfuse.media import LangfuseMedia

# Create a LangfuseMedia object from a file

with open("static/bitcoin.pdf", "rb") as pdf_file:
pdf_bytes = pdf_file.read()

# Wrap media in LangfuseMedia class

pdf_media = LangfuseMedia(content_bytes=pdf_bytes, content_type="application/pdf")

# Using with the decorator

@observe()
def process_document():
    langfuse = get_client()

    # Propagate metadata (including media) to all child observations
    with propagate_attributes(
        metadata={"document": pdf_media}
    ):
        pass

    # Or update the current span
    langfuse.update_current_span(
        input={"document": pdf_media}
    )

# Using with context managers

langfuse = get_client()

with langfuse.start_as_current_observation(as_type="span", name="analyze-document") as span: # Include media in the span input, output, or metadata
    span.update(
        input={"document": pdf_media},
        metadata={"file_size": len(pdf_bytes)}
    )

    # Process document...

    # Add results with media to the output
    span.update(output={
        "summary": "This document explains Bitcoin...",
        "original": pdf_media
    })

```

</Tab>

```typescript
import fs from "fs";
import { LangfuseMedia } from "@langfuse/core";

// Wrap media in LangfuseMedia class
const wrappedMedia = new LangfuseMedia({
  source: "bytes",
  contentBytes: fs.readFileSync(new URL("./bitcoin.pdf", import.meta.url)),
  contentType: "application/pdf",

// Optionally, access media via wrappedMedia.obj
console.log(wrappedMedia.obj);

// Include media in any trace or observation

  model: 'gpt-4',
  input: wrappedMedia,
}, {asType: "generation"});

generation3.end();

span3.end();
```

### API

Langfuse에 트레이스를 로깅하기 위해 API를 직접 사용하는 경우, 다음 단계를 따라야 합니다.

<Steps>

### Langfuse에 미디어 업로드

1. base64로 인코딩된 미디어를 사용하는 경우: Langfuse SDK가 수행하는 방식과 유사하게 트레이스 페이로드에서 이를 추출해야 합니다.
2. 업로드를 초기화하고 `mediaId`와 `presignedURL`을 받습니다: [`POST /api/public/media`](https://api.reference.langfuse.com/#tag/media/post/api/public/media).
3. 미디어 파일을 업로드합니다: `PUT [presignedURL]`.

API를 직접 사용하여 미디어 파일을 업로드하는 방법에 대해서는 이 [엔드투엔드 예시](/guides/cookbook/example_multi_modal_traces#custom-via-api)(Python)를 참조하세요.

### 트레이스/관측치에 mediaId 참조 추가

트레이스 또는 관측치의 `input`, `output`, `metadata`에서 `mediaId`를 참조하려면 [Langfuse 미디어 토큰](#media-token)을 사용하세요.

</Steps>

## 내부 동작 방식

외부 URL로 참조되지 않은 미디어 파일을 사용할 때, Langfuse는 다음과 같은 방식으로 처리합니다.

### 1. 미디어 업로드 프로세스

#### 감지 및 추출

- Langfuse는 트레이스와 관측치의 `input`, `output`, `metadata` 필드에서 미디어 파일을 지원합니다
- SDK는 성능 최적화를 위해 클라이언트 측에서 미디어를 트레이싱 데이터와 분리합니다
- 미디어 파일은 오브젝트 스토리지(AWS S3 또는 호환 스토리지)에 직접 업로드됩니다
- 원본 미디어 콘텐츠는 참조 문자열로 대체됩니다

#### 보안 및 최적화

- 업로드는 콘텐츠 검증(콘텐츠 길이, 콘텐츠 타입, 콘텐츠 SHA256 해시)이 포함된 presigned URL을 사용합니다
- 중복 제거: 이미 업로드된 파일은 단순히 `mediaId` 참조 문자열로 대체됩니다
- 파일의 고유성은 프로젝트, 콘텐츠 타입, 콘텐츠 SHA256 해시로 결정됩니다

#### 구현 세부 사항

- Python SDK: 논블로킹 실행을 위한 백그라운드 스레드 처리
- JS/TS SDK: 비동기, 논블로킹 구현
- 직접 업로드를 위한 API 지원 ([가이드](/guides/cookbook/example_multi_modal_traces#custom-via-api) 참조)

### 2. 미디어 참조 시스템 [#media-reference]

Langfuse 트레이스의 base64 데이터 URI와 래핑된 `LangfuseMedia` 객체는 필요 시 원본 페이로드를 재구성하는 데 도움이 되는 다음과 같은 표준화된 토큰 형식으로 `mediaId`에 대한 참조로 대체됩니다.

```
@@@langfuseMedia:type={MIME_TYPE}|id={LANGFUSE_MEDIA_ID}|source={SOURCE_TYPE}@@@
```

- `MIME_TYPE`: 미디어 파일의 MIME 타입, 예: `image/jpeg`
- `LANGFUSE_MEDIA_ID`: Langfuse 오브젝트 스토리지 내 미디어 파일의 ID
- `SOURCE_TYPE`: 미디어 파일의 소스 타입으로, `base64_data_uri`, `bytes`, `file` 중 하나입니다

이 토큰을 기반으로 Langfuse UI는 `mediaId`를 자동으로 감지하여 미디어 파일을 인라인으로 렌더링할 수 있습니다. `LangfuseMedia` 클래스는 참조 문자열에서 `mediaId`를 추출하는 유틸리티 함수를 제공합니다.

멀티모달 데이터셋의 경우, [SDK를 통한 실험](/docs/evaluation/experiments/experiments-via-sdk#multimodal-experiments)을 사용하여 해석된 `LangfuseMediaReference` 객체가 포함된 데이터셋 항목을 가져오고 미디어를 모델 제공업체로 전달하세요.

### 3. 미디어 참조 해석

미디어 참조가 포함된 트레이스, 관측치 또는 데이터셋 항목을 다룰 때, Langfuse 클라이언트가 제공하는 `resolve_media_references` 유틸리티 메서드를 사용하여 이를 base64 데이터 URI 형식으로 다시 변환할 수 있습니다. 이는 파인튜닝, 데이터셋 실행, 또는 생성(generation) 재실행 시 원본 콘텐츠를 다시 삽입할 때 특히 유용합니다. 이 유틸리티 메서드는 파싱된 객체를 순회하며 모든 미디어 참조 문자열을 해당하는 base64 데이터 URI 표현으로 대체한 깊은 복사본을 반환합니다.

<LangTabs items={["Python SDK", "Python SDK (v2)", "JS/TS SDK"]}>
<Tab>

```python
from langfuse import get_client

# Initialize Langfuse client
langfuse = get_client()

# Example object with media references
obj = {
    "image": "@@@langfuseMedia:type=image/jpeg|id=some-uuid|source=bytes@@@",
    "nested": {
        "pdf": "@@@langfuseMedia:type=application/pdf|id=some-other-uuid|source=bytes@@@"
    }
}

# Resolve media references to base64 data URIs
resolved_obj = langfuse.resolve_media_references(
    obj=obj,
    resolve_with="base64_data_uri"
)

# Result:
# {
#     "image": "data:image/jpeg;base64,/9j/4AAQSkZJRg...",
#     "nested": {
#         "pdf": "data:application/pdf;base64,JVBERi0xLjcK..."
#     }
# }
```
