# 트레이스 ID 및 분산 트레이싱

**트레이스 ID**는 요청이 시스템을 통과하는 동안 이를 추적하는 고유 식별자입니다. 분산 시스템에서 트레이스 ID를 사용하면 여러 서비스에 걸친 작업을 서로 연관지어 전체 요청 생명주기를 재구성할 수 있습니다.

기본적으로 Langfuse는 32자리 16진수 트레이스 ID와 16자리 16진수 관측(observation) ID를 무작위로 할당합니다.

## 트레이스 ID 생성 및 접근

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab title="Python SDK">

[`create_trace_id()`](https://python.reference.langfuse.com/langfuse#Langfuse.create_trace_id)를 사용해 트레이스 ID를 생성할 수 있습니다. `seed`를 제공하면 ID가 결정론적으로 생성됩니다. 동일한 시드를 사용하면 동일한 ID를 얻을 수 있습니다. 이는 외부 ID를 Langfuse 트레이스와 연관짓는 데 유용합니다.

```python /create_trace_id/
from langfuse import get_client, Langfuse
langfuse = get_client()

external_request_id = "req_12345"
deterministic_trace_id = langfuse.create_trace_id(seed=external_request_id)
```

[`get_current_trace_id()`](https://python.reference.langfuse.com/langfuse#Langfuse.get_current_trace_id)를 사용해 현재 트레이스 ID를 가져오고, [`get_current_observation_id`](https://python.reference.langfuse.com/langfuse#Langfuse.get_current_observation_id)를 사용해 현재 관측 ID를 가져올 수 있습니다.

또한 LangfuseSpan이나 LangfuseGeneration 객체에서 `observation.trace_id`와 `observation.id`를 사용해 트레이스 및 관측 ID에 직접 접근할 수도 있습니다.

```python /create_trace_id/ /get_current_trace_id/ /get_current_observation_id/
from langfuse import get_client, Langfuse
langfuse = get_client()

with langfuse.start_as_current_observation(as_type="span", name="my-op") as current_op:
    trace_id = langfuse.get_current_trace_id()
    observation_id = langfuse.get_current_observation_id()
    print(trace_id, observation_id)
```

</Tab>
<Tab title="JS/TS SDK">
[`createTraceId`](https://langfuse-js-git-main-langfuse.vercel.app/functions/_langfuse_tracing.createTraceId.html)를 사용해 시드로부터 결정론적 트레이스 ID를 생성할 수 있습니다.

```ts /createTraceId/ /getActiveTraceId/
import { createTraceId, startObservation } from "@langfuse/tracing";

const externalId = "support-ticket-54321";
const langfuseTraceId = await createTraceId(externalId);
```

[`getActiveTraceId`](https://langfuse-js-git-main-langfuse.vercel.app/functions/_langfuse_tracing.getActiveTraceId.html)를 사용해 활성 트레이스 ID를 가져오고, [`getActiveSpanId`](https://langfuse-js-git-main-langfuse.vercel.app/functions/_langfuse_tracing.getActiveSpanId.html)를 사용해 현재 관측 ID를 가져올 수 있습니다.

```ts /getActiveTraceId/
import { startObservation, getActiveTraceId } from "@langfuse/tracing";

await startObservation("run", async (span) => {
  const traceId = getActiveTraceId();
  console.log(`Current trace ID: ${traceId}`);
});
```

## 커스텀 트레이스 ID 설정

애플리케이션 코드를 Langfuse SDK로 감쌀 때 커스텀 트레이스 ID를 설정할 수 있습니다.

<LangTabs items={["Python SDK", "JS/TS SDK"]}>
<Tab title="Python SDK">

**컨텍스트 매니저 사용**

```python
from langfuse import get_client

langfuse = get_client()

# trace_context 매개변수로 미리 정의한 트레이스 ID 사용
with langfuse.start_as_current_observation(
    as_type="span",
    name="my-operation",
    trace_context={
        "trace_id": "abcdef1234567890abcdef1234567890",  # 32자리 16진수여야 함
        "parent_span_id": "fedcba0987654321"  # 선택 사항, 16자리 16진수
    }
) as observation:
    print(f"This observation has trace_id: {observation.trace_id}")
    # 여기에 애플리케이션 코드 작성
```

**데코레이터 사용**

```python
from langfuse import observe

@observe()
def my_operation(input):
    # 여기에 애플리케이션 코드 작성
    result = call_llm(input)
    return result

process_user_request(
    input="Hello",
    langfuse_trace_id="abcdef1234567890abcdef1234567890" # 32자리 16진수여야 함
)
```

</Tab>
<Tab title="JS/TS SDK">

**결정론적 트레이스 ID**

미리 정해진 `traceId`로 새 트레이스를 시작할 때는 부모 관측을 위한 임의의 부모 `spanId`도 함께 제공해야 합니다. 부모 span ID 값 자체는 중요하지 않으며, 유효한 16자리 16진수 문자열이기만 하면 됩니다. 이 span은 실제로 트레이스 내에 존재하는 것이 아니라 생성된 관측의 트레이스 ID 상속을 위해서만 사용되기 때문입니다.

`createTraceId`를 사용하면 시드 문자열로부터 유효하고 결정론적인 트레이스 ID를 생성할 수 있습니다. 이는 지원 티켓 ID와 같은 외부 시스템의 ID를 Langfuse 트레이스와 연관짓는 데 유용합니다.

```typescript
import { createTraceId, startObservation } from "@langfuse/tracing";

const externalId = "support-ticket-54321";

// 외부 ID로부터 유효하고 결정론적인 traceId 생성
const langfuseTraceId = await createTraceId(externalId);

// 이제 이 ID로 새 트레이스를 시작할 수 있습니다
const rootSpan = startObservation(
  "process-ticket",
  {},
  {
    parentSpanContext: {
      traceId: langfuseTraceId,
      spanId: "0123456789abcdef", // 유효한 16자리 16진수 문자열; 부모 span은 실제로 존재하지 않고 상속에만 사용되므로 값 자체는 중요하지 않음
      traceFlags: 1, // 트레이스를 샘플링됨으로 표시
    },
  }
);

// 이후 동일한 external ID로 traceId를 다시 생성해 트레이스를 채점하거나 조회할 수 있습니다
const scoringTraceId = await createTraceId(externalId);
// scoringTraceId는 langfuseTraceId와 동일합니다
```

parentSpanContext를 설정하면 생성된 span은 현재 컨텍스트의 활성 span으로부터 더 이상 상속받지 않으므로 활성 span 컨텍스트와 분리됩니다.

자세한 내용은 [Langfuse SDK instrumentation 문서](/docs/observability/sdk/instrumentation#trace-ids)를 참고하세요.
