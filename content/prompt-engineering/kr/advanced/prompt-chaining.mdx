# 프롬프트 체이닝

프롬프트 체이닝(prompt chaining)은 복잡한 작업을 일련의 더 작은 프롬프트로 나누는 방식으로, 한 단계의 출력이 다음 단계의 입력이 되며, 모든 것을 단일 호출로 처리하려 하지 않습니다.

## 작업을 왜 나눠야 하는가 [#why-split]

"이 계약서를 읽고, 핵심 조항을 추출하고, 이상한 부분을 표시하고, 요약 이메일을 작성하라"고 모델에게 요청하는 하나의 큰 프롬프트는 사실상 네 가지 별개의 작업을 한 번에, 그것도 중간에 결과를 확인할 기회 없이 수행하도록 요구하는 것입니다. 이를 추출, 표시, 작성이라는 별도의 단계로 나누면 다음과 같은 구체적인 이점이 있습니다.

- **각 단계를 독립적으로 평가할 수 있습니다.** "추출이 올바른 조항을 가져왔는가"를 그 자체로 평가하는 것이, 단일 통합 출력에서 잘못된 추출과 잘못된 요약을 뒤섞인 상태로 풀어내는 것보다 훨씬 쉽습니다.
- **각 단계에 다른 모델을 사용할 수 있습니다.** 추출 단계에는 더 저렴하고 빠른 모델로 충분할 수 있는 반면, 더 섬세한 판단이 필요한 표시 단계는 더 강력한 모델의 도움을 받을 수 있습니다 — 단일 프롬프트로는 모든 단계를 같은 모델에게 맡길 수밖에 없습니다.
- **중간 출력을 검사할 수 있습니다.** 문제가 발생했을 때, 최종 답변 하나만 놓고 역추적하는 대신 정확히 어느 단계가 잘못된 출력을 만들었는지 바로 확인할 수 있습니다.
- **단계별로 독립적으로 재시도하거나 캐시할 수 있습니다.** 3단계가 실패해도 1, 2단계를 다시 실행할 필요가 없습니다.

## 패턴 [#the-pattern]

```text
Step 1 — Extract:
"Extract the party names, effective date, and termination clause from this
contract. Return as JSON."

Step 2 — Flag (takes step 1's output as input):
"Given these extracted terms: {{step_1_output}}, flag any term that deviates
from standard commercial terms. Explain why for each flag."

Step 3 — Draft (takes both prior outputs as input):
"Write a two-paragraph summary email covering the extracted terms and flags
below, addressed to a non-lawyer: {{step_1_output}} {{step_2_output}}"
```

## 체이닝이 추가 복잡도를 감수할 만한 경우 [#when-its-worth-it]

체이닝은 단일 프롬프트에 비해 지연 시간(여러 번의 왕복 호출)과 엔지니어링 복잡도(단계 간 상태 전달, 부분 실패 처리)를 늘립니다. 다음과 같은 경우에는 이 비용을 감수할 가치가 있습니다.

- 작업이 인위적으로 나눈 것이 아니라 실제로 서로 다른 실패 양상을 가진 별개의 하위 문제들로 구성되어 있는 경우.
- 다음 단계로 넘어가기 전에 중간 결과를 검사하거나 검증해야 하는 경우(예: 추출된 데이터를 기반으로 무언가를 생성하기 전에 사용자에게 확인을 요청하는 경우).
- 서로 다른 모델을 사용할 만큼 단계별로 요구되는 정확도나 비용이 의미 있게 다른 경우.

더 단순한 작업이라면 잘 구조화된 단일 프롬프트가 대체로 더 빠르게 만들 수 있고, 실행 비용도 저렴하며, 이해하기도 쉽습니다 — 체이닝은 기본값이 아니라 실제 복잡도를 다루기 위한 도구입니다.

## 체이닝도 단일 프롬프트와 동일한 평가 원칙이 필요합니다 [#evaluation]

체인은 가장 약한 단계만큼만 신뢰할 수 있으며, 초반 단계에서의 작은 정확도 저하는 이후 체인 전체에 걸쳐 누적됩니다. 각 단계의 프롬프트를 독립된 프롬프트와 동일한 엄격함으로 다뤄야 합니다 — [프롬프트에 왜 평가가 필요한가](/prompt-engineering/kr/evaluation/why-evaluate-prompts)를 참고하세요 — 그리고 [체인 전체를 트레이싱](/docs/observability/overview)하여, 최종 답변이 잘못됐을 때 이를 하나의 불투명한 실패로 취급하는 대신 오류를 유발한 특정 단계까지 역추적할 수 있도록 해야 합니다.
